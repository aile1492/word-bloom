## 1. 엔진 설치

### 1.1 Godot 4.6 Stable 다운로드

공식 사이트에서 **Godot 4.6 Stable** 표준(Standard) 빌드를 다운로드한다.

- **다운로드 URL**: https://godotengine.org/download
- **선택 빌드**: `Godot Engine - Standard` (**.NET 빌드가 아님**)
- **지원 플랫폼**: Windows, macOS, Linux

### 1.2 Standard 빌드를 선택하는 이유

| 비교 항목 | Standard (GDScript) | .NET (C#) |
|-----------|---------------------|-----------|
| 언어 | GDScript | C# |
| 외부 의존성 | 없음 | .NET SDK 8.0+ 필요 |
| 빌드 크기 | 작음 (~40MB) | 큼 (~80MB+) |
| 모바일 배포 | 간편 | 추가 설정 필요 |
| 학습 곡선 | 낮음 (Python과 유사) | 중간 |
| 에디터 성능 | 가벼움 | 상대적으로 무거움 |

> **결론**: 본 프로젝트는 모바일 2D 퍼즐 게임이므로, GDScript의 간결함과 가벼운 빌드 크기가 적합하다. C# 의존성을 제거하여 빌드 파이프라인을 단순하게 유지한다.

### 1.3 설치 확인

다운로드한 실행 파일을 원하는 경로에 배치한다. Godot는 별도 설치 과정이 없으며, 실행 파일 하나로 동작한다.

```
# Windows 예시 배치 경로
C:\Godot\Godot_v4.6-stable_win64.exe
```

실행 후 **Project Manager** 화면이 나타나면 설치가 정상 완료된 것이다.

## 2. 프로젝트 생성 및 디렉토리 구조

### 2.1 새 프로젝트 생성

1. Godot를 실행하여 **Project Manager**를 연다.
2. `New Project` 버튼을 클릭한다.
3. 프로젝트 설정을 다음과 같이 입력한다.

| 설정 항목 | 값 |
|-----------|-----|
| Project Name | `WordPuzzle` |
| Project Path | 작업 디렉토리 경로 |
| Renderer | **Compatibility** (OpenGL ES 3.0) |
| Version Control Metadata | **Git** |

4. `Create & Edit` 버튼을 눌러 에디터를 연다.

### 2.2 디렉토리 구조

프로젝트 루트(`res://`) 아래에 다음과 같은 폴더 구조를 생성한다. 초기 단계에서 모든 폴더를 미리 만들어 두면 팀원 간 파일 배치 혼선을 방지할 수 있다.

```
res://
├── assets/              # 디자이너 관리 영역
│   ├── ui/              # UI 이미지, 아이콘 (PNG, SVG)
│   ├── fonts/           # Noto Sans KR, EN (.ttf, .otf)
│   ├── audio/           # 사운드 리소스
│   │   ├── sfx/         # 효과음 (tap, correct, hint 등)
│   │   └── bgm/         # 배경 음악
│   └── themes/          # Godot Theme 리소스 (.tres)
│
├── data/                # 단어 데이터, 게임 설정 파일
│   ├── words/
│   │   ├── en/          # English JSON word packs
│   │   └── ko/          # Korean JSON word packs
│   └── config/          # level config, constants
│
├── scenes/              # .tscn 씬 파일
│   ├── screens/         # 화면 단위 씬 (title, game, result, daily, stats)
│   ├── ui/              # 재사용 가능한 UI 컴포넌트
│   └── game/            # 게임 플레이 씬 (grid, cell, word_list)
│
├── scripts/             # .gd 스크립트 파일
│   ├── autoload/        # 싱글톤 (Autoload)
│   ├── managers/        # 게임 로직 매니저
│   ├── ui/              # UI 관련 스크립트
│   └── utils/           # 유틸리티 (hangul, seeded_random 등)
│
└── addons/              # 서드파티 플러그인
```

### 2.3 디렉토리별 역할 상세

| 디렉토리 | 담당 | 설명 |
|-----------|------|------|
| `assets/` | 디자이너 | 모든 시각/청각 리소스. 코드 파일 없음 |
| `data/` | 기획/개발 | JSON 기반 단어 팩, 레벨 설정, 상수 정의 |
| `scenes/` | 개발 | Godot 씬 트리 구조 파일 |
| `scripts/` | 개발 | GDScript 코드 전용 |
| `addons/` | 개발 | Godot AssetLib 또는 외부 플러그인 |

### 2.4 초기 폴더 생성 스크립트

Godot 에디터의 `FileSystem` 패널에서 수동으로 생성하거나, 프로젝트 루트에서 다음 명령을 실행한다.

```bash
# 프로젝트 루트 디렉토리에서 실행
mkdir -p assets/{ui,fonts,audio/{sfx,bgm},themes}
mkdir -p data/words/{en,ko} data/config
mkdir -p scenes/{screens,ui,game}
mkdir -p scripts/{autoload,managers,ui,utils}
mkdir -p addons
```

각 폴더 안에 `.gdkeep` 또는 `.gitkeep` 파일을 넣어 Git에서 빈 디렉토리를 추적하도록 한다.

```bash
find . -type d -empty -exec touch {}/.gitkeep \;
```

## 3. project.godot 핵심 설정

`project.godot` 파일은 Godot 프로젝트의 전역 설정을 관리한다. 에디터의 **Project > Project Settings** 메뉴에서 설정하거나, 파일을 직접 편집할 수 있다.

### 3.1 디스플레이/윈도우 설정

모바일 세로 모드(Portrait)에 최적화된 해상도를 설정한다.

```ini
[display]
window/size/viewport_width=1080
window/size/viewport_height=1920
window/stretch/mode="canvas_items"
window/stretch/aspect="expand"
window/handheld/orientation="portrait"
```

| 설정 | 값 | 설명 |
|------|----|------|
| `viewport_width` | 1080 | 기준 해상도 가로 |
| `viewport_height` | 1920 | 기준 해상도 세로 (FHD 세로) |
| `stretch/mode` | `canvas_items` | 2D UI가 해상도에 맞게 스케일링 |
| `stretch/aspect` | `expand` | 다양한 종횡비에서 콘텐츠 영역 확장 |
| `orientation` | `portrait` | 세로 모드 고정 |

> **참고**: `canvas_items` 모드는 2D 게임에 가장 적합한 스트레치 모드이다. UI 요소가 해상도에 따라 자연스럽게 스케일링되며, `expand` 종횡비 설정으로 노치/펀치홀 기기에서도 콘텐츠가 잘리지 않는다.

### 3.2 입력 디바이스 설정

PC에서 개발할 때 마우스 입력을 터치 입력으로 에뮬레이션하여 모바일 동작을 테스트할 수 있도록 한다.

```ini
[input_devices]
pointing/emulate_touch_from_mouse=true
```

이 설정을 활성화하면 에디터에서 마우스 클릭이 `InputEventScreenTouch`로 변환되어, 터치 기반 제스처 로직을 PC에서 바로 테스트할 수 있다.

### 3.3 렌더링 설정

모바일 호환성을 위해 **Compatibility** 렌더러(OpenGL ES 3.0)를 사용한다.

```ini
[rendering]
renderer/rendering_method="gl_compatibility"
```

| 렌더러 | API | 용도 |
|--------|-----|------|
| **Forward+** | Vulkan | 고사양 PC/콘솔 3D |
| **Mobile** | Vulkan | 모바일 3D |
| **Compatibility** | OpenGL ES 3.0 | **모바일 2D (본 프로젝트 선택)** |

> **선택 근거**: 본 프로젝트는 2D 퍼즐 게임이며, 타깃 디바이스에 구형 Android 기기를 포함한다. Compatibility 렌더러는 가장 넓은 기기 호환성을 제공한다.

### 3.4 국제화(Internationalization) 설정

한국어(ko)와 영어(en)를 지원하는 다국어 퍼즐 게임이므로, 로케일 관련 설정을 미리 구성한다.

```ini
[internationalization]
locale/translations=PackedStringArray()
locale/fallback="en"
locale/test=""
```

실제 번역 파일(.csv 또는 .po)을 추가하면 `locale/translations` 배열에 경로를 등록한다. 초기 단계에서는 비워 두고, UI 텍스트 작업 시 채운다.

### 3.5 전체 project.godot 핵심 설정 요약

```ini
[application]
config/name="WordPuzzle"
config/description="낱말 찾기 퍼즐 게임"
run/main_scene="res://scenes/screens/title_screen.tscn"
config/features=PackedStringArray("4.6")

[display]
window/size/viewport_width=1080
window/size/viewport_height=1920
window/stretch/mode="canvas_items"
window/stretch/aspect="expand"
window/handheld/orientation="portrait"

[input_devices]
pointing/emulate_touch_from_mouse=true

[rendering]
renderer/rendering_method="gl_compatibility"

[internationalization]
locale/fallback="en"
```

## 4. Autoload 싱글톤 등록

Autoload(싱글톤)는 씬 전환 시에도 유지되는 전역 노드이다. **Project > Project Settings > Autoload** 탭에서 등록한다.

### 4.1 싱글톤 목록

| 순서 | Name | Script Path | 역할 |
|------|------|-------------|------|
| 1 | `GameManager` | `res://scripts/autoload/game_manager.gd` | 게임 상태 머신, 스테이지/레벨 진행 관리 |
| 2 | `SaveManager` | `res://scripts/autoload/save_manager.gd` | 로컬 세이브/로드, `user://` 경로 JSON 직렬화 |
| 3 | `AudioManager` | `res://scripts/autoload/audio_manager.gd` | BGM/SFX 재생, 볼륨 제어, 크로스페이드 |
| 4 | `AdManager` | `res://scripts/autoload/ad_manager.gd` | AdMob SDK 래퍼, 보상형/전면 광고 관리 |
| 5 | `LayoutManager` | `res://scripts/autoload/layout_manager.gd` | 플랫폼 감지(모바일/태블릿/PC), 레이아웃 전환 |
| 6 | `CoinManager` | `res://scripts/autoload/coin_manager.gd` | 코인 경제 시스템, 힌트 소비/보상 관리 |

### 4.2 등록 순서의 중요성

Autoload는 등록 순서대로 초기화된다. 의존 관계를 고려하여 위 순서를 유지해야 한다.

```
GameManager (상태 정의)
  └─> SaveManager (상태를 저장/복원)
        └─> AudioManager (설정값 로드 후 볼륨 적용)
              └─> AdManager (게임 상태에 따라 광고 제어)
                    └─> LayoutManager (레이아웃 결정)
                          └─> CoinManager (코인 경제, SaveManager에 의존)
```

### 4.3 싱글톤 기본 템플릿

각 싱글톤 스크립트의 기본 구조는 다음과 같다.

```gdscript
# res://scripts/autoload/game_manager.gd
extends Node

## 게임 전체 상태와 스테이지 진행을 관리하는 싱글톤.

signal state_changed(new_state: StringName)

enum GameState {
    TITLE,
    PLAYING,
    PAUSED,
    RESULT,
    DAILY_CHALLENGE,
}

var current_state: GameState = GameState.TITLE


func _ready() -> void:
    process_mode = Node.PROCESS_MODE_ALWAYS  # pause 중에도 동작


func change_state(new_state: GameState) -> void:
    current_state = new_state
    state_changed.emit(GameState.keys()[new_state])
```

```gdscript
# res://scripts/autoload/save_manager.gd
extends Node

## 로컬 세이브/로드를 담당하는 싱글톤.

const SAVE_PATH: String = "user://save_data.json"

var _data: Dictionary = {}


func _ready() -> void:
    _load_data()


func save_value(key: String, value: Variant) -> void:
    _data[key] = value
    _save_data()


func load_value(key: String, default: Variant = null) -> Variant:
    return _data.get(key, default)


func _save_data() -> void:
    var file := FileAccess.open(SAVE_PATH, FileAccess.WRITE)
    if file:
        file.store_string(JSON.stringify(_data, "\t"))
        file.close()


func _load_data() -> void:
    if FileAccess.file_exists(SAVE_PATH):
        var file := FileAccess.open(SAVE_PATH, FileAccess.READ)
        if file:
            var json := JSON.new()
            var result := json.parse(file.get_as_text())
            if result == OK:
                _data = json.data
            file.close()
```

```gdscript
# res://scripts/autoload/audio_manager.gd
extends Node

## BGM/SFX 재생을 관리하는 싱글톤.

@onready var bgm_player: AudioStreamPlayer = AudioStreamPlayer.new()
@onready var sfx_player: AudioStreamPlayer = AudioStreamPlayer.new()


func _ready() -> void:
    bgm_player.bus = "BGM"
    sfx_player.bus = "SFX"
    add_child(bgm_player)
    add_child(sfx_player)


func play_bgm(stream: AudioStream, volume_db: float = 0.0) -> void:
    bgm_player.stream = stream
    bgm_player.volume_db = volume_db
    bgm_player.play()


func play_sfx(stream: AudioStream, volume_db: float = 0.0) -> void:
    sfx_player.stream = stream
    sfx_player.volume_db = volume_db
    sfx_player.play()


func stop_bgm() -> void:
    bgm_player.stop()
```

```gdscript
# res://scripts/autoload/coin_manager.gd
extends Node

## 코인 경제 시스템을 관리하는 싱글톤.

signal coins_changed(new_amount: int)

var _coins: int = 0


func _ready() -> void:
    _coins = SaveManager.load_value("coins", 0)


func get_coins() -> int:
    return _coins


func add_coins(amount: int) -> void:
    _coins += amount
    SaveManager.save_value("coins", _coins)
    coins_changed.emit(_coins)


func spend_coins(amount: int) -> bool:
    if _coins >= amount:
        _coins -= amount
        SaveManager.save_value("coins", _coins)
        coins_changed.emit(_coins)
        return true
    return false
```

### 4.4 project.godot Autoload 섹션

위 싱글톤을 등록하면 `project.godot`에 다음과 같은 섹션이 자동 생성된다.

```ini
[autoload]
GameManager="*res://scripts/autoload/game_manager.gd"
SaveManager="*res://scripts/autoload/save_manager.gd"
AudioManager="*res://scripts/autoload/audio_manager.gd"
AdManager="*res://scripts/autoload/ad_manager.gd"
LayoutManager="*res://scripts/autoload/layout_manager.gd"
CoinManager="*res://scripts/autoload/coin_manager.gd"
```

> **참고**: 경로 앞의 `*`는 해당 Autoload가 **활성화(enabled)** 상태임을 나타낸다.

## 5. 외부 도구 설정

### 5.1 Git (버전 관리)

```bash
# Git 설치 확인
git --version
# 출력 예: git version 2.44.0

# 프로젝트 초기화 (프로젝트 루트에서)
git init
git add .
git commit -m "feat: initial project setup"
```

Git은 프로젝트 생성 시 **Version Control Metadata: Git** 옵션을 선택했다면 `.gitattributes` 파일이 자동 생성되어 있다.

### 5.2 VS Code + Godot Tools 확장

Godot 내장 스크립트 에디터 대신 **VS Code**를 외부 에디터로 사용하면 더 나은 개발 경험을 얻을 수 있다.

**설치 절차:**

1. VS Code 설치: https://code.visualstudio.com/
2. 확장(Extension) 설치:
   - **Godot Tools** (`geequlim.godot-tools`) - GDScript 구문 강조, 자동 완성, 디버깅
   - **GDScript Formatter** - 코드 포매팅 (선택 사항)
3. Godot 에디터에서 외부 에디터 연결:

```
Editor > Editor Settings > Text Editor > External
├── Use External Editor: ON
├── Exec Path: C:/Users/{username}/AppData/Local/Programs/Microsoft VS Code/Code.exe
└── Exec Flags: {project} --goto {file}:{line}:{col}
```

### 5.3 Android SDK + JDK 17

Android 빌드를 위해 필요한 도구를 설치한다.

**JDK 17 설치:**

```bash
# Windows: adoptium.net에서 JDK 17 LTS 다운로드
# 또는 winget으로 설치
winget install EclipseAdoptium.Temurin.17.JDK
```

**Android SDK 설정:**

1. Android Studio를 설치하거나, 커맨드라인 도구만 다운로드한다.
2. 필요한 SDK 컴포넌트:

| 컴포넌트 | 최소 버전 |
|----------|-----------|
| Android SDK Platform | API 33 (Android 13) |
| Android SDK Build-Tools | 33.0.2+ |
| Android SDK Command-line Tools | latest |
| Android SDK Platform-Tools | latest |

3. Godot 에디터에서 Android SDK 경로 설정:

```
Editor > Editor Settings > Export > Android
├── Android SDK Path: C:/Users/{username}/AppData/Local/Android/Sdk
├── Java SDK Path: C:/Program Files/Eclipse Adoptium/jdk-17.x.x-hotspot
└── Debug Keystore: (자동 생성됨)
```

### 5.4 Keystore 생성 (Android 릴리즈 서명)

릴리즈 빌드에 사용할 Keystore를 생성한다. **이 파일은 절대 Git에 커밋하지 않는다.**

```bash
keytool -genkeypair \
  -v \
  -keystore wordpuzzle-release.keystore \
  -alias wordpuzzle \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -storepass <비밀번호> \
  -keypass <비밀번호> \
  -dname "CN=WordPuzzle, OU=Dev, O=Studio, L=Seoul, ST=Seoul, C=KR"
```

> **주의**: Keystore 파일과 비밀번호는 안전한 곳에 별도 보관한다. 분실 시 Google Play에 동일 서명으로 앱 업데이트가 불가능해진다.

### 7.2 커밋 메시지 컨벤션

[Conventional Commits](https://www.conventionalcommits.org/) 규칙을 기반으로, 프로젝트 특성에 맞게 확장한 커밋 타입을 사용한다.

| 타입 | 용도 | 예시 |
|------|------|------|
| `feat` | 새로운 기능 추가 | `feat: 힌트 시스템 구현` |
| `fix` | 버그 수정 | `fix: 한글 자모 분리 오류 수정` |
| `refactor` | 기능 변경 없는 코드 개선 | `refactor: GridManager 시그널 구조 정리` |
| `docs` | 문서 변경 | `docs: 프로젝트 셋업 가이드 작성` |
| `asset` | 리소스(이미지/사운드/폰트) 추가/변경 | `asset: Noto Sans KR 폰트 추가` |
| `chore` | 빌드, 설정, 기타 작업 | `chore: .gitignore 업데이트` |

**커밋 메시지 형식:**

```
<타입>: <간결한 설명 (한국어 또는 영어)>

[선택] 본문 - 변경 이유, 상세 내용
[선택] 관련 이슈: #123
```

**예시:**

```
feat: 그리드 드래그 선택 기능 구현

- InputEventScreenDrag 기반 셀 선택 로직
- 대각선 포함 8방향 드래그 지원
- 선택 중 시각 피드백 (하이라이트 색상)
```

### 7.3 브랜치 전략

```
main                    # 안정 릴리즈 브랜치
├── feature/*           # 기능 개발 브랜치
│   ├── feature/grid-system
│   ├── feature/hint-system
│   ├── feature/daily-challenge
│   └── ...
├── fix/*               # 버그 수정 브랜치
│   └── fix/hangul-decompose
└── admob               # 광고 SDK 통합 전용 브랜치 (별도 분리)
```

**브랜치 운영 규칙:**

| 규칙 | 설명 |
|------|------|
| `main`은 항상 실행 가능한 상태 유지 | 빌드 실패 코드 병합 금지 |
| `feature/*` 브랜치에서 개발 | main에서 분기 후 작업, 완료 시 main으로 병합 |
| `admob` 브랜치 별도 관리 | 광고 SDK는 의존성이 크므로 별도 브랜치에서 통합 후 병합 |
| 병합 전 테스트 필수 | 최소한 에디터에서 실행하여 크래시 없음을 확인 |

**광고 브랜치(`admob`)를 별도로 관리하는 이유:**

- AdMob 플러그인은 네이티브 Android/iOS 빌드에 의존하여 에디터에서 직접 테스트가 어렵다.
- 광고 관련 코드 변경이 핵심 게임 로직에 영향을 주지 않도록 격리한다.
- 광고 없는 순수 게임 빌드를 언제든 main에서 생성할 수 있다.
