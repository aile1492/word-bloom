## 1. GDScript 코딩 스타일

### 1.1 네이밍 컨벤션

| 대상 | 규칙 | 예시 |
|------|------|------|
| 변수, 함수, 파일명 | `snake_case` | `my_variable`, `do_something()`, `game_manager.gd` |
| 클래스, 노드 이름 | `PascalCase` | `GridGenerator`, `GameController`, `TitleScreen` |
| 상수 | `UPPER_SNAKE_CASE` | `MAX_GRID_SIZE`, `DEFAULT_COIN`, `MIN_WORD_LENGTH` |
| 시그널 | `snake_case` (과거형 권장) | `word_found`, `stage_completed`, `coin_changed` |
| Enum 값 | `UPPER_SNAKE_CASE` | `enum Direction { UP, DOWN, LEFT, RIGHT }` |

### 1.2 들여쓰기

- **탭(Tab) 사용** (Godot 기본 설정 그대로 유지)
- 스페이스로 변환하지 않는다

### 1.3 타입 힌트 (Type Hints)

모든 변수와 함수에 타입 힌트를 명시하는 것을 **강력 권장**한다.

```gdscript
# -- 변수 타입 힌트 --
var score: int = 0
var player_name: String = ""
var is_playing: bool = false
var grid_data: Array[Array] = []
var found_words: Array[String] = []

# -- 함수 반환 타입 힌트 --
func get_player_name() -> String:
    return player_name

func calculate_score(base: int, multiplier: float) -> int:
    return int(base * multiplier)

func is_word_valid(word: String) -> bool:
    return word.length() >= MIN_WORD_LENGTH

# -- 반환값 없는 함수 --
func update_ui() -> void:
    score_label.text = str(score)
```

### 1.4 코드 구조 순서

하나의 GDScript 파일 내에서 아래 순서를 따른다:

```gdscript
# 1. class_name 선언 (필요시)
class_name GridGenerator

# 2. extends 선언
extends Control

# 3. 상수
const MAX_GRID_SIZE: int = 12
const MIN_WORD_LENGTH: int = 2

# 4. 시그널
signal word_found(word: String, cells: Array[Vector2i])
signal grid_ready()

# 5. @export 변수 (에디터 노출)
@export var cell_size: int = 80
@export var grid_columns: int = 8

# 6. @onready 변수 (노드 참조)
@onready var grid_container: GridContainer = $GridContainer
@onready var score_label: Label = $UI/ScoreLabel

# 7. 일반 변수
var current_score: int = 0
var selected_cells: Array[Vector2i] = []

# 8. _ready(), _process() 등 내장 콜백
func _ready() -> void:
    pass

func _process(delta: float) -> void:
    pass

# 9. public 함수
func start_game() -> void:
    pass

# 10. private 함수 (언더스코어 접두사)
func _calculate_bonus() -> int:
    return 0

# 11. 시그널 핸들러 (_on_ 접두사)
func _on_button_pressed() -> void:
    pass
```

## 2. .tscn 씬 기반 개발 원칙

### 2.1 핵심 원칙: UI는 반드시 씬 파일에서 배치

```
[필수] UI 요소는 .tscn 씬 에디터에서 Control 노드로 배치한다.
[금지] 코드에서 동적으로 UI를 생성하지 않는다.
```

### 2.2 올바른 노드 참조 방식

```gdscript
# -- @onready로 노드 참조 (권장) --
@onready var grid: GridContainer = $GridContainer
@onready var word_label: Label = $UI/VBoxContainer/WordLabel
@onready var timer_bar: ProgressBar = $UI/TimerBar
@onready var popup: PopupPanel = $ResultPopup

# -- @export로 에디터에서 튜닝 가능하게 노출 --
@export var cell_size: int = 80
@export var grid_columns: int = 8
@export var animation_speed: float = 0.3
@export var hint_color: Color = Color.YELLOW
```

### 2.3 레이아웃 구성 시 Container 활용

```
[권장 Container 노드]
- VBoxContainer : 세로 배치
- HBoxContainer : 가로 배치
- GridContainer : 격자 배치 (낱말 찾기 그리드에 적합)
- MarginContainer : 여백 설정
- CenterContainer : 중앙 정렬
- PanelContainer : 배경 패널 + 자식 노드
```

### 2.4 씬 트리 구조 예시 (게임 화면)

```
GameScreen (Control)
├── Background (TextureRect)
├── SafeArea (MarginContainer)
│   ├── VBoxContainer
│   │   ├── TopBar (HBoxContainer)
│   │   │   ├── BackButton (TextureButton)
│   │   │   ├── StageLabel (Label)
│   │   │   └── CoinDisplay (HBoxContainer)
│   │   ├── GridArea (CenterContainer)
│   │   │   └── GridContainer
│   │   └── WordListPanel (PanelContainer)
│   │       └── WordListContainer (VBoxContainer)
├── InputOverlay (Control)  ← 터치 입력 감지용
└── ResultPopup (PopupPanel)
```

### 2.5 동적 생성이 허용되는 예외 상황

아래 경우에 한하여 코드에서 노드를 동적으로 생성할 수 있다:

```gdscript
# 예외 1: 격자 셀을 데이터 기반으로 반복 생성
func _create_grid_cells(grid_data: Array[Array]) -> void:
    var cell_scene: PackedScene = preload("res://scenes/components/grid_cell.tscn")
    for row in grid_data.size():
        for col in grid_data[row].size():
            var cell: Control = cell_scene.instantiate()
            cell.setup(grid_data[row][col], Vector2i(col, row))
            grid_container.add_child(cell)

# 예외 2: 찾아야 할 단어 목록을 동적으로 구성
func _create_word_list(words: Array[String]) -> void:
    var word_item_scene: PackedScene = preload("res://scenes/components/word_list_item.tscn")
    for word in words:
        var item: Control = word_item_scene.instantiate()
        item.set_word(word)
        word_list_container.add_child(item)
```

> **주의:** 동적 생성 시에도 반드시 `.tscn` 씬 파일을 `preload()`하여 `instantiate()`하는 방식을 사용한다. 코드에서 `new()`로 Control 노드를 직접 생성하지 않는다.

## 3. AI에게 전달할 필수 컨텍스트

AI(ChatGPT, Claude, Copilot 등)에게 코드를 요청할 때 **반드시 아래 컨텍스트를 함께 전달**한다. 이 정보가 빠지면 AI가 Godot 3.x 문법, C# 코드, 또는 잘못된 API를 생성할 확률이 높아진다.

### 3.1 필수 컨텍스트 블록

```
엔진: Godot 4.6 Stable (GDScript, NOT C#)
UI: Control 노드 기반 (.tscn 씬에서 배치)
입력: InputEventScreenTouch / InputEventScreenDrag
씬 전환: get_tree().change_scene_to_packed() or custom ScreenManager
Autoload: GameManager, SaveManager, AudioManager, AdManager, LayoutManager, CoinManager
데이터: JSON (FileAccess.open), 세이브 user://save_data.json
```

### 3.2 각 Autoload 역할 요약

| Autoload 이름 | 역할 |
|---------------|------|
| `GameManager` | 게임 상태 관리, 스테이지 진행, 글로벌 시그널 중계 |
| `SaveManager` | `user://save_data.json` 읽기/쓰기, 진행 상황 저장 |
| `AudioManager` | BGM/SFX 재생, 볼륨 제어 |
| `AdManager` | 광고 연동 (보상형 광고, 배너 등) |
| `LayoutManager` | 해상도/Safe Area 대응, 기기별 레이아웃 조정 |
| `CoinManager` | 코인 획득/소비, 잔액 관리 |

### 3.3 AI에게 제공해야 할 추가 정보

코드를 요청할 때 아래 정보를 가능한 한 함께 제공한다:

- 관련 `.tscn` 씬의 노드 트리 구조
- 연결된 시그널 목록
- 참조하는 다른 스크립트의 함수명
- 사용 중인 데이터 형식 (JSON 구조 등)
- 대상 플랫폼 (Android/iOS/PC)

## 4. Godot 3.x vs 4.6 문법 차이 표

AI가 Godot 3.x 문법을 출력하는 경우가 매우 빈번하므로, 아래 표를 기준으로 반드시 검증한다.

| 항목 | 3.x (잘못된 코드) | 4.6 (올바른 코드) |
|------|-------------------|-------------------|
| 비동기 대기 | `yield(obj, "signal")` | `await obj.signal` |
| 시그널 연결 | `connect("signal", target, "method")` | `signal_name.connect(callable)` |
| 최소 크기 | `$Node.rect_min_size` | `$Node.custom_minimum_size` |
| 위치 | `$Node.rect_position` | `$Node.position` |
| 크기 | `$Node.rect_size` | `$Node.size` |
| 사전 로드 | `load()` (런타임) | `preload()` (컴파일 타임) / `load()` (런타임) |
| 문자열 배열 | `PoolStringArray` | `PackedStringArray` |
| 바이트 배열 | `PoolByteArray` | `PackedByteArray` |
| 파일 열기 | `File.new()` | `FileAccess.open()` |
| 디렉토리 접근 | `Directory.new()` | `DirAccess.open()` |
| onready 변수 | `onready var x = ...` | `@onready var x = ...` |
| export 변수 | `export var x = ...` | `@export var x = ...` |
| 함수 참조 | `funcref(obj, "method")` | `Callable(obj, "method")` |
| 인스턴스 생성 | `scene.instance()` | `scene.instantiate()` |
| 렌더 서버 | `VisualServer` | `RenderingServer` |
| Tween 사용 | `tween.interpolate_property(...)` | `var tw = create_tween(); tw.tween_property(...)` |
| 타이머 대기 | `yield(get_tree().create_timer(1.0), "timeout")` | `await get_tree().create_timer(1.0).timeout` |
| 입력 확인 | `Input.is_action_just_pressed()` (동일) | `Input.is_action_just_pressed()` (동일, InputMap 설정 방식 변경) |
| 타입 힌트 | `var x = 5` | `var x: int = 5` (타입 힌트 권장) |

## 5. AI가 자주 틀리는 패턴

AI가 생성한 코드에서 아래 패턴이 발견되면 **즉시 수정**한다.

### 5.1 노드 속성 접근

```gdscript
# ❌ 잘못된 코드 (Godot 3.x)
$Node.rect_position = Vector2(100, 200)
$Node.rect_size = Vector2(300, 400)
$Node.rect_min_size = Vector2(50, 50)

# ✅ 올바른 코드 (Godot 4.6)
$Node.position = Vector2(100, 200)
$Node.size = Vector2(300, 400)
$Node.custom_minimum_size = Vector2(50, 50)
```

### 5.2 시그널 연결

```gdscript
# ❌ 잘못된 코드 (Godot 3.x)
connect("pressed", self, "_on_button")
button.connect("pressed", self, "_on_button_pressed")

# ✅ 올바른 코드 (Godot 4.6)
button.pressed.connect(_on_button)
button.pressed.connect(_on_button_pressed)
```

### 5.3 파일 입출력

```gdscript
# ❌ 잘못된 코드 (Godot 3.x)
var f = File.new()
f.open("user://data.json", File.READ)
var text = f.get_as_text()
f.close()

# ✅ 올바른 코드 (Godot 4.6)
var f: FileAccess = FileAccess.open("user://data.json", FileAccess.READ)
if f:
    var text: String = f.get_as_text()
    f.close()
```

### 5.4 타이머 대기

```gdscript
# ❌ 잘못된 코드 (Godot 3.x)
yield(get_tree().create_timer(1.0), "timeout")

# ✅ 올바른 코드 (Godot 4.6)
await get_tree().create_timer(1.0).timeout
```

### 5.5 씬 인스턴스 생성

```gdscript
# ❌ 잘못된 코드 (Godot 3.x)
var cell = cell_scene.instance()

# ✅ 올바른 코드 (Godot 4.6)
var cell: Control = cell_scene.instantiate()
```

### 5.6 렌더 서버

```gdscript
# ❌ 잘못된 코드 (Godot 3.x)
VisualServer.set_default_clear_color(Color.BLACK)

# ✅ 올바른 코드 (Godot 4.6)
RenderingServer.set_default_clear_color(Color.BLACK)
```

### 5.7 Tween 애니메이션

```gdscript
# ❌ 잘못된 코드 (Godot 3.x)
var tween = Tween.new()
add_child(tween)
tween.interpolate_property($Sprite, "modulate:a", 1.0, 0.0, 0.5)
tween.start()

# ✅ 올바른 코드 (Godot 4.6)
var tween: Tween = create_tween()
tween.tween_property($Sprite, "modulate:a", 0.0, 0.5).from(1.0)
```

### 5.8 OS 이름 확인 (이 부분은 3.x와 동일)

```gdscript
# ✅ 올바른 코드 (3.x, 4.6 모두 동일)
if OS.get_name() == "Android":
    print("안드로이드 환경")
elif OS.get_name() == "iOS":
    print("iOS 환경")
```

[요청] game_screen.gd에서 단어 선택 로직을 수정해줘.

```

**사용 예시:**

```

## 7. 파일명 규칙

### 7.1 기본 원칙

- **모든 파일명은 영어 소문자 + 언더스코어(`_`)만 사용**한다
- 한글 파일명 사용 금지 (경로 인코딩 문제 방지)
- 대문자 사용 금지
- 하이픈(`-`) 사용 금지, 반드시 언더스코어(`_`) 사용
- 공백 사용 금지

## 8. Signal 기반 통신 패턴

낱말 찾기 게임에서 노드 간 통신은 **Signal(시그널)** 을 기본으로 한다. 직접 함수 호출 대신 시그널을 통해 느슨한 결합(loose coupling)을 유지한다.

### 8.1 시그널 정의

```gdscript
# 게임 화면 스크립트 (game_screen.gd)
signal word_found(word: String, cells: Array[Vector2i])
signal stage_completed(result: Dictionary)
signal hint_used(remaining: int)
```

### 8.2 시그널 연결 (`_ready`에서 수행)

```gdscript
func _ready() -> void:
    # 자식 노드의 시그널을 부모에서 연결
    grid_input.word_selected.connect(_on_word_selected)
    hint_button.pressed.connect(_on_hint_button_pressed)

    # Autoload 시그널 연결
    GameManager.stage_completed.connect(_on_stage_completed)
    CoinManager.coin_changed.connect(_on_coin_changed)
```

### 8.3 시그널 발신

```gdscript
func _check_word(cells: Array[Vector2i]) -> void:
    var word: String = _cells_to_word(cells)
    if is_valid_word(word):
        word_found.emit(word, cells)

func _complete_stage() -> void:
    var result: Dictionary = {
        "stage_id": current_stage_id,
        "score": current_score,
        "time": elapsed_time,
        "hints_used": hints_used
    }
    stage_completed.emit(result)
```

### 8.4 Autoload 시그널 (글로벌 이벤트)

Autoload(예: `GameManager`)에 시그널을 정의하면 **어디서든 접근** 가능하다.

```gdscript
# ===== game_manager.gd (Autoload) =====
extends Node

signal stage_started(stage_id: int)
signal stage_completed(result: Dictionary)
signal game_paused()
signal game_resumed()

var current_stage_id: int = 0
var is_paused: bool = false

func start_stage(stage_id: int) -> void:
    current_stage_id = stage_id
    stage_started.emit(stage_id)

func complete_stage(result: Dictionary) -> void:
    stage_completed.emit(result)
```

```gdscript
# ===== 다른 씬에서 Autoload 시그널 사용 =====
# result_popup.gd
extends Control

func _ready() -> void:
    # Autoload 시그널에 연결 (어디서든 가능)
    GameManager.stage_completed.connect(_on_stage_completed)

func _on_stage_completed(result: Dictionary) -> void:
    _show_result(result)
```

### 8.5 시그널 통신 흐름 요약

```
[입력 감지]
InputOverlay → word_selected 시그널 발신
       ↓
[게임 로직]
GameScreen → _on_word_selected() 에서 검증
       ↓  word_found 시그널 발신
[UI 업데이트]
WordListPanel → _on_word_found() 에서 UI 갱신
ScoreDisplay → _on_word_found() 에서 점수 표시
       ↓
[스테이지 완료 판정]
GameScreen → _complete_stage()
       ↓  GameManager.complete_stage() 호출
[글로벌 이벤트]
GameManager → stage_completed 시그널 발신
       ↓
[결과 처리]
ResultPopup → _on_stage_completed() 에서 결과 팝업 표시
SaveManager → _on_stage_completed() 에서 진행 상황 저장
```

### 8.6 시그널 네이밍 가이드

| 시점 | 네이밍 패턴 | 예시 |
|------|------------|------|
| 동작 완료 후 | 과거형 | `word_found`, `stage_completed`, `coin_changed` |
| 요청/시작 시 | 동사 원형 또는 명사 | `hint_requested`, `stage_started` |
| 상태 변경 시 | 과거형 | `game_paused`, `game_resumed`, `muted_changed` |
