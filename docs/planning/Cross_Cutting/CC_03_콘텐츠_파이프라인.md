## 1. 콘텐츠 전략 개요

### 1.1 콘텐츠의 정의

본 프로젝트에서 **콘텐츠**란 게임의 핵심 자산인 **단어 데이터(Word Data)**를 의미한다. 단어 데이터는 격자(Grid)에 배치되어 플레이어가 탐색하는 대상이며, 게임 경험의 품질을 직접적으로 결정하는 요소이다.

```
콘텐츠 = 단어 데이터 (Word Data)
       = 테마별 JSON 파일 (theme_id.json)
       = 게임의 핵심 자산
```

### 1.2 콘텐츠 목표

| 항목 | 목표 |
|------|------|
| 테마당 최소 단어 수 | **100개 이상** |
| 초기 출시 테마 수 | **8개 테마** |
| 확장 목표 테마 수 | **20개 이상** (v1.6까지) |
| 지원 언어 | **EN (영어), KO (한국어)** |
| 단어 품질 | 부적절 단어 0건, 중복 0건 |

### 1.3 언어별 독립 관리 원칙

단어 데이터는 **언어별로 완전히 분리된 JSON 파일**로 관리한다. 동일한 `category_id`를 공유하지만, 각 언어의 단어 목록은 독립적으로 큐레이션된다.

```
res://data/words/en/animals.json   ← 영어 동물 테마 (독립)
res://data/words/ko/animals.json   ← 한국어 동물 테마 (독립)
```

**독립 관리의 이유:**

- 영어와 한국어의 단어 풀(pool) 크기와 난이도 분포가 다르다
- 번역이 아닌 **각 언어에 자연스러운 단어**를 선정해야 한다
- 글자 길이 기준이 언어마다 다르다 (EN: 3~12글자, KO: 2~6음절)
- 언어별로 부적절 단어 기준이 다르다

### 1.4 파이프라인 전체 흐름

```
[1. 소싱]          [2. 빌드]              [3. 검증]          [4. 배포]

 영어 소스          build_english_words.py   validate_words.py   res://data/words/en/
 (GitHub wordlists) ──────────────────────► ─────────────────► *.json

 한국어 소스        build_korean_words.py    validate_words.py   res://data/words/ko/
 (우리말샘 API)     ──────────────────────► ─────────────────► *.json

                    [블랙리스트 필터]        [스키마 검증]
                    en_blacklist.txt         [중복 검출]
                    ko_blacklist.txt         [최소 수량 검증]
```

## 2. 데이터 형식 (JSON Schema)

### 2.1 영어 JSON 스키마

각 테마별 JSON 파일은 다음 구조를 따른다.

**파일 경로 예시**: `res://data/words/en/animals.json`

```json
{
  "language": "en",
  "category_id": "animals",
  "display_name": "Animals",
  "icon": "res://assets/ui/category_animals.png",
  "words": [
    {
      "word": "TIGER",
      "length": 5,
      "difficulty": 1,
      "hint": "Striped big cat"
    },
    {
      "word": "ELEPHANT",
      "length": 8,
      "difficulty": 2,
      "hint": "Largest land animal"
    },
    {
      "word": "CHAMELEON",
      "length": 9,
      "difficulty": 3,
      "hint": "Color-changing reptile"
    }
  ]
}
```

**필드 정의 (루트):**

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `language` | `String` | O | ISO 639-1 언어 코드 (`"en"`) |
| `category_id` | `String` | O | 테마 고유 ID (snake_case, 영문) |
| `display_name` | `String` | O | 테마 표시 이름 (해당 언어) |
| `icon` | `String` | O | 테마 아이콘 리소스 경로 |
| `words` | `Array` | O | 단어 엔트리 배열 |

**필드 정의 (words 배열 요소):**

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `word` | `String` | O | 격자에 배치할 단어 (대문자, A-Z만 허용) |
| `length` | `int` | O | `word`의 글자 수 = 격자 칸 수 |
| `difficulty` | `int` | O | 난이도 등급 (1=Easy, 2=Normal, 3=Hard) |
| `hint` | `String` | O | 힌트 텍스트 (해당 언어) |

**영어 `difficulty` 기준:**

| 등급 | 값 | 글자 수 | 설명 |
|:----:|:--:|:-------:|------|
| Easy | 1 | 3~5글자 | 일상적이고 인지도 높은 단어 |
| Normal | 2 | 5~8글자 | 중간 난이도, 테마 관련 단어 |
| Hard | 3 | 8~12글자 | 긴 단어, 전문 용어 포함 |

### 2.2 한국어 JSON 스키마

**파일 경로 예시**: `res://data/words/ko/animals.json`

```json
{
  "language": "ko",
  "category_id": "animals",
  "display_name": "동물",
  "icon": "res://assets/ui/category_animals.png",
  "words": [
    {
      "word": "호랑이",
      "length": 3,
      "display": "호랑이",
      "difficulty": 1,
      "hint": "줄무늬 큰 고양이"
    },
    {
      "word": "코끼리",
      "length": 3,
      "display": "코끼리",
      "difficulty": 1,
      "hint": "긴 코를 가진 큰 동물"
    },
    {
      "word": "카멜레온",
      "length": 4,
      "display": "카멜레온",
      "difficulty": 3,
      "hint": "색이 변하는 파충류"
    }
  ]
}
```

**한국어 전용 필드:**

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `display` | `String` | O | 격자 및 UI에 표시할 텍스트 (음절 완성형) |

> **참고**: 영어에서는 `word`와 화면 표시 텍스트가 동일하지만, 한국어에서는 향후 자모 분리 기반 격자 모드 확장 가능성을 위해 `display` 필드를 명시적으로 유지한다. 현재는 `word`와 `display`가 동일하며, CC_01(다국어 지원)에서 정의한 **음절 완성형 원칙**을 따른다.

**한국어 `difficulty` 기준:**

| 등급 | 값 | 음절 수 | 설명 |
|:----:|:--:|:-------:|------|
| Easy | 1 | 2음절 | 일상적이고 인지도 높은 단어 |
| Normal | 2 | 3~4음절 | 중간 난이도, 테마 관련 단어 |
| Hard | 3 | 5~6음절 | 긴 단어, 전문 용어 포함 |

### 2.3 JSON과 GDScript Resource 간 매핑

CC_01에서 정의한 `WordPack` / `WordEntry` Resource와의 매핑 관계:

| JSON 필드 | WordPack/WordEntry 필드 | 비고 |
|-----------|------------------------|------|
| `language` | `WordPack.language` | |
| `category_id` | `WordPack.theme` | `theme` 필드에 매핑 |
| `display_name` | (별도 관리) | 테마 선택 UI에서 사용 |
| `icon` | (별도 관리) | 테마 선택 UI에서 사용 |
| `words[].word` | `WordEntry.word` | |
| `words[].length` | `WordEntry.length` | |
| `words[].display` | `WordEntry.display` | 영어: `word`와 동일 |
| `words[].difficulty` | (난이도 필터용) | 스테이지 난이도에 따라 필터링 |
| `words[].hint` | (힌트 시스템 연동) | P04_01 힌트 시스템 참조 |

| 소스 | URL | 설명 | 라이선스 |

#### 3.1.2 큐레이션 기준

영어 단어 소싱 시 다음 기준을 적용한다.

| 기준 | 규칙 | 비고 |
|------|------|------|
| 글자 수 | **3~12글자** | 3글자 미만은 너무 짧아 격자에서 찾기 어려움, 12글자 초과는 격자 크기 제한 |
| 문자 구성 | **A-Z 알파벳만** | 하이픈(-), 아포스트로피('), 숫자, 공백 포함 단어 제외 |
| 대소문자 | **모두 대문자로 변환** | 격자 표시 일관성 |
| 중복 | **테마 내 중복 제거** | 동일 테마 안에서 같은 단어 불허 |
| 부적절 단어 | **블랙리스트 기반 자동 제거** | SS3.3 참조 |
| 테마 적합성 | **수동 큐레이션 포함** | 자동 분류 후 사람이 최종 확인 |

#### 3.1.3 큐레이션 워크플로우

```
[1] GitHub raw 텍스트 다운로드
         │
         ▼
[2] 글자 수 필터 (3 <= length <= 12)
         │
         ▼
[3] 알파벳 전용 필터 (A-Z만 허용)
         │
         ▼
[4] 대문자 변환 (UPPER)
         │
         ▼
[5] 중복 제거 (set)
         │
         ▼
[6] 블랙리스트 필터 (en_blacklist.txt)
         │
         ▼
[7] 테마별 분류 (키워드 기반 + 수동 보정)
         │
         ▼
[8] difficulty 등급 부여 (글자 수 기준 자동 + 수동 보정)
         │
         ▼
[9] hint 텍스트 작성 (수동 또는 AI 보조)
         │
         ▼
[10] JSON 저장 → res://data/words/en/{theme}.json
```

### 3.3 부적절 단어 필터

#### 3.3.1 블랙리스트 관리

각 언어별로 별도의 블랙리스트 파일을 관리한다.

| 파일 | 경로 | 설명 |
|------|------|------|
| 영어 블랙리스트 | `res://data/words/_validation/en_blacklist.txt` | 영어 비속어/부적절 단어 목록 |
| 한국어 블랙리스트 | `res://data/words/_validation/ko_blacklist.txt` | 한국어 비속어/부적절 단어 목록 |

**블랙리스트 파일 형식:**

```
# en_blacklist.txt
# 한 줄에 하나의 부적절 단어 (대문자)
# '#'으로 시작하는 줄은 주석
BADWORD1
BADWORD2
OFFENSIVE1
```

#### 3.3.2 블랙리스트 적용 시점

```
빌드 시 (Python)                    런타임 시 (GDScript)
─────────────────                   ────────────────────
build_*_words.py에서                단어 데이터 로드 후
블랙리스트 파일 읽기                추가 검증 없음
→ 매칭 단어 자동 제거              (빌드 시 이미 필터됨)
→ 제거 로그 출력
```

빌드 단계에서 **100% 필터링**하므로, 런타임에서는 별도의 블랙리스트 검증을 수행하지 않는다.

#### 3.3.3 빈칸 검증용 사전

격자에서 단어가 배치되지 않은 빈 칸에 랜덤 문자를 채울 때, **우연히 부적절한 단어가 형성**되는 상황을 방지해야 한다.

| 파일 | 경로 | 설명 |
|------|------|------|

#!/usr/bin/env python3
"""
build_english_words.py
영어 단어 데이터를 소싱하여 Godot 호환 JSON으로 빌드한다.

파이프라인:
  1. GitHub raw txt 다운로드 (또는 로컬 캐시 사용)
  2. 글자 수 필터링 (3~12글자)
  3. 알파벳 전용 필터 (A-Z만 허용)
  4. 대문자 변환
  5. 중복 제거
  6. 블랙리스트 필터
  7. difficulty 자동 분류
  8. JSON 저장
"""

import json
import os
import re
import urllib.request
from typing import Optional
from config import (
    OUTPUT_DIR_EN, EN_BLACKLIST,
    EN_MIN_LENGTH, EN_MAX_LENGTH,
    EN_DIFFICULTY_THRESHOLDS, EN_DISPLAY_NAMES
)


def load_blacklist(path: str) -> set[str]:
    """블랙리스트 파일을 로드하여 set으로 반환한다."""
    if not os.path.exists(path):
        print(f"[WARN] 블랙리스트 파일 없음: {path}")
        return set()
    with open(path, "r", encoding="utf-8") as f:
        words = set()
        for line in f:
            line = line.strip()
            if line and not line.startswith("#"):
                words.add(line.upper())
        return words


def fetch_wordlist(url: str, cache_path: Optional[str] = None) -> list[str]:
    """
    GitHub raw URL에서 단어 목록을 다운로드한다.
    cache_path가 지정되면 로컬 캐시를 우선 사용한다.
    """
    if cache_path and os.path.exists(cache_path):
        print(f"[INFO] 캐시 사용: {cache_path}")
        with open(cache_path, "r", encoding="utf-8") as f:
            return [line.strip() for line in f if line.strip()]

    print(f"[INFO] 다운로드: {url}")
    response = urllib.request.urlopen(url)
    text = response.read().decode("utf-8")
    lines = [line.strip() for line in text.splitlines() if line.strip()]

    if cache_path:
        os.makedirs(os.path.dirname(cache_path), exist_ok=True)
        with open(cache_path, "w", encoding="utf-8") as f:
            f.write("\n".join(lines))

    return lines


def is_alpha_only(word: str) -> bool:
    """A-Z 알파벳만으로 구성된 단어인지 확인한다."""
    return bool(re.match(r"^[A-Z]+$", word))


def assign_difficulty(length: int) -> int:
    """글자 수 기반으로 difficulty 등급을 자동 분류한다."""
    for diff, (min_len, max_len) in EN_DIFFICULTY_THRESHOLDS.items():
        if min_len <= length <= max_len:
            return diff
    return 2  # fallback: Normal


def build_theme(theme_id: str, raw_words: list[str],
                blacklist: set[str]) -> dict:
    """
    하나의 테마에 대해 단어를 필터링하고 JSON 구조를 생성한다.
    """
    filtered = []
    seen = set()

    for word in raw_words:
        word = word.upper().strip()

        # 길이 필터
        if not (EN_MIN_LENGTH <= len(word) <= EN_MAX_LENGTH):
            continue

        # 알파벳 전용 필터
        if not is_alpha_only(word):
            continue

        # 중복 제거
        if word in seen:
            continue
        seen.add(word)

        # 블랙리스트 필터
        if word in blacklist:
            print(f"  [BLOCK] 블랙리스트 단어 제거: {word}")
            continue

        entry = {
            "word": word,
            "length": len(word),
            "difficulty": assign_difficulty(len(word)),
            "hint": ""  # hint는 수동 작성 또는 별도 스크립트로 생성
        }
        filtered.append(entry)

    # difficulty 기준 정렬 후 알파벳 순 정렬
    filtered.sort(key=lambda e: (e["difficulty"], e["word"]))

    result = {
        "language": "en",
        "category_id": theme_id,
        "display_name": EN_DISPLAY_NAMES.get(theme_id, theme_id.title()),
        "icon": f"res://assets/ui/category_{theme_id}.png",
        "words": filtered
    }
    return result


def save_json(data: dict, theme_id: str) -> None:
    """JSON 파일로 저장한다."""
    os.makedirs(OUTPUT_DIR_EN, exist_ok=True)
    path = os.path.join(OUTPUT_DIR_EN, f"{theme_id}.json")
    with open(path, "w", encoding="utf-8") as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
    print(f"[OK] 저장 완료: {path} ({len(data['words'])}개 단어)")

    """메인 빌드 함수."""

    # 테마별 원본 단어 소스 (실제 URL 또는 로컬 파일)
    # 아래는 예시 구조. 실제 소스는 테마별로 다름.
    theme_sources = {
        "animals": "tools/raw/en_animals.txt",
        "food":    "tools/raw/en_food.txt",
        "space":   "tools/raw/en_space.txt",
        "sports":  "tools/raw/en_sports.txt",
        "science": "tools/raw/en_science.txt",
        "music":   "tools/raw/en_music.txt",
        "ocean":   "tools/raw/en_ocean.txt",
        "mythology": "tools/raw/en_mythology.txt",
    }

    for theme_id, source_path in theme_sources.items():
        print(f"\n--- {theme_id} ---")
        if os.path.exists(source_path):
            with open(source_path, "r", encoding="utf-8") as f:

def fetch_from_opendict(keyword: str, api_key: str,
                         num: int = 100) -> list[str]:
    """
    국립국어원 우리말샘 API에서 키워드로 단어를 검색한다.

    Args:
        keyword: 검색 키워드
        api_key: 우리말샘 API 키
        num: 검색 결과 수 (최대 100)

    Returns:
        명사 단어 목록
    """
    import urllib.request
    import xml.etree.ElementTree as ET

    base_url = "https://opendict.korean.go.kr/api/search"
    params = (
        f"?key={api_key}"
        f"&q={keyword}"
        f"&req_type=xml"
        f"&num={num}"
        f"&part=word"
        f"&pos=1"  # 1 = 명사
        f"&sort=popular"
    )

    url = base_url + params
    try:
        response = urllib.request.urlopen(url)
        xml_text = response.read().decode("utf-8")
        root = ET.fromstring(xml_text)

        words = []
        for item in root.iter("item"):
            word_elem = item.find("word")
            if word_elem is not None and word_elem.text:

                if clean and is_all_hangul(clean):
                    words.append(clean)
        return words

    except Exception as e:
        print(f"[ERROR] API 호출 실패: {e}")
        return []

validate_words.py

  4. 글자 길이 범위 검증
  5. 블랙리스트 검증 (빌드 후에도 누락 없는지 재확인)

class ValidationResult:
    """검증 결과를 수집하는 클래스."""

    def __init__(self):
        self.errors: list[str] = []
        self.warnings: list[str] = []

    def error(self, msg: str):
        self.errors.append(msg)
        print(f"  [ERROR] {msg}")

    def warn(self, msg: str):
        self.warnings.append(msg)
        print(f"  [WARN]  {msg}")

    @property
    def passed(self) -> bool:
        return len(self.errors) == 0


# ──────────────────────────────────────────────────────
# 유틸리티
# ──────────────────────────────────────────────────────

HANGUL_BASE = 0xAC00
HANGUL_END = 0xD7A3


def is_hangul_syllable(ch: str) -> bool:
    return len(ch) == 1 and HANGUL_BASE <= ord(ch) <= HANGUL_END


def load_word_set(path: str) -> set[str]:
    """텍스트 파일에서 단어 set을 로드한다."""
    if not os.path.exists(path):
        return set()
    with open(path, "r", encoding="utf-8") as f:
        return {
            line.strip().upper()
            for line in f
            if line.strip() and not line.startswith("#")
        }

def validate_schema(data: dict, file_path: str, result: ValidationResult):
    """JSON 스키마 필수 필드 및 타입을 검증한다."""

    # 루트 필수 필드
    required_root = {
        "language": str,
        "category_id": str,
        "display_name": str,
        "icon": str,
        "words": list,
    }
    for field, expected_type in required_root.items():
        if field not in data:
            result.error(f"[{file_path}] 루트 필수 필드 누락: '{field}'")
        elif not isinstance(data[field], expected_type):
            result.error(
                f"[{file_path}] 필드 '{field}' 타입 불일치: "
                f"기대={expected_type.__name__}, "
                f"실제={type(data[field]).__name__}"
            )

    # words 배열 요소 필수 필드
    lang = data.get("language", "")
    required_word = ["word", "length", "difficulty", "hint"]
    if lang == "ko":
        required_word.append("display")

    for i, entry in enumerate(data.get("words", [])):
        for field in required_word:
            if field not in entry:
                result.error(
                    f"[{file_path}] words[{i}] 필수 필드 누락: '{field}'"
                )

        # 타입 검증
        if "word" in entry and not isinstance(entry["word"], str):
            result.error(f"[{file_path}] words[{i}].word 타입 불일치")
        if "length" in entry and not isinstance(entry["length"], int):
            result.error(f"[{file_path}] words[{i}].length 타입 불일치")
        if "difficulty" in entry and not isinstance(entry["difficulty"], int):
            result.error(f"[{file_path}] words[{i}].difficulty 타입 불일치")


def validate_duplicates(data: dict, file_path: str, result: ValidationResult):
    """테마 내 중복 단어를 검출한다."""
    seen = {}
    for i, entry in enumerate(data.get("words", [])):
        word = entry.get("word", "")
        if word in seen:
            result.error(
                f"[{file_path}] 중복 단어: '{word}' "
                f"(인덱스 {seen[word]}과 {i})"
            )
        else:
            seen[word] = i


def validate_word_count(data: dict, file_path: str, result: ValidationResult):
    """최소 단어 수를 검증한다."""
    count = len(data.get("words", []))
    if count < MIN_WORDS_PER_THEME:
        result.warn(
            f"[{file_path}] 단어 수 부족: {count}개 "
            f"(최소 {MIN_WORDS_PER_THEME}개 필요)"
        )


def validate_length_range(data: dict, file_path: str,
                           result: ValidationResult):
    """단어 길이 범위를 검증한다."""
    lang = data.get("language", "")

    if lang == "en":
        min_len, max_len = EN_MIN_LENGTH, EN_MAX_LENGTH
    elif lang == "ko":
        min_len, max_len = KO_MIN_SYLLABLES, KO_MAX_SYLLABLES
    else:
        result.warn(f"[{file_path}] 알 수 없는 언어: '{lang}'")
        return

    for i, entry in enumerate(data.get("words", [])):
        word = entry.get("word", "")
        length = entry.get("length", 0)

        # length 필드와 실제 길이 일치 확인
        actual_len = len(word)
        if length != actual_len:
            result.error(
                f"[{file_path}] words[{i}] '{word}': "
                f"length={length}, 실제={actual_len}"
            )

        # 길이 범위 검증
        if not (min_len <= actual_len <= max_len):
            result.error(
                f"[{file_path}] words[{i}] '{word}': "
                f"길이 {actual_len}이 범위 [{min_len}, {max_len}] 밖"
            )


def validate_blacklist(data: dict, file_path: str,
                        blacklist: set[str], result: ValidationResult):
    """블랙리스트에 포함된 단어가 없는지 재확인한다."""
    for i, entry in enumerate(data.get("words", [])):
        word = entry.get("word", "").upper()
        if word in blacklist:
            result.error(
                f"[{file_path}] words[{i}] 블랙리스트 단어 검출: '{word}'"
            )


def validate_difficulty(data: dict, file_path: str,
                         result: ValidationResult):
    """difficulty 값이 유효 범위(1~3)인지 검증한다."""
    for i, entry in enumerate(data.get("words", [])):
        diff = entry.get("difficulty", 0)
        if diff not in (1, 2, 3):
            result.error(
                f"[{file_path}] words[{i}] '{entry.get('word', '')}': "
                f"difficulty={diff} (유효 범위: 1~3)"
            )


def validate_hints(data: dict, file_path: str, result: ValidationResult):
    """hint가 비어있는 엔트리를 경고한다."""
    empty_count = 0
    for entry in data.get("words", []):
        if not entry.get("hint", "").strip():
            empty_count += 1

    if empty_count > 0:
        result.warn(
            f"[{file_path}] hint가 비어있는 단어: {empty_count}개"
        )


def validate_english_charset(data: dict, file_path: str,
                               result: ValidationResult):
    """영어 단어가 A-Z만으로 구성되어 있는지 검증한다."""
    if data.get("language") != "en":
        return
    import re
    for i, entry in enumerate(data.get("words", [])):
        word = entry.get("word", "")
        if not re.match(r"^[A-Z]+$", word):
            result.error(
                f"[{file_path}] words[{i}] '{word}': "
                f"A-Z 이외 문자 포함"
            )


def validate_korean_charset(data: dict, file_path: str,
                              result: ValidationResult):
    """한국어 단어가 완성형 한글 음절만으로 구성되어 있는지 검증한다."""
    if data.get("language") != "ko":
        return
    for i, entry in enumerate(data.get("words", [])):
        word = entry.get("word", "")
        for ch in word:
            if not is_hangul_syllable(ch):
                result.error(
                    f"[{file_path}] words[{i}] '{word}': "
                    f"비한글 문자 포함 ('{ch}', U+{ord(ch):04X})"
                )
                break

def validate_file(file_path: str, blacklist: set[str]) -> ValidationResult:
    """하나의 JSON 파일을 전체 검증한다."""
    result = ValidationResult()

    # JSON 파싱
    try:
        with open(file_path, "r", encoding="utf-8") as f:
            data = json.load(f)
    except json.JSONDecodeError as e:
        result.error(f"[{file_path}] JSON 파싱 오류: {e}")
        return result
    except FileNotFoundError:
        result.error(f"[{file_path}] 파일 없음")
        return result

    # 모든 검증 수행
    validate_schema(data, file_path, result)
    validate_duplicates(data, file_path, result)
    validate_word_count(data, file_path, result)
    validate_length_range(data, file_path, result)
    validate_blacklist(data, file_path, blacklist, result)
    validate_difficulty(data, file_path, result)
    validate_hints(data, file_path, result)
    validate_english_charset(data, file_path, result)
    validate_korean_charset(data, file_path, result)

    return result


def main():
    """전체 검증 실행."""
    print("=" * 60)

    # 블랙리스트 로드

    # 영어 검증
    print("\n[영어 데이터 검증]")

            if result.passed:
                print(f"  [PASS] {filename}")
    else:
        print(f"  [SKIP] 디렉토리 없음: {OUTPUT_DIR_EN}")

    # 한국어 검증
    print("\n[한국어 데이터 검증]")
    if os.path.isdir(OUTPUT_DIR_KO):
        for filename in sorted(os.listdir(OUTPUT_DIR_KO)):
            if not filename.endswith(".json"):
                continue
            total_files += 1
            path = os.path.join(OUTPUT_DIR_KO, filename)
            print(f"\n  검증 중: {filename}")
            result = validate_file(path, ko_blacklist)
            total_errors += len(result.errors)
            total_warnings += len(result.warnings)
            if result.passed:
                print(f"  [PASS] {filename}")
    else:
        print(f"  [SKIP] 디렉토리 없음: {OUTPUT_DIR_KO}")

    # 테마 완전성 검증
    print("\n[테마 완전성 검증]")
    for theme_id in INITIAL_THEMES:
        en_path = os.path.join(OUTPUT_DIR_EN, f"{theme_id}.json")
        ko_path = os.path.join(OUTPUT_DIR_KO, f"{theme_id}.json")
        if not os.path.exists(en_path):
            print(f"  [WARN] 영어 테마 파일 누락: {theme_id}.json")
            total_warnings += 1
        if not os.path.exists(ko_path):
            print(f"  [WARN] 한국어 테마 파일 누락: {theme_id}.json")
            total_warnings += 1

    # 결과 요약
    print("\n" + "=" * 60)
    print("검증 결과 요약")
    print("=" * 60)
    print(f"  검증 파일 수: {total_files}")
    print(f"  오류(ERROR):  {total_errors}")
    print(f"  경고(WARN):   {total_warnings}")

    if total_errors > 0:
        print("\n  [FAIL] 검증 실패 -- 오류를 수정하세요.")
        sys.exit(1)
    else:
        print("\n  [PASS] 모든 검증 통과")
        sys.exit(0)


if __name__ == "__main__":

|------|------|
| 테마 브라우저 | 드롭다운에서 테마를 선택하면 단어 목록을 테이블로 표시 |
| 통계 대시보드 | 테마당 단어 수, difficulty 분포, 평균 글자 수 표시 |
| 검증 실행 | 에디터 내에서 validate_words.py에 해당하는 검증을 실행 |
| 단어 검색 | 특정 단어가 어느 테마에 포함되어 있는지 검색 |

### 7.2 테마 추가 마법사

신규 테마를 추가할 때의 작업을 자동화하는 에디터 마법사 기능이다.

**마법사 단계:**

```
[1단계] 테마 기본 정보 입력
  - theme_id (snake_case)
  - 영어 display_name
  - 한국어 display_name
  - 아이콘 이미지 선택

[2단계] 원본 단어 입력
  - 텍스트 영역에 단어를 줄 단위로 입력
  - 또는 .txt 파일 드래그 앤 드롭

[3단계] 자동 빌드 + 검증
  - 필터링, 중복 제거, 블랙리스트 적용
  - 검증 결과 표시

[4단계] 저장
  - JSON 파일 자동 생성
  - config.py 자동 업데이트
```

## 8. 콘텐츠 업데이트 전략

### 8.1 앱 업데이트를 통한 새 테마 추가

기본적으로 새로운 테마는 **앱 업데이트(스토어 배포)**를 통해 추가한다.

```
[v1.0 출시] 8개 테마 (animals, food, space, sports, science, music, ocean, mythology)
     │
     ▼
[v1.1 업데이트] +2개 테마 (nature, technology)
     │
     ▼
[v1.2 업데이트] +2개 테마 (history, geography)
     │
     ▼
  ... 이하 반복 ...
```

### 8.2 시즌 이벤트 테마

시즌 이벤트 테마(예: `holidays`)는 **앱에 사전 탑재**하되, **날짜 기반으로 활성화/비활성화**한다.

```gdscript
# season_manager.gd (pseudocode)

## 시즌 이벤트 테마 활성화 조건을 정의한다.
const SEASONAL_THEMES: Dictionary = {
    "holidays_christmas": {
        "start": {"month": 12, "day": 20},
        "end":   {"month": 12, "day": 31}
    },
    "holidays_newyear": {
        "start": {"month": 12, "day": 30},
        "end":   {"month": 1, "day": 5}
    },
    "holidays_chuseok": {
        "start": {"month": 9, "day": 10},  # 추석 전후 (매년 날짜 변동)
        "end":   {"month": 9, "day": 20}
    }
}


## 현재 날짜 기준으로 활성화된 시즌 테마 목록을 반환한다.
func get_active_seasonal_themes() -> Array[String]:
    var now: Dictionary = Time.get_datetime_dict_from_system()
    var month: int = now["month"]
    var day: int = now["day"]
    var active: Array[String] = []

    for theme_id in SEASONAL_THEMES:
        var config: Dictionary = SEASONAL_THEMES[theme_id]
        if _is_date_in_range(month, day, config["start"], config["end"]):
            active.append(theme_id)

    return active
```

### 8.3 Remote Config (향후)

Phase 3 이후, 서버 기반 Remote Config를 도입하여 앱 업데이트 없이 다음 항목을 동적으로 변경할 수 있도록 한다.

| 항목 | 설명 | 우선순위 |
|------|------|:--------:|
| 테마 해금 조건 | 이벤트 기간 중 해금 조건 완화 | 중 |
| 시즌 이벤트 날짜 | 추석 등 음력 기반 이벤트 날짜 보정 | 중 |
| 단어 데이터 핫픽스 | 부적절 단어 발견 시 긴급 제거 | 높 |

> **참고**: Remote Config 구현은 P06_01(수익화 시스템)의 서버 인프라 구축과 연계하여 계획한다.

## 9. 품질 기준

### 9.1 정량적 품질 기준

| 항목 | 기준 | 검증 방법 | 실패 시 조치 |
|------|------|-----------|-------------|
| 테마당 최소 단어 수 | **100개** | `validate_words.py` | 단어 추가 소싱 |
| 영어 단어 길이 | **3~12글자** | `validate_words.py` | 범위 외 단어 제거 |
| 한국어 단어 길이 | **2~6음절** | `validate_words.py` | 범위 외 단어 제거 |
| 부적절 단어 | **0건** | `validate_words.py` + 블랙리스트 | 블랙리스트 업데이트 |
| JSON 유효성 | **100% 통과** | `validate_words.py` | JSON 수정 |
| 중복 단어 | **0건** | `validate_words.py` | 중복 제거 |
| difficulty 값 범위 | **1~3** | `validate_words.py` | 값 보정 |
| length 필드 정합성 | **100% 일치** | `validate_words.py` | 자동 재계산 |

### 9.2 정성적 품질 기준

| 항목 | 기준 | 검증 방법 |
|------|------|-----------|
| 테마 적합성 | 단어가 해당 테마에 적절한가 | 수동 리뷰 |
| 일상성 (한국어) | 일반 사용자가 알 만한 단어인가 | 수동 리뷰 |
| hint 품질 | 힌트가 단어를 유추하는 데 도움이 되는가 | 수동 리뷰 |
| difficulty 적절성 | 자동 분류된 난이도가 체감과 일치하는가 | 플레이테스트 |

### 9.3 빌드 전 체크리스트

새로운 단어 데이터를 빌드할 때마다 다음을 확인한다.

```
[ ] 블랙리스트 파일이 최신 상태인가
[ ] 원본 소스 파일에 올바른 인코딩(UTF-8)이 적용되었는가
[ ] Python 3.8+ 환경에서 스크립트가 실행되는가
[ ] validate_words.py 실행 결과 ERROR가 0건인가
[ ] WARN 항목을 검토하고 필요한 조치를 완료했는가
[ ] hint가 비어있는 단어에 대해 hint를 작성했는가
```
