## 2. 소셜 전략 개요

### 2.1 핵심 원칙

본 프로젝트의 소셜 시스템은 **자체 서버(Backend) 없이 플랫폼 API만을 활용**하는 것을 원칙으로 한다.

```
[핵심 아키텍처]

  +-----------------+         +-------------------------+
  |  Word Search    |         |   플랫폼 서비스          |
  |  Puzzle Game    | ------> |   (GPGS / Game Center)  |
  |  (Godot 4.6)   |         +-------------------------+
  +-----------------+                |
        |                            |
        v                            v
  +----------------+         +------------------+
  | 로컬 세이브     |         | 리더보드 / 업적   |
  | (user://save)  |         | (플랫폼 관리)     |
  +----------------+         +------------------+
```

**설계 근거:**

1. **서버 비용 제로** - 별도 서버 인프라 불필요, 유지보수 부담 없음
2. **플랫폼 신뢰성** - Google/Apple이 서버 안정성, 보안, 확장성 보장
3. **네이티브 UI 활용** - 플랫폼 리더보드 UI를 그대로 사용하여 개발 비용 절감
4. **사용자 친숙성** - 플레이어가 이미 익숙한 플랫폼 소셜 인터페이스 제공
5. **자체 DB 불필요** - 모든 점수/업적 데이터는 플랫폼이 저장하고, 로컬에는 캐시만 유지

### 2.2 플랫폼별 서비스 매핑

| 기능 | Android | iOS |

## 3. 리더보드 시스템

### 3.1 리더보드 종류

총 5개의 리더보드를 운영하며, 각 리더보드는 플랫폼 콘솔(Google Play Console / App Store Connect)에서 사전 등록해야 한다.

| 리더보드 ID | 이름 | 점수 기준 | 정렬 | 초기화 주기 |
|------------|------|-----------|------|------------|
| `daily_challenge_score` | Daily Challenge 점수 | 일일 최고 점수 | 내림차순 (Larger is better) | 매일 |
| `daily_challenge_time` | Daily Challenge 시간 | 클리어 시간(ms) | 오름차순 (Smaller is better) | 매일 |
| `weekly_total_score` | 주간 누적 점수 | 7일간 총 점수 | 내림차순 (Larger is better) | 매주 월요일 |
| `classic_best_stage` | Classic 최고 스테이지 | 도달 스테이지 번호 | 내림차순 (Larger is better) | 없음 (전체 기간) |
| `marathon_high_score` | Marathon 최고 점수 | 누적 점수 | 내림차순 (Larger is better) | 없음 (전체 기간) |

**점수 기준 상세:**

- **daily_challenge_score**: 보너스 워드, 콤보, 시간 보너스가 합산된 최종 점수. 하루 중 최고 점수만 반영.
- **daily_challenge_time**: 밀리초(ms) 단위. 모든 단어를 찾는 데 걸린 순수 시간. 일시정지 시간 제외.
- **weekly_total_score**: 월~일 7일간 Daily Challenge 점수 합산. 월요일 00:00(UTC) 기준 초기화.
- **classic_best_stage**: 도달한 가장 높은 스테이지 번호. 게임 오버 없이 도달한 최고 기록.
- **marathon_high_score**: Marathon 모드에서 얻은 전체 누적 점수. 세션 종료 시점 점수.

### 3.2 리더보드 표시 시간 범위

각 리더보드는 플랫폼 네이티브 UI에서 다음 시간 범위를 지원한다:

| 범위 | 설명 |
|------|------|
| Daily | 오늘 기록 (GPGS 자동 지원) |
| Weekly | 이번 주 기록 (GPGS 자동 지원) |
| All Time | 전체 기간 기록 |

### 3.3 Google Play Games Services 연동

#### 3.3.1 사전 설정

1. Google Play Console에서 게임 서비스 등록
2. 리더보드 5개 생성 (각 ID를 콘솔에서 발급)
3. Godot 프로젝트에 `godot-play-game-services` 플러그인 설치
4. `AndroidManifest.xml`에 게임 서비스 메타데이터 추가
5. `export_presets.cfg`에 플러그인 활성화

#### 3.3.2 연동 흐름

```
[앱 시작]
    |
    v
[GPGS 자동 로그인 시도]
    |
    +-- 성공 --> [signed_in 상태, 캐시된 점수 제출]
    |
    +-- 실패 --> [로컬 모드, 나중에 수동 로그인 가능]
```

#### 3.3.3 API 호출 요약

| 동작 | API 호출 |
|------|----------|
| 로그인 | `GamesSignInClient.signIn()` |
| 점수 제출 | `LeaderboardsClient.submitScore(id, score)` |
| 리더보드 UI 표시 | `LeaderboardsClient.getLeaderboardIntent(id)` |
| 전체 리더보드 UI | `LeaderboardsClient.getAllLeaderboardsIntent()` |

### 3.5 소셜 로그인 보상

Google Play Games (Android) 또는 Game Center (iOS) 연동 시 **150코인** 1회성 보상을 지급한다.

**트리거 위치**: SettingsPopup 내 "🎮 Google Play Games 로그인" 버튼

**동작 플로우**:
1. 사용자가 SettingsPopup에서 소셜 로그인 버튼 탭
2. 플랫폼별 인증 플로우 실행
3. 인증 성공 시:
   - `social_login_rewarded`가 false이면: 150코인 지급 + 코인 획득 연출
   - `social_login_rewarded`를 true로 설정
   - 버튼 텍스트: "✅ 연동 완료" (비활성)
4. 인증 실패 시: 에러 토스트 메시지

**SaveManager 저장**: `social_login_rewarded` (bool)

**GDScript 의사코드**:
```gdscript
func _on_social_login_success() -> void:
    if not SaveManager.get_value("social_login_rewarded", false):
        SaveManager.add_coins(150)
        SaveManager.set_value("social_login_rewarded", true)
        ToastManager.show("소셜 로그인 보상: 150코인 획득!")
    _update_social_button_state()

func _update_social_button_state() -> void:
    if SaveManager.get_value("social_login_rewarded", false):
        social_login_button.text = "✅ 연동 완료"
        social_login_button.disabled = true
```

### 3.6 leaderboard_manager.gd

```gdscript
## leaderboard_manager.gd
## 리더보드 관리 매니저 - 플랫폼별 리더보드 점수 제출 및 표시를 담당한다.
## Autoload 싱글톤으로 등록한다.
class_name LeaderboardManager
extends Node

## 리더보드 ID 상수 정의
## Google Play Console / App Store Connect에서 등록한 실제 ID로 교체해야 한다.
const LEADERBOARD_IDS := {
	"daily_challenge_score": {
		"gpgs": "CgkI_DAILY_SCORE_ID",     # Google Play Console에서 발급된 ID
		"gamecenter": "daily_challenge_score" # App Store Connect에서 설정한 ID
	},
	"daily_challenge_time": {
		"gpgs": "CgkI_DAILY_TIME_ID",
		"gamecenter": "daily_challenge_time"
	},
	"weekly_total_score": {
		"gpgs": "CgkI_WEEKLY_SCORE_ID",
		"gamecenter": "weekly_total_score"
	},
	"classic_best_stage": {
		"gpgs": "CgkI_CLASSIC_STAGE_ID",
		"gamecenter": "classic_best_stage"
	},
	"marathon_high_score": {
		"gpgs": "CgkI_MARATHON_SCORE_ID",
		"gamecenter": "marathon_high_score"
	}
}

## 현재 플랫폼 열거
enum Platform { ANDROID, IOS, OTHER }

## 로그인 상태
var is_signed_in: bool = false

## 현재 플랫폼
var current_platform: Platform = Platform.OTHER

## 제출 대기 중인 점수 캐시 (오프라인 시 축적)
## 구조: Array[Dictionary] - 각 항목은 {leaderboard_key: String, score: int}
var _pending_scores: Array[Dictionary] = []

## 시그널 정의
signal sign_in_succeeded
signal sign_in_failed(error_message: String)
signal score_submitted(leaderboard_key: String, score: int)
signal score_submit_failed(leaderboard_key: String, error_message: String)


func _ready() -> void:
	_detect_platform()
	_attempt_auto_sign_in()


## 플랫폼 자동 감지
func _detect_platform() -> void:
	var os_name := OS.get_name()
	if os_name == "Android":
		current_platform = Platform.ANDROID
	elif os_name == "iOS":
		current_platform = Platform.IOS
	else:
		current_platform = Platform.OTHER


## 앱 시작 시 자동 로그인 시도
func _attempt_auto_sign_in() -> void:
	match current_platform:
		Platform.ANDROID:
			_sign_in_gpgs()
		Platform.IOS:
			_sign_in_gamecenter()
		Platform.OTHER:
			# PC/TV 등 - 리더보드 미지원 플랫폼
			is_signed_in = false


## ========== 로그인 ==========

## Google Play Games Services 로그인
func _sign_in_gpgs() -> void:
	if not Engine.has_singleton("GodotPlayGameServices"):
		push_warning("LeaderboardManager: GPGS 플러그인을 찾을 수 없음")
		is_signed_in = false
		sign_in_failed.emit("GPGS plugin not found")
		return

	var gpgs = Engine.get_singleton("GodotPlayGameServices")
	gpgs.signIn()
	# 콜백 시그널 연결
	if not gpgs.is_connected("signInSuccess", _on_gpgs_sign_in_success):
		gpgs.connect("signInSuccess", _on_gpgs_sign_in_success)
	if not gpgs.is_connected("signInFailed", _on_gpgs_sign_in_failed):
		gpgs.connect("signInFailed", _on_gpgs_sign_in_failed)


func _on_gpgs_sign_in_success() -> void:
	is_signed_in = true
	sign_in_succeeded.emit()
	_flush_pending_scores()


func _on_gpgs_sign_in_failed(error: String) -> void:
	is_signed_in = false
	sign_in_failed.emit(error)


## Game Center 로그인
func _sign_in_gamecenter() -> void:
	if not Engine.has_singleton("GameCenter"):
		push_warning("LeaderboardManager: GameCenter 플러그인을 찾을 수 없음")
		is_signed_in = false
		sign_in_failed.emit("GameCenter plugin not found")
		return

	var gc = Engine.get_singleton("GameCenter")
	gc.authenticate()
	if not gc.is_connected("authenticated", _on_gc_authenticated):
		gc.connect("authenticated", _on_gc_authenticated)
	if not gc.is_connected("authentication_failed", _on_gc_auth_failed):
		gc.connect("authentication_failed", _on_gc_auth_failed)


func _on_gc_authenticated() -> void:
	is_signed_in = true
	sign_in_succeeded.emit()
	_flush_pending_scores()


func _on_gc_auth_failed(error: String) -> void:
	is_signed_in = false
	sign_in_failed.emit(error)


## 수동 로그인 요청 (설정 화면 등에서 호출)
func request_sign_in() -> void:
	match current_platform:
		Platform.ANDROID:
			_sign_in_gpgs()
		Platform.IOS:
			_sign_in_gamecenter()
		Platform.OTHER:
			sign_in_failed.emit("Unsupported platform")


## ========== 점수 제출 ==========

## 점수 제출 (외부에서 호출하는 메인 인터페이스)
## leaderboard_key: LEADERBOARD_IDS의 키 (예: "daily_challenge_score")
## score: 제출할 점수 (정수)
func submit_score(leaderboard_key: String, score: int) -> void:
	if leaderboard_key not in LEADERBOARD_IDS:
		push_error("LeaderboardManager: 알 수 없는 리더보드 키 - %s" % leaderboard_key)
		return

	# 로컬 캐시에 항상 기록 (세이브 데이터와 동기화)
	_cache_score_locally(leaderboard_key, score)

	if is_signed_in:
		_submit_to_platform(leaderboard_key, score)
	else:
		# 오프라인이면 대기열에 추가
		_pending_scores.append({
			"leaderboard_key": leaderboard_key,
			"score": score
		})
		_save_pending_scores()


## 플랫폼에 실제로 점수 제출
func _submit_to_platform(leaderboard_key: String, score: int) -> void:
	var platform_id := _get_platform_leaderboard_id(leaderboard_key)
	if platform_id.is_empty():
		return

	match current_platform:
		Platform.ANDROID:
			var gpgs = Engine.get_singleton("GodotPlayGameServices")
			gpgs.submitScore(platform_id, score)
			score_submitted.emit(leaderboard_key, score)
		Platform.IOS:
			var gc = Engine.get_singleton("GameCenter")
			gc.report_score(platform_id, score)
			score_submitted.emit(leaderboard_key, score)
		Platform.OTHER:
			score_submit_failed.emit(leaderboard_key, "Unsupported platform")


## 현재 플랫폼에 맞는 리더보드 ID 반환
func _get_platform_leaderboard_id(leaderboard_key: String) -> String:
	var ids: Dictionary = LEADERBOARD_IDS.get(leaderboard_key, {})
	match current_platform:
		Platform.ANDROID:
			return ids.get("gpgs", "")
		Platform.IOS:
			return ids.get("gamecenter", "")
		_:
			return ""

## ========== 오프라인 처리 ==========

## 로컬에 점수 캐싱
func _cache_score_locally(leaderboard_key: String, score: int) -> void:
	# SaveManager를 통해 로컬 세이브 데이터에 기록
	# leaderboard_cache Dictionary에 최고 점수 유지
	var save_data: Dictionary = SaveManager.get_data()
	var cache: Dictionary = save_data.get("leaderboard_cache", {})

	var existing: Dictionary = cache.get(leaderboard_key, {"score": 0, "submitted": false})

	# 내림차순 리더보드: 더 높은 점수만 갱신
	# 오름차순 리더보드 (시간): 더 낮은 점수만 갱신
	var is_ascending := (leaderboard_key == "daily_challenge_time")

	if is_ascending:
		if existing["score"] == 0 or score < existing["score"]:
			cache[leaderboard_key] = {"score": score, "submitted": is_signed_in}
	else:
		if score > existing["score"]:
			cache[leaderboard_key] = {"score": score, "submitted": is_signed_in}

	save_data["leaderboard_cache"] = cache
	SaveManager.save_data(save_data)


## 대기 중인 점수 일괄 제출 (온라인 복귀 시)
func _flush_pending_scores() -> void:
	if _pending_scores.is_empty():
		# 세이브에서 미제출 점수도 확인
		_load_pending_scores()

	for entry in _pending_scores:
		_submit_to_platform(entry["leaderboard_key"], entry["score"])

	_pending_scores.clear()
	_save_pending_scores()

	# 로컬 캐시의 submitted 상태도 갱신
	var save_data: Dictionary = SaveManager.get_data()
	var cache: Dictionary = save_data.get("leaderboard_cache", {})
	for key in cache:
		cache[key]["submitted"] = true
	save_data["leaderboard_cache"] = cache
	SaveManager.save_data(save_data)

## 4. 업적(Achievement) 연동

### 4.1 설계 원칙

게임 내 업적 시스템(P07_01에서 정의)과 플랫폼 업적(GPGS Achievement / Game Center Achievement)을 **동기화**한다. 게임 내 업적이 해금되면 해당 플랫폼 업적도 함께 해금된다.

```
[게임 내 업적 해금]
       |
       v
[AchievementSync.sync_achievement(id)]
       |
       +-- [로그인 됨] --> [플랫폼에 보고]
       |
       +-- [로그인 안 됨] --> [대기열 저장, 나중에 동기화]
```

### 4.2 플랫폼 동기화 대상 업적

다음 업적은 플랫폼 콘솔(Google Play Console / App Store Connect)에 사전 등록하여 플랫폼 업적으로도 표시한다.

| 업적 ID | 이름 | 조건 | 유형 | 포인트(GPGS) |
|---------|------|------|------|-------------|
| `first_clear` | 첫 걸음 | 첫 스테이지 클리어 | 일반 | 10 |
| `stage_100` | 백전백승 | 100 스테이지 클리어 (누적) | 점진적 (100단계) | 50 |
| `streak_7` | 일주일의 기적 | 7일 연속 플레이 | 일반 | 25 |
| `streak_30` | 한 달의 습관 | 30일 연속 플레이 | 일반 | 50 |
| `all_themes` | 테마 마스터 | 모든 테마 해금 | 일반 | 30 |
| `grade_s_50` | 완벽주의자 | S등급 50회 달성 | 점진적 (50단계) | 50 |
| `bonus_words_100` | 탐험가 | 보너스 워드 100개 발견 | 점진적 (100단계) | 40 |
| `marathon_30` | 마라톤 러너 | 마라톤 30 스테이지 달성 | 일반 | 30 |

**업적 유형 설명:**

- **일반(Standard)**: 조건 충족 시 즉시 해금 (0% -> 100%)
- **점진적(Incremental)**: 진행도가 누적되어 목표 도달 시 해금 (예: 50/100 스테이지)

## 일반(Standard) 업적 해금 동기화
## 게임 내 업적 시스템에서 업적 해금 시 호출한다.
func sync_achievement(achievement_id: String) -> void:
	if achievement_id not in ACHIEVEMENT_IDS:
		push_warning("AchievementSync: 알 수 없는 업적 ID - %s" % achievement_id)
		return

	# 이미 동기화 완료된 업적은 건너뛰기
	if achievement_id in _synced_achievements:
		return

	var info: Dictionary = ACHIEVEMENT_IDS[achievement_id]

	if LeaderboardManager.is_signed_in:
		_unlock_on_platform(achievement_id, info)
	else:
		_pending_syncs.append({
			"achievement_id": achievement_id,
			"action": "unlock"
		})
		_save_sync_state()


## 점진적(Incremental) 업적 진행도 동기화
## current_steps: 현재 달성한 단계 수
func sync_incremental_achievement(achievement_id: String, current_steps: int) -> void:
	if achievement_id not in ACHIEVEMENT_IDS:
		push_warning("AchievementSync: 알 수 없는 업적 ID - %s" % achievement_id)
		return

	if achievement_id in _synced_achievements:
		return  # 이미 완전히 해금됨

	var info: Dictionary = ACHIEVEMENT_IDS[achievement_id]
	if info["type"] != "incremental":
		push_warning("AchievementSync: %s 는 incremental 타입이 아님" % achievement_id)
		return

	if LeaderboardManager.is_signed_in:
		_set_steps_on_platform(achievement_id, info, current_steps)
	else:
		_pending_syncs.append({
			"achievement_id": achievement_id,
			"action": "set_steps",
			"steps": current_steps
		})
		_save_sync_state()


## ========== 플랫폼 제출 ==========

## 일반 업적 해금
func _unlock_on_platform(achievement_id: String, info: Dictionary) -> void:
	match LeaderboardManager.current_platform:
		LeaderboardManager.Platform.ANDROID:
			var gpgs = Engine.get_singleton("GodotPlayGameServices")
			gpgs.unlockAchievement(info["gpgs"])
		LeaderboardManager.Platform.IOS:
			var gc = Engine.get_singleton("GameCenter")
			gc.report_achievement(info["gamecenter"], 100.0)  # 100% = 해금

	_synced_achievements.append(achievement_id)
	_save_sync_state()
	achievement_synced.emit(achievement_id)


## 점진적 업적 진행도 설정
func _set_steps_on_platform(achievement_id: String, info: Dictionary, steps: int) -> void:
	var max_steps: int = info.get("max_steps", 100)

	match LeaderboardManager.current_platform:
		LeaderboardManager.Platform.ANDROID:
			var gpgs = Engine.get_singleton("GodotPlayGameServices")
			gpgs.setStepsAtLeast(info["gpgs"], steps)
		LeaderboardManager.Platform.IOS:
			var gc = Engine.get_singleton("GameCenter")
			var percent := (float(steps) / float(max_steps)) * 100.0
			gc.report_achievement(info["gamecenter"], percent)

	# 목표 달성 시 동기화 완료 처리
	if steps >= max_steps:
		_synced_achievements.append(achievement_id)
		achievement_synced.emit(achievement_id)

	_save_sync_state()


## ========== 오프라인 복구 ==========

## 로그인 성공 시 대기 중인 업적 일괄 동기화
func _on_sign_in_succeeded() -> void:
	for entry in _pending_syncs:
		var achievement_id: String = entry["achievement_id"]
		if achievement_id not in ACHIEVEMENT_IDS:
			continue

		var info: Dictionary = ACHIEVEMENT_IDS[achievement_id]

		match entry["action"]:
			"unlock":
				_unlock_on_platform(achievement_id, info)
			"set_steps":
				_set_steps_on_platform(achievement_id, info, entry["steps"])

	_pending_syncs.clear()
	_save_sync_state()


## 플랫폼 업적 UI 표시
func show_achievements() -> void:
	if not LeaderboardManager.is_signed_in:
		# 로그인 유도
		return

	match LeaderboardManager.current_platform:
		LeaderboardManager.Platform.ANDROID:
			var gpgs = Engine.get_singleton("GodotPlayGameServices")
			gpgs.showAchievements()
		LeaderboardManager.Platform.IOS:
			var gc = Engine.get_singleton("GameCenter")
			gc.show_achievements()
```

		_:
			text += STORE_LINK_ANDROID  # 기본값

	return text


## 숫자 포맷팅 (1000 -> "1,000")
func _format_number(value: int) -> String:
	var s := str(value)
	var result := ""
	var count := 0
	for i in range(s.length() - 1, -1, -1):
		if count > 0 and count % 3 == 0:
			result = "," + result
		result = s[i] + result
		count += 1
	return result

## 6. 푸시 알림 (로컬 알림)

### 6.1 설계 원칙

서버 없이 **로컬 알림(Local Notification)** 을 우선으로 사용한다. 로컬 알림은 기기 내에서 예약되어 서버 통신 없이 동작한다. FCM/APNs 원격 푸시는 향후 이벤트 알림 등 서버 측 트리거가 필요한 경우에만 도입한다.

```
[로컬 알림 아키텍처]

  [게임 실행] --> [알림 스케줄 등록/갱신]
       |
       v
  [OS 알림 스케줄러]
       |
       v
  [지정 시간에 알림 표시] --> [사용자 탭] --> [앱 실행 / Deep Link]
```

### 6.2 알림 유형

| ID | 유형 | 시간 | 제목 | 내용 | 반복 |
|----|------|------|------|------|------|
| `daily_reminder` | Daily Challenge 리마인더 | 매일 09:00 | "오늘의 도전!" | "오늘의 Daily Challenge가 준비되었어요! 도전해보세요." | 매일 |
| `streak_warning` | 스트릭 위험 | 22:00 | "스트릭을 유지하세요!" | "오늘 아직 플레이하지 않았어요. 스트릭이 끊기기 전에 도전하세요!" | 매일 (조건부) |
| `event_start` | 이벤트 알림 | 이벤트 시작 시 | "특별 이벤트 시작!" | "{이벤트명}이(가) 시작되었어요! 한정 보상을 받아보세요." | 일회성 |
| `welcome_back` | Welcome-Back | 3일 미접속 후 | "다시 만나요!" | "돌아와주세요! 특별 보상이 기다리고 있어요." | 일회성 |

### 6.3 알림 조건부 로직

```
[매일 게임 실행 시]
    |
    v
[오늘 Daily Challenge 완료했는가?]
    |
    +-- YES --> [streak_warning 알림 취소]
    |
    +-- NO  --> [streak_warning 22:00 예약]

[daily_reminder]
    --> 항상 다음 날 09:00 예약

[welcome_back]
    --> 항상 현재 시각 + 72시간 예약
    --> 게임 실행 시 기존 예약 취소 후 재예약
```

```gdscript
## notification_manager.gd
## 로컬 알림 관리 매니저 - 알림 스케줄링, 취소, 설정 관리를 담당한다.
## Autoload 싱글톤으로 등록한다.
class_name NotificationManager
extends Node

## 알림 ID (플랫폼 알림 식별용 정수)
const NOTIF_ID_DAILY_REMINDER := 1001
const NOTIF_ID_STREAK_WARNING := 1002
const NOTIF_ID_EVENT_START := 1003
const NOTIF_ID_WELCOME_BACK := 1004

## 알림 채널 (Android Oreo 이상 필요)
const CHANNEL_ID := "word_search_notifications"
const CHANNEL_NAME := "Word Search Puzzle"

## 알림 플러그인 참조
var _notification_plugin = null

## 알림 설정 (사용자 제어)
var notification_settings: Dictionary = {
	"daily_reminder": true,
	"streak_warning": true,
	"event_start": true,
	"welcome_back": true

## 시그널
signal notification_scheduled(notif_type: String)
signal notification_cancelled(notif_type: String)
signal settings_changed(notif_type: String, enabled: bool)

	_load_settings()


## 알림 플러그인 초기화
func _init_plugin() -> void:
	# Godot Local Notification 플러그인 확인
	if Engine.has_singleton("LocalNotification"):
		_notification_plugin = Engine.get_singleton("LocalNotification")
		_create_notification_channel()
	else:
		push_warning("NotificationManager: LocalNotification 플러그인을 찾을 수 없음")


## Android 알림 채널 생성 (Oreo+)
func _create_notification_channel() -> void:
	if _notification_plugin == null:
		return
	if OS.get_name() == "Android":
		_notification_plugin.createNotificationChannel(
			CHANNEL_ID,
			CHANNEL_NAME,
			"게임 알림 및 리마인더"
		)


## 설정 로드
func _load_settings() -> void:
	var save_data: Dictionary = SaveManager.get_data()
	var saved_settings: Dictionary = save_data.get("notification_settings", {})
	# 저장된 설정이 있으면 덮어쓰기
	for key in saved_settings:
		if key in notification_settings:
			notification_settings[key] = saved_settings[key]


## 설정 저장
func _save_settings() -> void:
	var save_data: Dictionary = SaveManager.get_data()
	save_data["notification_settings"] = notification_settings
	SaveManager.save_data(save_data)


## ========== 알림 스케줄링 ==========

## 앱 시작 시 또는 게임 플레이 후 알림 스케줄 갱신
## 게임 실행 시마다 호출하여 알림을 최신 상태로 유지한다.
func refresh_all_notifications() -> void:
	_schedule_daily_reminder()
	_schedule_streak_warning()
	_schedule_welcome_back()


## Daily Challenge 리마인더 예약 (매일 09:00)
func _schedule_daily_reminder() -> void:
	if _notification_plugin == null:
		return
	if not notification_settings.get("daily_reminder", true):
		_cancel_notification(NOTIF_ID_DAILY_REMINDER)
		return

	# 기존 알림 취소 후 재예약
	_cancel_notification(NOTIF_ID_DAILY_REMINDER)

	# 다음 날 09:00 계산
	var now := Time.get_datetime_dict_from_system()
	var target_hour := 9
	var delay_seconds := _calculate_delay_to_hour(target_hour)

	# 이미 09:00 이후면 다음 날로
	if now["hour"] >= target_hour:
		delay_seconds += 86400  # 24시간

	_notification_plugin.scheduleLocalNotification(
		NOTIF_ID_DAILY_REMINDER,
		delay_seconds,
		"오늘의 도전!",
		"오늘의 Daily Challenge가 준비되었어요! 도전해보세요.",
		86400  # 반복 간격: 24시간 (매일)
	)

	notification_scheduled.emit("daily_reminder")


## 스트릭 위험 알림 예약 (오늘 22:00, 오늘 미플레이 시만)
func _schedule_streak_warning() -> void:
	if _notification_plugin == null:
		return
	if not notification_settings.get("streak_warning", true):
		_cancel_notification(NOTIF_ID_STREAK_WARNING)
		return

	# 오늘 이미 플레이했으면 알림 불필요
	if _has_played_today():
		_cancel_notification(NOTIF_ID_STREAK_WARNING)
		return

	# 기존 알림 취소 후 재예약
	_cancel_notification(NOTIF_ID_STREAK_WARNING)

	# 오늘 22:00까지 남은 시간 계산
	var now := Time.get_datetime_dict_from_system()
	var target_hour := 22

	if now["hour"] >= target_hour:
		# 이미 22:00 이후 - 스케줄하지 않음
		return

	var delay_seconds := _calculate_delay_to_hour(target_hour)

	_notification_plugin.scheduleLocalNotification(
		NOTIF_ID_STREAK_WARNING,
		delay_seconds,
		"스트릭을 유지하세요!",
		"오늘 아직 플레이하지 않았어요. 스트릭이 끊기기 전에 도전하세요!",
		0  # 반복 없음 (일회성)
	)

	notification_scheduled.emit("streak_warning")


## Welcome-Back 알림 예약 (현재 시각 + 72시간)
func _schedule_welcome_back() -> void:
	if _notification_plugin == null:
		return
	if not notification_settings.get("welcome_back", true):
		_cancel_notification(NOTIF_ID_WELCOME_BACK)
		return

	# 기존 알림 취소 후 재예약 (앱 실행 시마다 갱신)
	_cancel_notification(NOTIF_ID_WELCOME_BACK)

	var delay_seconds := 72 * 3600  # 3일 = 72시간

	_notification_plugin.scheduleLocalNotification(
		NOTIF_ID_WELCOME_BACK,
		delay_seconds,
		"다시 만나요!",
		"돌아와주세요! 특별 보상이 기다리고 있어요.",
		0  # 반복 없음
	)

	notification_scheduled.emit("welcome_back")


## 이벤트 알림 예약 (서버 없이 로컬 이벤트 시작 시각 기준)
## event_name: 이벤트 표시 이름
## delay_seconds: 현재부터 이벤트 시작까지 남은 초
func schedule_event_notification(event_name: String, delay_seconds: int) -> void:

		return
	if not notification_settings.get("event_start", true):
		return

	_notification_plugin.scheduleLocalNotification(
		NOTIF_ID_EVENT_START,
		delay_seconds,
		"특별 이벤트 시작!",
		"%s이(가) 시작되었어요! 한정 보상을 받아보세요." % event_name,
		0  # 반복 없음
	)

	notification_scheduled.emit("event_start")


## ========== 알림 취소 ==========

## 특정 알림 취소
func _cancel_notification(notif_id: int) -> void:

		return
	_notification_plugin.cancelLocalNotification(notif_id)


## 모든 알림 취소
func cancel_all_notifications() -> void:
	_cancel_notification(NOTIF_ID_DAILY_REMINDER)
	_cancel_notification(NOTIF_ID_STREAK_WARNING)
	_cancel_notification(NOTIF_ID_EVENT_START)
	_cancel_notification(NOTIF_ID_WELCOME_BACK)


## ========== 설정 제어 ==========

## 알림 유형별 ON/OFF 설정
func set_notification_enabled(notif_type: String, enabled: bool) -> void:
	if notif_type not in notification_settings:
		push_warning("NotificationManager: 알 수 없는 알림 유형 - %s" % notif_type)
		return

	notification_settings[notif_type] = enabled
	_save_settings()
	settings_changed.emit(notif_type, enabled)

	# 비활성화된 알림은 즉시 취소
	if not enabled:
		match notif_type:
			"daily_reminder":
				_cancel_notification(NOTIF_ID_DAILY_REMINDER)
			"streak_warning":
				_cancel_notification(NOTIF_ID_STREAK_WARNING)
			"event_start":
				_cancel_notification(NOTIF_ID_EVENT_START)
			"welcome_back":
				_cancel_notification(NOTIF_ID_WELCOME_BACK)
	else:
		# 활성화 시 다시 스케줄링
		refresh_all_notifications()


## 알림 설정 조회
func is_notification_enabled(notif_type: String) -> bool:
	return notification_settings.get(notif_type, true)


## ========== 유틸리티 ==========

## 오늘 플레이 여부 확인
func _has_played_today() -> bool:
	var save_data: Dictionary = SaveManager.get_data()
	var last_play_date: String = save_data.get("last_play_date", "")
	var today: String = Time.get_date_string_from_system()
	return last_play_date == today


## 특정 시각까지 남은 초 계산
func _calculate_delay_to_hour(target_hour: int) -> int:
	var now := Time.get_datetime_dict_from_system()
	var current_seconds := now["hour"] * 3600 + now["minute"] * 60 + now["second"]
	var target_seconds := target_hour * 3600
	var delay := target_seconds - current_seconds
	if delay < 0:
		delay += 86400  # 다음 날
	return delay

		var utils = Engine.get_singleton("GodotAndroidUtils")
		return utils.isTvDevice()

	# 대체 감지: 터치스크린 미지원 여부
	return not DisplayServer.is_touchscreen_available()


## 소셜 기능 가용성 확인
func is_social_available() -> bool:
	if is_tv_platform():
		return false
	if LeaderboardManager.current_platform == LeaderboardManager.Platform.OTHER:
		return false
	return true
```

### 9.2 데이터 마이그레이션

기존 세이브 파일에 소셜 필드가 없는 경우, 기본값으로 초기화한다.

```gdscript
## 세이브 데이터 마이그레이션 (소셜 필드 추가)
func _migrate_social_data(save_data: Dictionary) -> Dictionary:
	# 소셜 관련 필드가 없으면 기본값 추가
	if "leaderboard_cache" not in save_data:
		save_data["leaderboard_cache"] = {}

	if "pending_leaderboard_scores" not in save_data:
		save_data["pending_leaderboard_scores"] = []

	if "platform_achievements_synced" not in save_data:
		save_data["platform_achievements_synced"] = []

	if "pending_achievement_syncs" not in save_data:
		save_data["pending_achievement_syncs"] = []

	if "notification_settings" not in save_data:
		save_data["notification_settings"] = {
			"daily_reminder": true,
			"streak_warning": true,
			"event_start": true,
			"welcome_back": true
		}

	if "social_login_rewarded" not in save_data:
		save_data["social_login_rewarded"] = false

	if "share_count" not in save_data:
		save_data["share_count"] = 0

	if "last_play_date" not in save_data:
		save_data["last_play_date"] = ""

	return save_data
```

## 10. 프라이버시 & 동의

### 10.1 개인정보 수집 원칙

본 게임은 **최소한의 개인정보만을 사용**하며, 자체적으로 개인정보를 수집하지 않는다.

| 항목 | 수집 여부 | 관리 주체 | 설명 |
|------|----------|-----------|------|
| 닉네임 | 수집하지 않음 | Google/Apple | 플랫폼 계정 이름을 리더보드에 표시 |
| 이메일 | 수집하지 않음 | Google/Apple | 게임에서 접근하지 않음 |
| 게임 점수 | 플랫폼에 전송 | Google/Apple | 리더보드 목적으로만 사용 |
| 업적 진행도 | 플랫폼에 전송 | Google/Apple | 업적 시스템 목적으로만 사용 |
| 기기 식별자 | 수집하지 않음 | - | 광고 SDK가 별도 처리 (P06_01 참조) |

#### 10.5.2 앱 스토어 개인정보처리방침 요약

개인정보처리방침 웹 페이지에 다음 내용을 포함해야 한다:

1. **수집하는 정보**: 게임 점수, 업적 진행도 (플랫폼 API를 통해 전송)
2. **수집하지 않는 정보**: 이름, 이메일, 기기 식별자 등 개인 식별 정보
3. **정보 사용 목적**: 리더보드 순위 표시 및 업적 시스템 운영
4. **정보 공유**: Google/Apple 플랫폼에만 전송, 제3자에게 제공하지 않음
5. **데이터 보관**: 플랫폼 정책에 따름 (게임 자체 서버 없음)
6. **사용자 권리**: 플랫폼 계정 설정을 통해 데이터 삭제 가능
7. **아동 보호**: 13세 미만 아동의 경우 보호자 동의 필요 (COPPA)
8. **광고**: 광고 SDK의 개인정보 처리는 P06_01 참조
