### 2.4 Keystore 관리

```
# Keystore 생성 (최초 1회)
keytool -genkeypair \
  -v \
  -keystore wordsearch-release.keystore \
  -alias wordsearch \
  -keyalg RSA \
  -keysize 2048 \
  -validity 10000 \
  -storepass <STORE_PASSWORD> \
  -keypass <KEY_PASSWORD>
```

| 항목 | 값 |
|------|-----|
| **Keystore 파일명** | `wordsearch-release.keystore` |
| **Alias** | `wordsearch` |
| **Algorithm** | RSA 2048 |
| **유효기간** | 10000일 (약 27년) |

> **중요**: Keystore 파일과 비밀번호는 반드시 안전한 곳에 백업한다. Keystore를 분실하면 동일 패키지명으로 앱 업데이트가 불가능하다. Git에 커밋하지 않는다 (`.gitignore`에 `*.keystore` 추가).

## 3. Android 모바일 빌드

### 3.1 Godot Editor 사전 설정

Godot Editor 실행 후 `Editor > Editor Settings > Export > Android`에서 다음을 설정한다.

| 설정 항목 | 값 |
|-----------|-----|
| **Android SDK Path** | Android SDK 설치 경로 (예: `C:/Users/<user>/AppData/Local/Android/Sdk`) |
| **Java SDK Path** | JDK 17 설치 경로 |
| **Debug Keystore** | 기본 debug keystore 경로 (자동 생성됨) |

### 3.2 Export Preset 설정

`Project > Export`에서 **Android** Preset을 추가하고 다음과 같이 설정한다.

#### 3.2.1 기본 정보

| 항목 | 값 | 비고 |
|------|----|------|
| **Preset Name** | `Android Mobile` | 구분용 이름 |
| **Package Unique Name** | `com.yourcompany.wordsearch` | 실제 회사/앱명으로 변경 |
| **Version Code** | `1` | 업데이트마다 증가 (정수) |
| **Version Name** | `1.0.0` | 사용자에게 표시되는 버전 |
| **Min SDK** | `24` | Android 7.0 (Nougat) |
| **Target SDK** | `34` | Android 14 (Google Play 요구) |

#### 3.2.2 Keystore 설정

| 항목 | 값 |
|------|-----|
| **Release Keystore** | `wordsearch-release.keystore` 파일 경로 |
| **Release Keystore User** | `wordsearch` (alias) |
| **Release Keystore Password** | Keystore 생성 시 설정한 비밀번호 |

#### 3.2.3 Permissions (권한)

기본적으로 불필요한 권한은 제거한다. 광고 SDK 사용 시 필요한 권한만 포함한다.

| 권한 | 필요 여부 | 사유 |
|------|-----------|------|
| `INTERNET` | 필수 | 광고 로드, IAP |
| `ACCESS_NETWORK_STATE` | 필수 | 네트워크 상태 확인 |
| `AD_ID` | 조건부 | AdMob 사용 시 필수 |
| `VIBRATE` | 선택 | 햅틱 피드백 |

#### 3.2.4 Screen Orientation

| 항목 | 값 |
|------|-----|
| **Screen Orientation** | `Portrait` (폰 기본) |

> TV 빌드에서는 `Landscape`로 변경한다.

### 3.3 AAB 출력 (Google Play 필수)

Google Play는 2021년 8월부터 신규 앱에 대해 AAB(Android App Bundle) 형식을 필수로 요구한다.

**출력 절차:**

1. `Project > Export` 열기
2. `Android Mobile` Preset 선택
3. **Export Path**에 `.aab` 확장자로 저장 경로 지정
4. `Export Project` 클릭 (Debug 체크 해제)
5. 출력 파일: `wordsearch-release.aab`

```
출력 경로 예시:
builds/android/wordsearch-release.aab
```

### 3.4 APK 출력 (테스트 / Fire TV)

로컬 테스트 또는 Fire TV용으로 APK를 별도 출력한다.

**출력 절차:**

1. `Project > Export` 열기
2. `Android Mobile` Preset 선택
3. **Export Path**에 `.apk` 확장자로 저장 경로 지정
4. `Export Project` 클릭
5. 출력 파일: `wordsearch-release.apk`

```
출력 경로 예시:
builds/android/wordsearch-release.apk
```

### 3.5 Custom Build 활성화

AdMob, Google Play Billing Library 등 외부 Android 플러그인을 사용하는 경우 Custom Build를 활성화해야 한다.

**활성화 절차:**

1. Export Preset에서 `Use Custom Build` 체크
2. Godot가 `res://android/build/` 디렉토리에 Gradle 프로젝트를 생성한다
3. 필요한 플러그인의 `.aar` 파일 또는 Gradle 의존성을 추가한다
4. `AndroidManifest.xml` 수정이 필요한 경우 이 디렉토리에서 직접 편집한다

**Custom Build 디렉토리 구조:**

```
res://android/build/
  +-- build.gradle
  +-- gradle.properties
  +-- AndroidManifest.xml
  +-- res/
  |     +-- drawable/
  |           +-- banner.png          (TV 배너)
  +-- assets/
  +-- libs/
        +-- (외부 .aar 플러그인)
```

> **주의**: Custom Build를 활성화하면 `res://android/build/` 폴더가 프로젝트에 생성된다. 이 폴더는 Git에 포함시켜야 한다 (단, `res://android/build/.gradle/`, `res://android/build/build/` 등 빌드 캐시는 `.gitignore`에 추가).

### 3.6 Gradle 의존성 추가 (광고/IAP)

Custom Build 활성화 후 `res://android/build/build.gradle`에서 의존성을 추가한다.

```gradle
// res://android/build/build.gradle (dependencies 블록 예시)
dependencies {
    // AdMob
    implementation 'com.google.android.gms:play-services-ads:23.x.x'
    // Google Play Billing
    implementation 'com.android.billingclient:billing:7.x.x'
    // Firebase Crashlytics (선택)
    implementation 'com.google.firebase:firebase-crashlytics:19.x.x'
}
```

> 실제 버전 번호는 빌드 시점의 최신 안정 버전을 사용한다.

## 4. Android TV 빌드

### 4.1 개요

Android TV 빌드는 Android 모바일 빌드를 기반으로 하되, 다음 추가 설정이 필요하다:

- Custom Build 활성화 (AndroidManifest.xml 수동 편집 필요)
- Leanback Launcher 지원 선언
- 터치스크린 비필수 선언
- TV 배너 이미지 추가
- 화면 방향을 Landscape로 설정

### 4.2 Export Preset 설정 (TV 전용)

기존 Android Mobile Preset을 복제하여 TV 전용 Preset을 생성한다.

| 항목 | 값 | 비고 |
|------|----|------|
| **Preset Name** | `Android TV` | 구분용 |
| **Package Unique Name** | `com.yourcompany.wordsearch` | 모바일과 동일 |
| **Min SDK** | `24` | Android TV 호환 |
| **Target SDK** | `34` | |
| **Screen Orientation** | `Landscape` | TV는 가로 고정 |
| **Use Custom Build** | `true` | 필수 |

### 4.3 AndroidManifest.xml 수동 편집

Custom Build 활성화 후 생성되는 `res://android/build/AndroidManifest.xml`을 편집한다.

#### 4.3.1 Leanback 지원 선언

`<manifest>` 태그 내 `<uses-feature>` 섹션에 다음을 추가한다:

```xml
<!-- TV(Leanback) 지원: required="false"로 설정하여 모바일에서도 설치 가능 -->
<uses-feature
    android:name="android.software.leanback"
    android:required="false" />
```

#### 4.3.2 터치스크린 비필수 선언

TV에는 터치스크린이 없으므로 `required="false"`로 선언한다:

```xml
<!-- 터치스크린을 필수로 요구하지 않음 (TV 호환) -->
<uses-feature
    android:name="android.hardware.touchscreen"
    android:required="false" />
```

#### 4.3.3 Leanback Launcher Intent Filter

`<application>` 내 메인 `<activity>` 에 Leanback Launcher Intent Filter를 추가한다:

```xml
<activity android:name="com.godot.game.GodotApp"
          android:label="@string/godot_project_name_string"
          android:banner="@drawable/banner"
          ...>
    <!-- 기존 MAIN / LAUNCHER Intent Filter 유지 -->
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LAUNCHER" />
    </intent-filter>

    <!-- TV Leanback Launcher Intent Filter 추가 -->
    <intent-filter>
        <action android:name="android.intent.action.MAIN" />
        <category android:name="android.intent.category.LEANBACK_LAUNCHER" />
    </intent-filter>
</activity>
```

> `android:banner="@drawable/banner"` 속성이 `<activity>` 또는 `<application>` 태그에 포함되어야 TV 홈 화면에 배너가 표시된다.

#### 4.3.4 전체 Manifest 변경 요약

```xml
<?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android"
    package="com.yourcompany.wordsearch">

    <!-- 권한 -->
    <uses-permission android:name="android.permission.INTERNET" />
    <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

    <!-- TV 호환 Feature 선언 -->
    <uses-feature
        android:name="android.software.leanback"
        android:required="false" />
    <uses-feature
        android:name="android.hardware.touchscreen"
        android:required="false" />

    <application
        android:banner="@drawable/banner"
        ... >

        <activity
            android:name="com.godot.game.GodotApp"
            android:banner="@drawable/banner"
            ... >

            <!-- 모바일 Launcher -->
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LAUNCHER" />
            </intent-filter>

            <!-- TV Leanback Launcher -->
            <intent-filter>
                <action android:name="android.intent.action.MAIN" />
                <category android:name="android.intent.category.LEANBACK_LAUNCHER" />
            </intent-filter>
        </activity>

    </application>
</manifest>
```

### 4.4 TV 배너 이미지

| 항목 | 사양 |
|------|------|
| **크기** | 320 x 180 px |
| **포맷** | PNG |
| **배치 경로** | `res://android/build/res/drawable/banner.png` |
| **용도** | Android TV 홈 화면 앱 아이콘 대체 |

> TV 홈 화면에서는 일반 앱 아이콘 대신 이 배너 이미지가 표시된다. 게임 로고와 제목이 명확히 보이도록 디자인한다.

### 4.5 TV 빌드 출력

TV 전용 Preset으로 AAB를 출력한다.

```
출력 경로 예시:
builds/android-tv/wordsearch-tv-release.aab
```

> Google Play에 업로드할 때, 모바일과 TV를 동일 앱(동일 패키지명)으로 등록하거나 별도 앱으로 등록할 수 있다. 동일 앱으로 등록하는 경우 Manifest 설정만으로 모바일/TV 모두 지원 가능하다.

## 5. Fire TV 빌드

### 5.1 개요

Fire TV는 Amazon의 TV 플랫폼으로, Android 기반이지만 Google Play Services가 없다. Amazon Appstore를 통해 배포하며, 광고/IAP는 Amazon 전용 SDK를 사용한다.

### 5.2 Android TV와의 차이점

| 항목 | Android TV | Fire TV |
|------|-----------|---------|
| **스토어** | Google Play | Amazon Appstore |
| **출력 포맷** | AAB (필수) | APK (권장) 또는 AAB |
| **Google Play Services** | 사용 가능 | 사용 불가 |
| **광고 SDK** | AdMob (Google) | Amazon Mobile Ads / IMA SDK |
| **IAP** | Google Play Billing | Amazon IAP SDK |
| **기반 OS** | Android (AOSP) | Fire OS 7+ (Android 9 기반) |
| **Leanback** | 지원 | 지원 (호환) |

### 5.3 Fire TV Export Preset

Android TV Preset을 복제하여 Fire TV 전용 Preset을 생성한다.

| 항목 | 값 | 비고 |
|------|----|------|
| **Preset Name** | `Fire TV` | 구분용 |
| **Package Unique Name** | `com.yourcompany.wordsearch` | 동일하게 유지 |
| **Min SDK** | `28` | Fire OS 7 = Android 9 (API 28) |
| **Target SDK** | `34` | |
| **Screen Orientation** | `Landscape` | |
| **Use Custom Build** | `true` | |

### 5.4 Google Play Services 대체

Fire TV에서는 Google Play Services가 없으므로 다음을 대체한다:

| 기능 | Google (Android TV) | Amazon (Fire TV) |
|------|-------------------|-----------------|
| **광고** | AdMob | Amazon Mobile Ads SDK |
| **IAP** | Google Play Billing Library | Amazon In-App Purchasing (IAP) SDK |
| **분석** | Firebase Analytics | Amazon Mobile Analytics 또는 자체 구현 |
| **Crash Reporting** | Firebase Crashlytics | Amazon Device Messaging 또는 자체 구현 |

### 5.5 Amazon IAP SDK 통합

1. Amazon Developer Console에서 IAP SDK 다운로드
2. `amazon-iap-v3.x.x.jar` 또는 `.aar`를 `res://android/build/libs/`에 배치
3. `build.gradle`에 의존성 추가
4. GDScript에서 Amazon IAP 인터페이스 호출 (GDExtension 또는 Android Plugin 방식)

```gradle
// Fire TV 전용 build.gradle 의존성
dependencies {
    implementation files('libs/amazon-iap-v3.x.x.jar')
}
```

### 5.6 Fire TV 빌드 출력

```
출력 경로 예시:
builds/fire-tv/wordsearch-firetv-release.apk
```

> Amazon Appstore는 APK를 권장하지만 AAB도 지원한다. 호환성을 위해 APK를 우선 사용한다.

## 6. iOS 빌드

### 6.1 필수 환경

| 항목 | 요구사항 |
|------|----------|
| **하드웨어** | Mac (macOS) |
| **Xcode** | 최신 안정 버전 |
| **Apple Developer 계정** | 필수 ($99/년) |
| **Godot** | macOS용 Godot 4.6 Stable |

> **중요**: iOS 빌드는 Windows에서 불가능하다. 반드시 Mac 환경이 필요하다. Mac이 없는 경우 macOS 클라우드 빌드 서비스(MacStadium, GitHub Actions macOS Runner 등)를 고려한다.

### 6.2 Export Preset 설정 (iOS)

`Project > Export`에서 **iOS** Preset을 추가한다.

| 항목 | 값 | 비고 |
|------|----|------|
| **Preset Name** | `iOS` | |
| **Bundle Identifier** | `com.yourcompany.wordsearch` | Android와 동일 권장 |
| **Version** | `1.0.0` | CFBundleShortVersionString |
| **Build** | `1` | CFBundleVersion (업데이트마다 증가) |
| **Team ID** | Apple Developer 계정의 Team ID | |
| **Signing** | `Automatic` 또는 수동 선택 | |

### 6.3 빌드 및 제출 절차

iOS 빌드는 다음 단계로 진행된다:

```
[Godot Export]  -->  [Xcode Project]  -->  [Signing]  -->  [Archive]  -->  [App Store Connect]
```

**상세 절차:**

1. **Godot에서 Export**
   - `Project > Export > iOS` 선택
   - `Export Project` 클릭하여 Xcode 프로젝트 생성
   - 출력: `.xcodeproj` 파일이 포함된 디렉토리

2. **Xcode에서 열기**
   - 생성된 `.xcodeproj` 파일을 Xcode에서 열기
   - Bundle Identifier, Version, Build 확인

3. **Signing 설정**
   - `Signing & Capabilities` 탭에서 Team 선택
   - Automatic Signing 사용 시 Xcode가 자동으로 Certificate/Profile 관리
   - 수동 Signing 시 Apple Developer Portal에서 직접 생성

4. **Archive 생성**
   - `Product > Archive` 실행
   - Scheme을 `Any iOS Device`로 설정
   - Archive 완료 후 Organizer 창에서 확인

5. **App Store Connect 업로드**
   - Organizer에서 `Distribute App` 클릭
   - `App Store Connect` 선택
   - Upload 완료 후 App Store Connect 웹사이트에서 확인

### 6.4 ATT (App Tracking Transparency)

iOS 14.5 이상에서는 광고 추적을 위해 ATT 프롬프트가 필수이다.

| 항목 | 설명 |
|------|------|
| **대상** | 광고 SDK(AdMob, IMA 등)를 포함하는 모든 앱 |
| **시점** | 앱 최초 실행 시 또는 광고 표시 전 |
| **필수 Key** | `NSUserTrackingUsageDescription` (Info.plist) |
| **문구 예시** | "더 관련성 높은 광고를 제공하기 위해 추적 권한을 요청합니다." |

**Info.plist 추가 항목:**

```xml
<key>NSUserTrackingUsageDescription</key>
<string>이 앱은 관련성 높은 광고를 제공하기 위해 사용자의 활동 추적 권한을 요청합니다.</string>
```

**GDScript에서 ATT 요청:**

```gdscript
# iOS ATT 요청 (Godot iOS Plugin 또는 GDExtension 필요)
func request_att_permission() -> void:
```

### 6.6 SKAdNetwork 설정

광고 SDK를 포함하는 경우 SKAdNetwork ID를 Info.plist에 추가한다.

```xml
<key>SKAdNetworkItems</key>
<array>
    <!-- Google AdMob -->
    <dict>
        <key>SKAdNetworkIdentifier</key>
        <string>cstr6suwn9.skadnetwork</string>
    </dict>
    <!-- 기타 광고 네트워크 ID 추가 -->
    <!-- 각 광고 SDK 공식 문서에서 최신 목록 확인 -->
</array>
```

> SKAdNetwork ID 목록은 각 광고 SDK 제공사의 공식 문서에서 최신 버전을 확인하여 적용한다.

### 8.2 아이콘 제작 가이드

1. **마스터 아이콘 (1024 x 1024)** 을 먼저 제작한다
2. 투명 배경을 사용하지 않는다 (모든 스토어에서 거부됨)
3. 모서리 둥글림(rounded corner)을 적용하지 않는다 (각 OS/스토어가 자동으로 마스킹함)
4. 주요 시각 요소를 중앙 66% 영역에 배치한다 (Adaptive Icon 안전 영역)
5. 작은 크기(48 x 48)에서도 식별 가능한 디자인을 사용한다

## 9. CI/CD (향후 구현)

### 9.1 개요

출시 이후 업데이트 빈도가 높아지면 CI/CD 파이프라인을 구축하여 빌드 자동화를 도입한다.

### 9.2 GitHub Actions 기반 구성

```yaml
# .github/workflows/build.yml (개념 예시)
name: Build & Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build-android:
    runs-on: ubuntu-latest
    container:
      image: barichello/godot-ci:4.6-stable
    steps:
      - uses: actions/checkout@v4
      - name: Setup Android SDK
        run: |
          # Android SDK 설정
      - name: Build AAB
        run: |
          godot --headless --export-release "Android Mobile" \
            builds/android/wordsearch-release.aab
      - name: Build TV AAB
        run: |
          godot --headless --export-release "Android TV" \
            builds/android-tv/wordsearch-tv-release.aab
      - name: Upload Artifacts
        uses: actions/upload-artifact@v4
        with:
          name: android-builds
          path: builds/

  build-ios:
    runs-on: macos-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install Godot
        run: |
          # macOS에 Godot 설치
      - name: Export iOS
        run: |
          godot --headless --export-release "iOS" \
            builds/ios/wordsearch.xcodeproj
      - name: Build & Archive
        run: |
          xcodebuild -project builds/ios/wordsearch.xcodeproj \
            -scheme wordsearch \
            -archivePath builds/ios/wordsearch.xcarchive \
            archive
```

### 9.3 자동화 범위 (향후)

| 기능 | 설명 | 우선순위 |
|------|------|----------|
| **버전 자동 증가** | Git Tag 기반 Version Code/Name 자동 설정 | 높음 |
| **AAB 자동 생성** | Android 모바일 + TV AAB 자동 빌드 | 높음 |
| **APK 자동 생성** | Fire TV용 APK 자동 빌드 | 높음 |
| **IPA 자동 생성** | iOS Archive + Export 자동화 | 중간 |
| **Google Play 자동 업로드** | fastlane supply 활용 | 낮음 |
| **App Store 자동 업로드** | fastlane deliver 활용 | 낮음 |
| **Amazon 자동 업로드** | Amazon App Submission API | 낮음 |
| **슬랙 알림** | 빌드 성공/실패 알림 | 중간 |

### C. 버전 관리 규칙

| 항목 | 형식 | 예시 | 설명 |
|------|------|------|------|
| **Version Name** | `MAJOR.MINOR.PATCH` | `1.2.3` | Semantic Versioning |
| **Version Code (Android)** | 정수 | `10203` | `MAJOR*10000 + MINOR*100 + PATCH` |
| **CFBundleVersion (iOS)** | 정수 또는 문자열 | `1.2.3` | 빌드 번호 (업로드마다 증가) |
| **CFBundleShortVersionString** | `MAJOR.MINOR.PATCH` | `1.2.3` | 사용자에게 표시 |

**Version Code 계산 공식:**

```
Version Code = MAJOR * 10000 + MINOR * 100 + PATCH

예시:
  1.0.0 → 10000
  1.2.3 → 10203
  2.0.0 → 20000
```

### D. 문제 해결 (Troubleshooting)

| 증상 | 원인 | 해결 방법 |
|------|------|-----------|
| Export 실패: "No export template found" | Export Template 미설치 | `Editor > Manage Export Templates > Download` |
| AAB 빌드 실패: Gradle 오류 | JDK 버전 불일치 | JDK 17로 통일, Editor Settings에서 경로 확인 |
| TV에서 앱이 보이지 않음 | Leanback Launcher Intent Filter 누락 | AndroidManifest.xml에 `LEANBACK_LAUNCHER` 추가 |
| TV 배너가 표시되지 않음 | banner.png 경로 오류 | `res://android/build/res/drawable/banner.png` 확인 |
| iOS Signing 실패 | Certificate/Profile 불일치 | Xcode에서 Automatic Signing 활성화 |
| App Store 심사 거부: ATT 누락 | ATT 프롬프트 미구현 | `NSUserTrackingUsageDescription` 추가 + ATT 요청 코드 구현 |
| Fire TV에서 크래시 | Google Play Services 호출 | Google 의존성 제거, Amazon SDK로 대체 |
| 앱 크기 초과 | 리소스 미최적화 | 이미지 WebP 변환, 오디오 OGG 압축, 미사용 리소스 제거 |
