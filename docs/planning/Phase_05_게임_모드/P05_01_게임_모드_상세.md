## 1. Classic Mode

### 1.1 개요

Classic Mode는 **시간 제한 없이** 모든 단어를 찾는 기본 게임 모드이다. Home 화면에서 "Play" 버튼을 누르면 진입하는 기본 모드이며, 스테이지 기반 진행 시스템을 따른다.

### 1.2 규칙

| 항목 | 규칙 |
|------|------|
| **시간 제한** | 없음 |
| **승리 조건** | 모든 배치 단어를 찾으면 스테이지 클리어 |
| **패배 조건** | 없음 (포기 시 메뉴로 돌아감) |
| **스테이지 진행** | 클리어 시 다음 스테이지로 자동 진행 |
| **난이도** | 스테이지에 따라 격자 크기 및 단어 수 증가 (P01_01 참조) |
| **DDA** | 적용 (P03_01 참조) |
| **휴식 스테이지** | 10 스테이지마다 발동 (P03_01 참조) |
| **False Lead** | 스테이지 11부터 활성화 (P03_01 참조) |
| **힌트** | 모든 힌트 사용 가능 (TimerExtend 제외) |
| **점수** | 기본 점수 + 콤보 보너스 + 노힌트 보너스 (P01_01 참조) |
| **해금 조건** | 없음 (기본 해금) |

### 1.3 게임 플로우

```
[Home > Play] → [스테이지 N 시작]
    │
    ├── [격자 생성 + 단어 배치]
    │   ├── DDA offset 적용
    │   ├── 휴식 스테이지 판정
    │   └── False Lead 삽입 (stage >= 11)
    │
    ├── [플레이어 단어 탐색]
    │   ├── 드래그/탭으로 단어 선택
    │   ├── 힌트 사용 (선택)
    │   └── 셔플 사용 (선택)
    │
    ├── [모든 단어 발견?]
    │   ├── Yes → [결과 화면] → [다음 스테이지 N+1]
    │   └── No → 계속 탐색
    │
    └── [설정 > 포기] → [Home]
```

### 1.4 결과 화면 정보

| 표시 항목 | 설명 |
|-----------|------|
| 등급 (S/A/B/C) | 힌트 사용 횟수 + 클리어 시간 기반 |
| 점수 | 총 획득 점수 (콤보, 노힌트 보너스 포함) |
| 클리어 시간 | 스테이지 소요 시간 |
| 코인 보상 | 획득 코인 내역 |
| 최고 기록 여부 | 기존 기록 갱신 시 "New Record!" 표시 |
| Bonus Stage 여부 | 휴식 스테이지인 경우 "Bonus Stage!" 배너 |

## 2. Time Attack Mode

### 2.1 개요

Time Attack Mode는 **제한 시간 내에 모든 단어를 찾아야 하는** 긴장감 있는 모드이다. 단어를 찾을 때마다 시간이 추가되므로, 빠른 판단과 연속 발견이 중요하다.

### 2.2 규칙

| 항목 | 규칙 |
|------|------|
| **초기 시간** | 180초 (3분) |
| **단어 발견 시 추가** | +10초 |
| **승리 조건** | 모든 단어를 시간 내에 찾으면 클리어 |
| **패배 조건** | 시간 초과 (Game Over) |
| **스테이지 진행** | Classic과 동일한 스테이지 시스템 공유 (별도 진행 상태) |
| **DDA** | 적용 |
| **휴식 스테이지** | 적용 |
| **False Lead** | 적용 (스테이지 11부터) |
| **힌트** | **모든 힌트 사용 가능** (TimerExtend **포함**) |
| **점수** | 기본 점수 + 콤보 보너스 + **시간 보너스** |
| **해금 조건** | 없음 (기본 해금) |

### 2.3 타이머 시스템

```gdscript
## time_attack_timer.gd
## Time Attack 모드의 타이머를 관리한다.

const INITIAL_TIME: float = 180.0       ## 초기 시간 (초)
const TIME_PER_WORD: float = 10.0       ## 단어 발견 시 추가 시간 (초)
const WARNING_THRESHOLD: float = 30.0   ## 경고 표시 기준 (초)

var remaining_time: float = INITIAL_TIME
var is_running: bool = false

signal time_updated(remaining: float)
signal time_warning()     ## 남은 시간 <= 30초
signal time_expired()     ## 시간 초과


## 타이머를 시작한다.
func start() -> void:
	remaining_time = INITIAL_TIME
	is_running = true


## 매 프레임 호출하여 타이머를 갱신한다.
func tick(delta: float) -> void:
	if not is_running:
		return

	remaining_time -= delta
	time_updated.emit(remaining_time)

	if remaining_time <= WARNING_THRESHOLD and remaining_time + delta > WARNING_THRESHOLD:
		time_warning.emit()

	if remaining_time <= 0.0:
		remaining_time = 0.0
		is_running = false
		time_expired.emit()


## 단어 발견 시 시간을 추가한다.
func add_time(seconds: float) -> void:
	remaining_time += seconds
	time_updated.emit(remaining_time)


## TimerExtend 힌트에 의한 시간 추가.
func extend_time(seconds: float) -> void:
	add_time(seconds)
```

### 2.4 시간 보너스 점수

스테이지 클리어 시 남은 시간에 비례하는 보너스 점수를 부여한다.

```gdscript
## Time Attack 시간 보너스 계산
## 남은 시간(초) x 2 = 시간 보너스 점수
func calculate_time_bonus(remaining_seconds: float) -> int:
	return int(remaining_seconds * 2.0)
```

### 2.5 타이머 UI 표현

| 남은 시간 | 표시 색상 | 추가 효과 |
|-----------|-----------|-----------|
| > 30초 | 흰색 | 없음 |
| <= 30초 | 빨간색 | 깜빡임 애니메이션 |
| <= 10초 | 빨간색 + 크기 확대 | 깜빡임 + 틱틱 효과음 |

### 2.6 접근 방법

Home 화면에서 "Play" 버튼의 하위 메뉴 또는 토글을 통해 접근한다.

```
[Home]
  └── [Play 버튼]
       ├── Classic Mode (기본 선택)
       └── Time Attack Mode (서브 메뉴 또는 토글)
```

## 3. Daily Challenge

### 3.1 개요

Daily Challenge는 **매일 동일한 퍼즐**을 모든 플레이어에게 제공하는 모드이다. SeededRandom(시드 기반 의사난수)을 사용하여 동일 날짜에는 동일한 격자, 단어 배치, 빈 칸 문자가 생성된다. 플레이어 간 공정한 비교가 가능하며, 연속 출석(Streak) 시스템과 연동된다.

### 3.2 해금 조건

| 항목 | 값 |
|------|----|
| **해금 스테이지** | Classic Mode 스테이지 24 클리어 |
| **해금 이유** | 충분한 게임 경험 후 Daily Challenge 접근 (격자 8x7, 단어 8개 수준) |

### 3.3 규칙

| 항목 | 규칙 |
|------|------|
| **시간 제한** | 없음 |
| **난이도** | 고정 (격자 8x8, 단어 8개) |
| **DDA** | **비적용** (모든 플레이어 동일 퍼즐 보장) |
| **False Lead** | 비적용 (시드 기반 퍼즐의 공정성 보장) |
| **힌트** | 사용 가능 (단, 힌트 사용 여부가 리더보드에 표시) |
| **재도전** | 불가 (1일 1회) |
| **보상** | 클리어 시 100 코인 고정 + Streak 보너스 |
| **Streak** | 연속 클리어 일수 추적. 7일 연속 시 추가 보상. |

### 3.4 SeededRandom (시드 기반 의사난수 생성기)

동일 시드를 입력하면 **모든 기기에서 동일한 시퀀스**를 보장하는 LCG(Linear Congruential Generator)를 사용한다.

**LCG 파라미터**: `a = 1664525`, `c = 1013904223` (Numerical Recipes 계열)

```gdscript
## seeded_random.gd
## Daily Challenge용 시드 기반 의사난수 생성기 (LCG).
## 동일 시드 입력 시 모든 기기에서 동일한 시퀀스를 보장한다.
## C# 원본: SeededRandom.cs
class_name SeededRandom
extends RefCounted


## LCG 파라미터 (Numerical Recipes)
const LCG_A: int = 1664525
const LCG_C: int = 1013904223

## 비트 마스크 (양수 범위 보장, 31비트)
const MASK: int = 0x7FFFFFFF

## 현재 시드 상태
var state: int


## 시드 값으로 초기화한다.
func _init(seed_value: int) -> void:
	state = seed_value


## 다음 의사난수 정수를 반환한다 (0 ~ 0x7FFFFFFF).
func next_int() -> int:
	state = (state * LCG_A + LCG_C) & MASK
	return state


## min_val (inclusive) ~ max_val (inclusive) 범위의 의사난수 정수를 반환한다.
func next_range(min_val: int, max_val: int) -> int:
	if min_val >= max_val:
		return min_val
	return min_val + (next_int() % (max_val - min_val + 1))


## 0.0 ~ 1.0 범위의 의사난수 실수를 반환한다.
func next_float() -> float:
	return float(next_int()) / float(MASK)


## 배열을 시드 기반으로 셔플한다 (Fisher-Yates 알고리즘).
func shuffle_array(arr: Array) -> void:
	for i in range(arr.size() - 1, 0, -1):
		var j := next_range(0, i)
		var temp = arr[i]
		arr[i] = arr[j]
		arr[j] = temp


## 날짜 기반 시드를 생성한다.
## 예: 2026-03-11 -> 20260311
static func get_daily_seed() -> int:
	var now := Time.get_datetime_dict_from_system()
	return now.year * 10000 + now.month * 100 + now.day


## 특정 날짜의 시드를 생성한다.
## Calendar UI에서 과거 날짜의 퍼즐을 재생할 때 사용한다.
static func get_seed_for_date(year: int, month: int, day: int) -> int:
	return year * 10000 + month * 100 + day
```

### 3.5 Daily Challenge 격자 생성

일반 GridGenerator를 사용하되, 모든 랜덤 요소를 `SeededRandom`으로 대체한다.

```gdscript
## daily_challenge_generator.gd
## Daily Challenge용 격자 생성기.
## 일반 GridGenerator와 동일한 알고리즘을 사용하되,
## 모든 랜덤 함수를 SeededRandom으로 대체한다.

class_name DailyChallengeGenerator
extends RefCounted

## Daily Challenge 고정 난이도
const DAILY_GRID_WIDTH: int = 8
const DAILY_GRID_HEIGHT: int = 8
const DAILY_WORD_COUNT: int = 8


## 오늘의 Daily Challenge 격자를 생성한다.
func generate_today() -> GridData:
	var seed := SeededRandom.get_daily_seed()
	return generate_for_seed(seed)


## 특정 시드로 Daily Challenge 격자를 생성한다.
func generate_for_seed(seed: int) -> GridData:
	var rng := SeededRandom.new(seed)

	# GridData 초기화
	var data := GridData.new()
	data.width = DAILY_GRID_WIDTH
	data.height = DAILY_GRID_HEIGHT
	data.language = _get_current_language()

	# 빈 격자 생성
	data.grid = _create_empty_grid(data.width, data.height)

	# 단어 선택 (시드 기반)
	var word_pack := _get_daily_word_pack(rng)
	var selected_words := _select_words_seeded(word_pack, DAILY_WORD_COUNT, rng)

	# 단어 배치 (시드 기반)
	for entry in selected_words:
		_try_place_word_seeded(data, entry, rng)

	# 빈 칸 채우기 (시드 기반)
	_fill_empty_cells_seeded(data, word_pack, rng)

	return data


## 시드 기반으로 단어를 배치한다.
## randi() 대신 rng.next_range()를 사용한다.
func _try_place_word_seeded(data: GridData, entry: WordEntry, rng: SeededRandom) -> bool:
	var directions := GridGenerator.DIRECTIONS.duplicate()

	for attempt in range(GridGenerator.MAX_PLACEMENT_ATTEMPTS):
		var dir_index := rng.next_range(0, directions.size() - 1)
		var dir: Vector2i = directions[dir_index]
		var start := Vector2i(
			rng.next_range(0, data.width - 1),
			rng.next_range(0, data.height - 1)
		)

		if GridGenerator._can_place(data.grid, entry.display, start, dir, data.width, data.height):
			GridGenerator._place_word(data, entry, start, dir)
			return true

	return false


## 시드 기반 빈 칸 채우기.
func _fill_empty_cells_seeded(data: GridData, word_pack: WordPack, rng: SeededRandom) -> void:
	# 풀 구성 (일반 로직과 동일)
	var fill_pool: Array[String] = _build_fill_pool(data, word_pack)

	# 빈 셀에 시드 기반 랜덤 문자 배치
	for y in range(data.height):
		for x in range(data.width):
			if data.grid[y][x] == "":
				var index := rng.next_range(0, fill_pool.size() - 1)
				data.grid[y][x] = fill_pool[index]


func _create_empty_grid(width: int, height: int) -> Array:
	var grid: Array = []
	for y in range(height):
		var row: Array = []
		for x in range(width):
			row.append("")
		grid.append(row)
	return grid


func _get_current_language() -> String:
	# SettingsManager에서 현재 언어를 가져온다
	return "en"  # 기본값


func _get_daily_word_pack(_rng: SeededRandom) -> WordPack:
	# 시드 기반으로 오늘의 WordPack을 선택한다
	return null  # 실제 구현에서는 WordPack 선택 로직


func _select_words_seeded(_word_pack: WordPack, _count: int, _rng: SeededRandom) -> Array:
	# 시드 기반으로 단어를 선택한다
	return []  # 실제 구현에서는 단어 선택 로직


func _build_fill_pool(_data: GridData, _word_pack: WordPack) -> Array[String]:
	return []  # 실제 구현에서는 빈도 기반 풀 구성
```

### 3.7 Streak 시스템

| 항목 | 규칙 |
|------|------|
| **Streak 증가** | Daily Challenge 클리어 시 +1 |
| **Streak 리셋** | 하루라도 미플레이 시 0으로 리셋 |
| **7일 보너스** | 연속 7일 클리어 시 300 코인 보상 |
| **30일 보너스** | 연속 30일 클리어 시 1000 코인 + 특별 테마 해금 |
| **Streak 표시** | Calendar UI에 연속 클리어 일수 표시 |

```gdscript
## daily_streak.gd
## Daily Challenge 연속 출석 추적.

var current_streak: int = 0
var best_streak: int = 0
var last_clear_date: String = ""  ## "YYYYMMDD" 형식


## Daily Challenge 클리어 시 호출한다.
func on_daily_cleared() -> Dictionary:
	var today := _get_today_string()
	var yesterday := _get_yesterday_string()
	var reward: Dictionary = {"coins": 100, "streak_bonus": 0}

	if last_clear_date == yesterday:
		# 연속 출석
		current_streak += 1
	elif last_clear_date == today:
		# 이미 오늘 클리어함 (중복 호출 방지)
		return reward
	else:
		# Streak 끊김, 1부터 재시작
		current_streak = 1

	last_clear_date = today
	best_streak = maxi(best_streak, current_streak)

	# Streak 보너스 확인
	if current_streak % 7 == 0:
		reward.streak_bonus = 300
	if current_streak == 30:
		reward.streak_bonus = 1000

	return reward


func _get_today_string() -> String:
	var now := Time.get_datetime_dict_from_system()
	return "%04d%02d%02d" % [now.year, now.month, now.day]


func _get_yesterday_string() -> String:
	var unix := Time.get_unix_time_from_system() - 86400
	var yesterday := Time.get_datetime_dict_from_unix_time(int(unix))
	return "%04d%02d%02d" % [yesterday.year, yesterday.month, yesterday.day]
```

## 4. Bonus Words

### 4.1 개요

Bonus Words는 **퍼즐에 포함되지 않은 유효 단어**를 플레이어가 우연히 또는 의도적으로 찾았을 때 보너스 보상을 지급하는 시스템이다. 격자에 배치된 문자들이 우연히 사전에 등록된 단어를 형성하는 경우를 활용하며, 플레이어에게 "숨겨진 보물 찾기"와 같은 추가적인 재미 요소를 제공한다.

### 4.2 규칙

| 항목 | 규칙 |
|------|------|
| **보상** | 보너스 단어 1개당 20 코인 |
| **최대 횟수** | 스테이지당 **최대 5개** |
| **최소 길이** | 3글자(음절) 이상 |
| **유효성 검증** | 사전(Dictionary)에 등록된 단어여야 함 |
| **제외 조건** | 퍼즐에 포함된 정답 단어는 보너스로 인정하지 않음 |
| **중복 불가** | 같은 보너스 단어를 두 번 찾아도 1회만 인정 |
| **점수** | 게임 점수에는 반영되지 않음 (코인만 지급) |
| **적용 모드** | Classic, Time Attack, Marathon (Daily Challenge 제외) |

### 4.3 검증 로직

```gdscript
## bonus_word_checker.gd
## 플레이어가 선택한 문자열이 보너스 단어인지 검증한다.

class_name BonusWordChecker
extends RefCounted


## 보너스 단어 1개당 지급 코인
const BONUS_COIN_PER_WORD: int = 20

## 스테이지당 최대 보너스 단어 수
const MAX_BONUS_WORDS_PER_STAGE: int = 5

## 보너스 단어로 인정하는 최소 글자 수
const MIN_BONUS_WORD_LENGTH: int = 3

## 이 스테이지에서 찾은 보너스 단어 목록
var _found_bonus_words: Array[String] = []

## 이 스테이지의 정답 단어 목록 (보너스 제외 대상)
var _puzzle_words: Array[String] = []

## 사전 데이터 (유효 단어 검증용)
var _dictionary: Dictionary = {}  ## { "WORD": true }


## 초기화.
## 스테이지 시작 시 호출한다.
func setup(placed_words: Array, dictionary_data: Dictionary) -> void:
	_found_bonus_words.clear()
	_puzzle_words.clear()
	_dictionary = dictionary_data

	for pw in placed_words:
		_puzzle_words.append(pw.display.to_upper())


## 선택된 텍스트가 보너스 단어인지 확인한다.
## 보너스 단어이면 true를 반환하고 코인을 지급한다.
func check_bonus_word(selected_text: String) -> bool:
	var normalized := selected_text.to_upper()

	# 최소 길이 미달
	if normalized.length() < MIN_BONUS_WORD_LENGTH:
		return false

	# 이미 찾은 보너스 단어
	if normalized in _found_bonus_words:
		return false

	# 최대 보너스 수 초과
	if _found_bonus_words.size() >= MAX_BONUS_WORDS_PER_STAGE:
		return false

	# 퍼즐 정답 단어인 경우 보너스 아님
	if normalized in _puzzle_words:
		return false

	# 사전에 등록된 유효 단어인지 확인
	if _is_valid_dictionary_word(normalized):
		_found_bonus_words.append(normalized)
		CoinManager.earn(BONUS_COIN_PER_WORD, "bonus_word")
		return true

	return false


## 사전에 등록된 유효 단어인지 확인한다.
func _is_valid_dictionary_word(word: String) -> bool:
	return _dictionary.has(word)


## 이 스테이지에서 찾은 보너스 단어 수를 반환한다.
func get_found_count() -> int:
	return _found_bonus_words.size()


## 이 스테이지에서 찾은 보너스 단어 목록을 반환한다.
func get_found_words() -> Array[String]:
	return _found_bonus_words.duplicate()
```

### 4.5 사전 데이터 관리

보너스 단어 검증을 위해 별도의 사전 파일이 필요하다. 이 사전은 격자에 우연히 형성될 수 있는 단어를 검증하는 용도이다.

| 항목 | 영어 | 한국어 |
|------|------|--------|
| **파일 형식** | `res://data/dictionary_en.txt` | `res://data/dictionary_ko.txt` |
| **단어 수** | 약 10,000~50,000 | 약 5,000~20,000 |
| **로딩** | 게임 시작 시 Dictionary로 로드 | 동일 |
| **메모리** | HashSet 형태 (O(1) 조회) | 동일 |

```gdscript
## dictionary_loader.gd
## 보너스 단어 검증용 사전을 로드한다.

static func load_dictionary(language: String) -> Dictionary:
	var path := "res://data/dictionary_%s.txt" % language
	var dict: Dictionary = {}

	if not FileAccess.file_exists(path):
		push_warning("Dictionary file not found: " + path)
		return dict

	var file := FileAccess.open(path, FileAccess.READ)
	while not file.eof_reached():
		var word := file.get_line().strip_edges().to_upper()
		if word.length() >= 3:
			dict[word] = true

	return dict
```

## 5. Marathon / Infinite Mode

### 5.1 개요

Marathon Mode는 **끝없이 이어지는 스테이지**를 최대한 오래 생존하며 높은 점수를 쌓는 모드이다. 3개의 라이프(Lives)가 주어지며, 각 스테이지에 120초 시간 제한이 있다. 시간 초과 시 라이프 1개를 잃으며, 모든 라이프를 잃으면 게임 오버가 된다.

### 5.2 규칙

| 항목 | 규칙 |
|------|------|
| **라이프** | 3개 (하트 아이콘으로 표시) |
| **스테이지별 시간** | 120초 |
| **시간 초과** | 라이프 -1, 다음 스테이지로 이동 (미발견 단어는 건너뜀) |
| **게임 오버** | 라이프 0이 되면 종료 |
| **휴식 스테이지** | **없음** (끊임없는 긴장감 유지) |
| **DDA** | **적용** (생존 시간 연장을 위한 밸런싱) |
| **False Lead** | 적용 (스테이지 11부터) |
| **난이도 증가** | Classic과 동일한 격자/단어 수 증가 규칙 |
| **점수** | 누적 점수 (모든 스테이지 합산) |
| **힌트** | 모든 힌트 사용 가능 (TimerExtend 제외: 고정 120초) |
| **리더보드** | 누적 점수 기반 글로벌/지역 리더보드 |
| **해금 조건** | Classic Mode 스테이지 20 클리어 |

### 5.3 라이프 시스템

```gdscript
## marathon_life_manager.gd
## Marathon Mode의 라이프를 관리한다.

const INITIAL_LIVES: int = 3
const MAX_LIVES: int = 3
const STAGE_TIME_LIMIT: float = 120.0  ## 스테이지별 시간 제한 (초)

var current_lives: int = INITIAL_LIVES
var total_score: int = 0
var stages_cleared: int = 0

signal life_lost(remaining: int)
signal game_over(final_score: int, stages_cleared: int)


## Marathon 초기화.
func start() -> void:
	current_lives = INITIAL_LIVES
	total_score = 0
	stages_cleared = 0


## 스테이지 클리어 시 호출한다.
func on_stage_cleared(stage_score: int) -> void:
	total_score += stage_score
	stages_cleared += 1


## 시간 초과 시 호출한다.
## 라이프를 1개 감소시키고, 게임 오버 여부를 반환한다.
func on_time_expired() -> bool:
	current_lives -= 1
	life_lost.emit(current_lives)

	if current_lives <= 0:
		game_over.emit(total_score, stages_cleared)
		return true  # 게임 오버

	return false  # 계속 플레이


## 라이프를 추가한다 (Rewarded Ad 등으로 부활 시).
func add_life(count: int = 1) -> void:
	current_lives = mini(current_lives + count, MAX_LIVES)
```

[Home > Marathon] → [초기화: 라이프 3, 점수 0]
    │

[스테이지 N 시작] ← ──────────────────────────────────────┐

### 5.5 Marathon 스코어링

| 항목 | 공식 | 설명 |
|------|------|------|
| **단어 점수** | Classic과 동일 | `50 + (length * 10)` + 콤보 |
| **스테이지 보너스** | `stages_cleared * 50` | 스테이지가 올라갈수록 누적 보너스 증가 |
| **라이프 보너스** | `remaining_lives * 500` | 게임 오버 시 남은 라이프에 비례 |
| **최종 점수** | 위 모든 점수 합산 | 리더보드에 등록 |

```gdscript
## Marathon 최종 점수 계산
func calculate_marathon_final_score(
	cumulative_score: int,
	stages_cleared: int,
	remaining_lives: int
) -> int:
	var stage_bonus := stages_cleared * 50
	var life_bonus := remaining_lives * 500
	return cumulative_score + stage_bonus + life_bonus
```

### 5.6 부활 시스템

Game Over 시 Rewarded Ad를 시청하면 라이프 1개를 회복하고 계속 플레이할 수 있다. 부활은 세션당 **최대 1회**로 제한한다.

```gdscript
## Marathon 부활
var _has_revived: bool = false

func try_revive() -> bool:
	if _has_revived:
		return false  # 이미 부활 사용함

	# Rewarded Ad 시청 성공 시
	_has_revived = true
	current_lives = 1
	return true
```
