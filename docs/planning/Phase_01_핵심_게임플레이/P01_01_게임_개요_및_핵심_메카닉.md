## 1. 게임 컨셉

**Classic Word Search Puzzle** (단어 찾기 퍼즐)

플레이어는 격자(Grid) 안에 숨겨진 단어를 찾아내는 퍼즐 게임을 플레이한다. 격자에는 알파벳 또는 한글 음절이 배치되어 있으며, 플레이어는 **드래그(Drag)** 또는 **탭(Tap)** 방식으로 첫 글자에서 마지막 글자까지 선택하여 단어를 찾는다.

### 기본 게임 루프

```
[스테이지 진입] → [격자 생성 + 단어 배치] → [플레이어 단어 탐색]
    → [단어 발견 시 점수 획득] → [모든 단어 발견] → [스테이지 클리어]
```

### 핵심 특징

- **스테이지 기반 진행**: 스테이지가 올라갈수록 격자 크기와 단어 수 증가
- **다국어 지원**: 영어(English), 한국어(Korean) 두 언어 지원
- **멀티 플랫폼**: 모바일(터치), PC(마우스), TV(D-pad) 입력 지원
- **캐주얼 지향**: 오답 패널티 없음, 힌트 시스템 제공

## 4. 격자(Grid) 시스템

### 4.1 격자 크기 증가 규칙

격자 크기는 **스테이지 기반으로 자동 계산**되며, 4스테이지마다 하나의 그룹(Group)을 형성한다. 최소 **5x5**에서 시작하여 최대 **10x10**까지 증가한다.

**증가 패턴:**
- 가로(width)와 세로(height)가 **교대로** 1씩 증가
- 홀수 그룹 전환 시 → 가로(width) +1
- 짝수 그룹 전환 시 → 세로(height) +1

### 4.2 계산 공식

```gdscript
## grid_calculator.gd
## 스테이지 번호로부터 격자 크기와 단어 수를 계산하는 유틸리티

class_name GridCalculator

## 스테이지에 해당하는 그룹 인덱스를 계산한다.
## 4 스테이지마다 하나의 그룹을 구성한다.
static func get_group_index(stage: int) -> int:
	return int(floor(float(stage - 1) / 4.0))

## 스테이지에 해당하는 격자 가로 크기를 반환한다.
## 홀수 그룹 전환 시 가로가 1 증가하며, 최대 10이다.
static func get_grid_width(stage: int) -> int:
	var group_index := get_group_index(stage)
	return mini(5 + int(floor(float(group_index + 1) / 2.0)), 10)

## 스테이지에 해당하는 격자 세로 크기를 반환한다.
## 짝수 그룹 전환 시 세로가 1 증가하며, 최대 10이다.
static func get_grid_height(stage: int) -> int:
	var group_index := get_group_index(stage)
	return mini(5 + int(floor(float(group_index) / 2.0)), 10)

## 스테이지에 해당하는 배치 단어 수를 반환한다.
## 5 스테이지마다 1개씩 증가하며, 최대 12개이다.
static func get_word_count(stage: int) -> int:
	return mini(4 + int(floor(float(stage - 1) / 5.0)), 12)

## 스테이지 설정 정보를 Dictionary로 반환한다.
static func get_stage_config(stage: int) -> Dictionary:
	return {
		"stage": stage,
		"width": get_grid_width(stage),
		"height": get_grid_height(stage),
		"word_count": get_word_count(stage),
		"group_index": get_group_index(stage)
	}
```

### 4.3 스테이지별 격자 크기 표

| 스테이지 | 그룹 Index | 격자 크기 (W x H) | 단어 수 | 비고 |
|----------|-----------|-------------------|---------|------|
| 1~4 | 0 | **5 x 5** | 4 | 튜토리얼 구간 |
| 5~8 | 1 | **6 x 5** | 4~5 | 가로 +1 |
| 9~12 | 2 | **6 x 6** | 5~6 | 세로 +1 |
| 13~16 | 3 | **7 x 6** | 6~7 | 가로 +1 |
| 17~20 | 4 | **7 x 7** | 7 | 세로 +1 |
| 21~24 | 5 | **8 x 7** | 8 | 가로 +1 |
| 25~28 | 6 | **8 x 8** | 8~9 | 세로 +1 |
| 29~32 | 7 | **9 x 8** | 9~10 | 가로 +1 |
| 33~36 | 8 | **9 x 9** | 10~11 | 세로 +1 |
| 37~40 | 9 | **10 x 9** | 11 | 가로 +1 |
| 41+ | 10 | **10 x 10** | 12 | 최대 크기 도달 |

### 4.4 단어 최대 길이 제약

단어 배치 가능한 최대 길이는 격자의 가로/세로 크기에 의해 제한된다.

**영어 (English):**
- 단어 최대 길이 = `min(grid_width, grid_height)`
- 예: 5x5 격자 → 최대 5글자, 7x6 격자 → 최대 6글자 (대각선 제외 시)
- 대각선 방향은 `min(grid_width, grid_height)` 길이까지 배치 가능

**한국어 (Korean):**
- 한글은 한 음절이 한 셀을 차지함
- 격자 크기 7x6 이상일 때 최대 **5음절** 제한 권장
- 이는 한국어 단어 데이터의 평균 길이와 가독성을 고려한 값

```gdscript
## 단어 최대 길이 계산
static func get_max_word_length(width: int, height: int, language: String) -> int:
	var grid_max := mini(width, height)
	if language == "ko":
		# 한국어: 최대 5음절 또는 격자 제한 중 작은 값
		if width >= 7 and height >= 6:
			return mini(5, grid_max)
		else:
			return grid_max
	else:
		# 영어: 격자 크기에 따른 제한
		return grid_max
```

## 5. 단어 배치 방향

격자 내 단어는 **8방향**으로 배치될 수 있다. 각 방향은 `Vector2i`로 표현한다.

### 방향 벡터 정의

| 방향 이름 | Vector2i | 설명 |
|-----------|----------|------|
| Right (→) | `(1, 0)` | 좌 → 우 수평 |
| Left (←) | `(-1, 0)` | 우 → 좌 수평 |
| Down (↓) | `(0, 1)` | 위 → 아래 수직 |
| Up (↑) | `(0, -1)` | 아래 → 위 수직 |
| Down-Right (↘) | `(1, 1)` | 좌상 → 우하 대각선 |
| Up-Left (↖) | `(-1, -1)` | 우하 → 좌상 대각선 |
| Down-Left (↙) | `(-1, 1)` | 우상 → 좌하 대각선 |
| Up-Right (↗) | `(1, -1)` | 좌하 → 우상 대각선 |

### 역방향 매칭 (Reverse Matching)

플레이어가 단어를 **역방향으로 드래그**해도 정답으로 인정한다.

예: "HELLO"가 `(1,0)` 방향(→)으로 배치된 경우, 플레이어가 `(-1,0)` 방향(←)으로 "OLLEH"를 드래그해도 "HELLO"로 매칭되어 정답 처리된다.

```gdscript
## 역방향 매칭 검증
func check_word_match(selected_cells: Array[Vector2i], placed_words: Array[PlacedWord]) -> PlacedWord:
	for pw in placed_words:
		if pw.is_found:
			continue
		# 정방향 매칭
		if selected_cells == pw.cells:
			return pw
		# 역방향 매칭
		var reversed_cells := pw.cells.duplicate()
		reversed_cells.reverse()
		if selected_cells == reversed_cells:
			return pw
	return null
```

## 6. 단어 선택 인터랙션

### 6.1 드래그 방식 (Drag Mode)

가장 기본적인 입력 방식으로, 모바일 터치 및 PC 마우스에 모두 대응한다.

**플로우:**
1. 플레이어가 **첫 번째 글자 셀**을 **누른 상태로 유지** (Press)
2. 손가락/마우스를 **드래그**하며 이동
3. `DirectionSnapper`가 드래그 방향을 **8방향 중 하나로 스냅**
4. 스냅된 방향의 경로에 있는 셀들이 **실시간 하이라이트** 표시
5. 손가락/마우스를 **뗌** (Release)
6. 선택된 셀 목록을 기반으로 **단어 매칭 판정**

```
[Press 셀(2,3)] → [Drag 이동] → [DirectionSnapper 스냅]
    → [셀 하이라이트 갱신] → [Release] → [단어 매칭 검사]
```

### 6.2 탭 방식 (Tap Mode)

모바일 환경에서 작은 셀을 정확히 드래그하기 어려운 경우를 위한 보조 입력 방식이다.

**플로우:**
1. 플레이어가 **첫 번째 글자 셀**을 **탭** (Tap)
2. 해당 셀이 **선택 상태**로 하이라이트
3. 플레이어가 **마지막 글자 셀**을 **탭** (Tap)
4. 첫 번째 셀과 마지막 셀 사이의 방향을 `DirectionSnapper`로 계산
5. 경로 상의 모든 셀이 **자동 선택**
6. **단어 매칭 판정**

```
[Tap 셀(2,3)] → [첫 번째 셀 하이라이트]
    → [Tap 셀(5,3)] → [방향 계산: (1,0)]
    → [경로 셀 자동 선택: (2,3)(3,3)(4,3)(5,3)]
    → [단어 매칭 검사]
```

**탭 모드 취소:** 첫 번째 셀을 다시 탭하면 선택이 취소된다.

### 6.3 시각적 피드백

| 상태 | 동작 | 시각적 표현 |
|------|------|-------------|
| **실시간 하이라이트** | 드래그 중 경로 셀 표시 | 셀 배경색 변경 (연한 파란색) + 라인 오버레이 |
| **정답 시 색상 고정** | 매칭 성공 | 셀 배경색 고정 (단어별 고유 색상) + 완료 애니메이션 |
| **오답 시 해제** | 매칭 실패 | 하이라이트 해제 + 원래 상태로 복귀 (패널티 없음) |

```gdscript
## 드래그 종료 시 판정 처리
func _on_drag_ended(selected_cells: Array[Vector2i]) -> void:
	var matched_word := check_word_match(selected_cells, grid_data.placed_words)
	if matched_word:
		# 정답: 색상 고정 + 점수 처리
		matched_word.is_found = true
		_highlight_found_word(matched_word)
		_add_score(matched_word)
		word_found.emit(matched_word)
	else:
		# 오답: 하이라이트 해제 (패널티 없음)
		_clear_highlight(selected_cells)
```

## 7. 빈 칸 채우기

단어 배치 후 남은 빈 셀(empty cell)은 **랜덤 문자로 채워야** 한다. 이때 언어별로 다른 전략을 사용한다.

### 7.1 영어 (English)

**빈도 가중 방식 (Frequency-Weighted):** 배치된 단어들에 사용된 글자를 기반으로 빈도 가중치를 적용하여 랜덤 문자를 선택한다.

```gdscript
## 영어 빈 칸 채우기 - 빈도 가중 방식
func _fill_empty_english(data: GridData) -> void:
	# 1. 배치된 단어에서 사용된 글자 빈도 계산
	var char_frequency: Dictionary = {}
	for pw in data.placed_words:
		for ch in pw.display:
			var upper_ch := ch.to_upper()
			char_frequency[upper_ch] = char_frequency.get(upper_ch, 0) + 1

	# 2. 배치된 단어에 없는 희귀 문자(Q, Z 등)는 풀에서 제외
	var fill_pool: Array[String] = []
	for ch in char_frequency:
		for i in range(char_frequency[ch]):
			fill_pool.append(ch)

	# 3. 풀이 비어있는 경우 기본 영문자 사용 (Q, Z 제외)
	if fill_pool.is_empty():
		var default_chars := "ABCDEFGHIJKLMNOPRSTUVWXY"
		for ch in default_chars:
			fill_pool.append(ch)

	# 4. 빈 셀에 랜덤 배치
	for y in range(data.height):
		for x in range(data.width):
			if data.grid[y][x] == "":
				data.grid[y][x] = fill_pool[randi() % fill_pool.size()]
```

**제외 문자:** 배치된 단어에 Q, Z 등 희귀 문자가 포함되지 않은 경우, 빈 칸 채우기 풀에서도 해당 문자를 제외한다. 이는 플레이어에게 불필요한 혼란을 방지하기 위함이다.

### 7.2 한국어 (Korean)

**WordPack 음절 풀 방식:** 현재 사용 중인 WordPack의 단어들에서 음절을 추출하여 풀을 구성하고, 부족할 경우 **상용 한글 음절**을 백업으로 사용한다.

```gdscript
## 한국어 빈 칸 채우기 - WordPack 음절 풀 방식
const BACKUP_SYLLABLES: Array[String] = [
	"가", "나", "다", "라", "마", "바", "사", "아", "자", "차",
	"카", "타", "파", "하", "고", "노", "도", "로", "모", "보",
	"소", "오", "조", "초", "코", "토", "포", "호", "구", "누",
	"두", "루", "무", "부", "수", "우", "주", "추", "쿠", "투",
	"기", "니", "디", "리", "미", "비", "시", "이", "지", "히",
]

func _fill_empty_korean(data: GridData, word_pack: WordPack) -> void:
	# 1. WordPack 내 모든 단어에서 음절 풀 구성
	var syllable_pool: Array[String] = []
	if word_pack:
		for entry in word_pack.entries:
			for syllable in entry.display:
				syllable_pool.append(syllable)

	# 2. 풀이 부족한 경우 백업 상용 음절 추가
	if syllable_pool.size() < 20:
		syllable_pool.append_array(BACKUP_SYLLABLES)

	# 3. 빈 셀에 랜덤 배치
	for y in range(data.height):
		for x in range(data.width):
			if data.grid[y][x] == "":
				data.grid[y][x] = syllable_pool[randi() % syllable_pool.size()]
```

### 7.3 검증 (Validation)

빈 칸을 채운 후 **의도하지 않은 단어가 우연히 형성**되지 않았는지 검증해야 한다.

```gdscript
## 빈 칸 채우기 후 검증
## - 빈 칸에 의해 새로운 유효 단어가 형성되었는지 확인
## - 발견 시 해당 셀의 문자를 교체하여 재시도
func _validate_no_unintended_words(data: GridData, dictionary: Array[String]) -> bool:
	# 모든 방향, 모든 시작 위치에서 가능한 문자열을 추출하여
	# dictionary에 존재하면서 placed_words에는 없는 단어가 있는지 확인
	var directions := [
		Vector2i(1, 0), Vector2i(0, 1),
		Vector2i(1, 1), Vector2i(1, -1)
	]
	for y in range(data.height):
		for x in range(data.width):
			for dir in directions:
				var word_str := _extract_string(data, Vector2i(x, y), dir, 3, 8)
				for length in range(3, word_str.length() + 1):
					var substr := word_str.substr(0, length)
					if substr in dictionary and not _is_placed_word(data, substr):
						return false  # 의도하지 않은 단어 발견
	return true
```

> **참고:** 검증 실패 시 해당 빈 셀의 문자를 다른 문자로 교체하고 재검증한다. 최대 10회 재시도 후에도 해결되지 않으면 경고 로그를 남기고 진행한다 (플레이에 치명적이지 않음).

## 8. 점수 시스템

### 8.1 Classic 모드

| 항목 | 공식 | 설명 |
|------|------|------|
| **기본 점수** | `base = 50 + (length × 10)` | 단어 길이에 비례한 기본 보상 |
| **콤보 보너스** | `combo = base × 1.5` | 이전 단어 발견 후 **10초 이내** 연속 발견 시 |
| **노힌트 보너스** | `+100` | 스테이지 전체에서 힌트를 **한 번도 사용하지 않은 경우** |

```gdscript
## Classic 모드 점수 계산
func calculate_classic_score(word_length: int, is_combo: bool, used_hint: bool) -> int:
	var base_score: int = 50 + (word_length * 10)

	# 콤보 보너스: 이전 단어 발견 후 10초 이내
	if is_combo:
		base_score = int(base_score * 1.5)

	# 노힌트 보너스는 스테이지 클리어 시 별도 합산
	return base_score

func calculate_stage_clear_bonus(used_hint: bool) -> int:
	if not used_hint:
		return 100
	return 0
```

### 8.2 Time Attack 모드

| 항목 | 공식 | 설명 |
|------|------|------|
| **기본 점수** | `base = 50 + (length × 10)` | Classic과 동일 |
| **시간 보너스** | `time_bonus = remaining_time × 2` | 남은 시간(초) × 2, 스테이지 클리어 시 |
| **콤보 보너스** | `combo = base × 1.5` | Classic과 동일 조건 |

```gdscript
## Time Attack 모드 점수 계산
func calculate_time_attack_score(word_length: int, is_combo: bool) -> int:
	var base_score: int = 50 + (word_length * 10)
	if is_combo:
		base_score = int(base_score * 1.5)
	return base_score

func calculate_time_attack_clear_bonus(remaining_seconds: float) -> int:
	return int(remaining_seconds * 2.0)
```

### 8.3 스테이지 클리어 등급

스테이지 클리어 시 플레이어의 성과에 따라 등급을 부여한다.

| 등급 | 조건 | 보상 배율 |
|------|------|-----------|
| **S** | 힌트 미사용 **AND** 클리어 시간 90초 미만 | ×2.0 |
| **A** | 힌트 미사용 | ×1.5 |
| **B** | 힌트 1회 사용 | ×1.0 |
| **C** | 힌트 2회 이상 사용 | ×0.8 |

```gdscript
## 스테이지 클리어 등급 판정
enum StageGrade { S, A, B, C }

func evaluate_grade(hint_count: int, clear_time_seconds: float) -> StageGrade:
	if hint_count == 0 and clear_time_seconds < 90.0:
		return StageGrade.S
	elif hint_count == 0:
		return StageGrade.A
	elif hint_count == 1:
		return StageGrade.B
	else:
		return StageGrade.C

func get_grade_multiplier(grade: StageGrade) -> float:
	match grade:
		StageGrade.S: return 2.0
		StageGrade.A: return 1.5
		StageGrade.B: return 1.0
		StageGrade.C: return 0.8
		_: return 1.0
```

### 콤보 타이머 관리

```gdscript
## 콤보 타이머 관리
const COMBO_WINDOW: float = 10.0  # 콤보 유효 시간 (초)
var _last_found_time: float = -999.0

func _on_word_found(word: PlacedWord) -> void:
	var current_time := Time.get_ticks_msec() / 1000.0
	var is_combo := (current_time - _last_found_time) <= COMBO_WINDOW
	_last_found_time = current_time

	var score := calculate_classic_score(word.display.length(), is_combo, _used_hint)
	total_score += score
```
