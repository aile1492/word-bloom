## 1. 접근성 철학

### 1.1 핵심 원칙

본 프로젝트의 접근성 설계는 **WCAG 2.1 AA** 가이드라인을 참고하되, 게임이라는 매체의 특성에 맞게 적용한다. 목표는 **"모든 플레이어가 즐길 수 있는 게임"**이며, 접근성 기능은 선택 기능이 아니라 **기본 기능**으로 취급한다.

| 원칙 | 설명 | 적용 예시 |
|------|------|-----------|
| **인지 가능(Perceivable)** | 정보와 UI 요소를 모든 사용자가 인지할 수 있어야 한다 | 색상 외에 아이콘/패턴 병행, 고대비 모드 |
| **조작 가능(Operable)** | 모든 기능을 다양한 입력 방식으로 조작할 수 있어야 한다 | 탭 선택 모드, D-pad 네비게이션, 여유로운 타이머 |
| **이해 가능(Understandable)** | UI와 피드백이 직관적이고 예측 가능해야 한다 | 명확한 진행 표시, 구체적 에러 메시지 |
| **견고함(Robust)** | 다양한 기기/환경에서 안정적으로 동작해야 한다 | 글꼴 크기 조절, 모션 감소, 플랫폼별 적응 |

### 1.2 설계 방향

- 접근성 설정은 **SaveData에 영구 저장**하여 앱 재시작 후에도 유지된다.
- 첫 실행 시 **OS 접근성 설정을 자동 감지**하여 초기값을 결정한다.
- 접근성 관련 변경은 **Signal 기반 반응형 시스템**으로 즉시 반영된다.
- 접근성 기능은 성능에 최소한의 영향만 미치도록 설계한다.

## 2. 접근성 설정 데이터 구조

### 2.1 SaveData 내 접근성 필드

접근성 설정은 `save_data.json`의 `accessibility` 키 하위에 저장된다. P02_03(세이브/로드 시스템)의 SaveManager를 통해 관리한다.

```json
{
  "accessibility": {
    "font_scale": 1,
    "dark_mode": false,
    "dark_mode_follow_system": true,
    "colorblind_mode": "none",
    "high_contrast": false,
    "tap_selection_mode": false,
    "reduce_motion": false,
    "relaxed_timer": false,
    "vibration_feedback": true
  }
}
```

### 2.2 설정 필드 상세

| 필드 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `font_scale` | `int` | `1` | 글꼴 크기 단계. 0=작게, 1=보통, 2=크게 |
| `dark_mode` | `bool` | `false` | 다크 모드 활성화 여부 |
| `dark_mode_follow_system` | `bool` | `true` | OS 다크 모드 연동 여부 |
| `colorblind_mode` | `String` | `"none"` | 색맹 모드. `"none"`, `"protanopia"`, `"deuteranopia"`, `"tritanopia"` |
| `high_contrast` | `bool` | `false` | 고대비 모드 활성화 여부 |
| `tap_selection_mode` | `bool` | `false` | 탭 선택 모드 활성화 여부 (드래그 대안) |
| `reduce_motion` | `bool` | `false` | 모션 감소 모드 활성화 여부 |
| `relaxed_timer` | `bool` | `false` | 여유로운 타이머 활성화 여부 (Time Attack 시간 x1.5) |
| `vibration_feedback` | `bool` | `true` | 진동 피드백 활성화 여부 (모바일 전용) |

### 2.3 열거형 정의

```gdscript
## accessibility_enums.gd
class_name AccessibilityEnums

## 글꼴 크기 단계
enum FontScale {
	SMALL = 0,   ## 작게 (0.85x)
	NORMAL = 1,  ## 보통 (1.0x, 기본값)
	LARGE = 2,   ## 크게 (1.2x)
}

## 색맹 모드 종류
enum ColorblindMode {
	NONE = 0,         ## 없음 (기본)
	PROTANOPIA = 1,   ## 적색맹 (Protanopia)
	DEUTERANOPIA = 2, ## 녹색맹 (Deuteranopia)
	TRITANOPIA = 3,   ## 청색맹 (Tritanopia)
}

## ColorblindMode를 문자열로 변환
static func colorblind_mode_to_string(mode: ColorblindMode) -> String:
	match mode:
		ColorblindMode.NONE: return "none"
		ColorblindMode.PROTANOPIA: return "protanopia"
		ColorblindMode.DEUTERANOPIA: return "deuteranopia"
		ColorblindMode.TRITANOPIA: return "tritanopia"
		_: return "none"

## 문자열을 ColorblindMode로 변환
static func string_to_colorblind_mode(s: String) -> ColorblindMode:
	match s:
		"protanopia": return ColorblindMode.PROTANOPIA
		"deuteranopia": return ColorblindMode.DEUTERANOPIA
		"tritanopia": return ColorblindMode.TRITANOPIA
		_: return ColorblindMode.NONE
```

## 3. 시각 접근성

### 3.1 글꼴 크기 조절

플레이어가 3단계로 글꼴 크기를 조절할 수 있다. 모든 UI 텍스트에 일괄 적용된다.

#### 3.1.1 크기 단계 정의

| 설정 | 배율 | Body 크기 | Grid Cell 글꼴 크기 | 비고 |
|------|:----:|:---------:|:-------------------:|------|
| 작게 | 0.85x | 14sp | 19sp | 시력이 좋은 사용자, 화면 정보 밀도 선호 |
| 보통 (기본) | 1.0x | 16sp | 22sp | 대부분의 사용자에게 적합 |
| 크게 | 1.2x | 19sp | 26sp | 시력이 약한 사용자, 큰 글꼴 선호 |

#### 3.1.2 배율 상수 테이블

```gdscript
## accessibility_manager.gd 내부

## 글꼴 크기 배율 테이블
const FONT_SCALE_MULTIPLIERS: Array[float] = [
	0.85,  # FontScale.SMALL
	1.0,   # FontScale.NORMAL
	1.2,   # FontScale.LARGE
]
```

#### 3.1.3 구현: Godot Theme의 font_size override

Godot 4.6의 Theme 시스템을 활용하여 전역 글꼴 크기를 일괄 변경한다. 루트 Control 노드의 Theme을 override하면 하위 모든 노드에 전파된다.

```gdscript
## accessibility_manager.gd

## 기본 폰트 크기 정의 (배율 1.0x 기준)
const BASE_FONT_SIZES: Dictionary = {
	"body": 16,
	"grid_cell": 22,
	"heading": 24,
	"button": 18,
	"caption": 12,
	"word_list": 18,
	"popup_title": 26,
	"popup_body": 15,
}

## 글꼴 크기를 적용한다.
## 루트 Theme의 font_size를 override하여 하위 노드에 전파한다.
func apply_font_scale(scale_index: int) -> void:
	var multiplier: float = FONT_SCALE_MULTIPLIERS[clampi(scale_index, 0, 2)]

	# 루트 노드의 Theme 참조
	var root_control: Control = _get_root_ui_control()
	if root_control == null:
		push_warning("[AccessibilityManager] 루트 UI Control을 찾을 수 없음")
		return

	var theme: Theme = root_control.theme
	if theme == null:
		theme = Theme.new()
		root_control.theme = theme

	# Label의 기본 font_size override
	var body_size: int = int(BASE_FONT_SIZES["body"] * multiplier)
	theme.set_font_size("font_size", "Label", body_size)

	# Button의 기본 font_size override
	var button_size: int = int(BASE_FONT_SIZES["button"] * multiplier)
	theme.set_font_size("font_size", "Button", button_size)

	# 각 커스텀 타입에 대해서도 설정
	for type_key in BASE_FONT_SIZES.keys():
		var base_size: int = BASE_FONT_SIZES[type_key]
		var scaled_size: int = int(base_size * multiplier)
		# 최소 크기 보장 (11sp 이하로 내려가지 않도록)
		scaled_size = maxi(scaled_size, 11)
		theme.set_font_size("font_size", type_key, scaled_size)

	print("[AccessibilityManager] 글꼴 크기 적용: 배율=%.2f, body=%dsp" % [
		multiplier, body_size
	])
```

#### 3.1.4 격자 셀 글꼴 크기 적용

격자 셀은 Theme 전파와 별도로, 셀 크기에 비례하여 글꼴 크기를 직접 계산한다.

```gdscript
## letter_cell.gd 내 접근성 글꼴 크기 반영

## 접근성 배율이 적용된 글꼴 크기를 계산한다.
func update_font_size_with_accessibility(scale_index: int) -> void:
	var multiplier: float = AccessibilityManager.FONT_SCALE_MULTIPLIERS[
		clampi(scale_index, 0, 2)
	]
	var base_size: int = int(_cell_size * 0.55)
	var target_size: int = int(base_size * multiplier)
	# 최소 14sp, 최대: 셀 크기의 80%
	target_size = clampi(target_size, 14, int(_cell_size * 0.8))
	letter_label.add_theme_font_size_override("font_size", target_size)
```

#### 3.1.5 큰 글꼴 레이아웃 보호

큰 글꼴 설정 시 UI가 깨지지 않도록 다음 규칙을 적용한다.

| 요소 | 보호 규칙 |
|------|-----------|
| 버튼 텍스트 | `text_overrun_behavior = OVERRUN_TRIM_ELLIPSIS` 설정으로 넘침 방지 |
| 단어 목록 | ScrollContainer 내부에 배치하여 스크롤 보장 |
| 상단 바 | `HBoxContainer` + `size_flags_stretch_ratio` 조정으로 비율 유지 |
| 팝업 | `MarginContainer` 최소 여백 보장, 내용이 길면 스크롤 |
| 격자 | 큰 글꼴에서도 격자가 화면에 맞도록 `_cell_size` 자동 축소 |

### 3.2 다크 모드

라이트/다크 두 가지 테마를 지원한다. P08_01(UI 디자인 시스템)에서 정의한 색상 팔레트를 기반으로 한다.

#### 3.2.1 테마 전환 방식

```gdscript
## accessibility_manager.gd

## OS 다크 모드 감지
## Godot 4.6의 DisplayServer API를 사용한다.
func _detect_os_dark_mode() -> bool:
	# DisplayServer.is_dark_mode()는 Godot 4.6에서 지원
	# Windows, macOS, Android, iOS에서 OS 다크 모드 설정을 감지한다
	return DisplayServer.is_dark_mode()


## 다크 모드를 적용한다.
func apply_dark_mode(enabled: bool) -> void:
	var theme_path: String
	if enabled:
		theme_path = "res://assets/themes/dark_theme.tres"
	else:
		theme_path = "res://assets/themes/light_theme.tres"

	var theme: Theme = load(theme_path) as Theme
	if theme == null:
		push_error("[AccessibilityManager] 테마 로드 실패: %s" % theme_path)
		return

	# 루트 Control에 테마 적용
	var root_control: Control = _get_root_ui_control()
	if root_control:
		root_control.theme = theme

	# 글꼴 크기 설정 재적용 (테마 교체 시 font_size override가 초기화됨)
	var current_scale: int = _get_saved_font_scale()
	apply_font_scale(current_scale)

	# 색맹 모드 팔레트도 재적용
	var current_cb_mode: String = _get_saved_colorblind_mode()
	if current_cb_mode != "none":
		apply_colorblind_palette(
			AccessibilityEnums.string_to_colorblind_mode(current_cb_mode)
		)

	print("[AccessibilityManager] 다크 모드: %s" % [
		"ON" if enabled else "OFF"
	])
```

#### 3.2.2 시스템 연동 옵션

`dark_mode_follow_system`이 `true`인 경우, 앱 포커스 복귀 시 OS 다크 모드 상태를 확인하여 자동 전환한다.

```gdscript
## accessibility_manager.gd

## 앱이 포그라운드로 돌아올 때 OS 다크 모드 상태를 확인한다.
func _notification(what: int) -> void:
	if what == NOTIFICATION_APPLICATION_FOCUS_IN:
		if _get_saved_dark_mode_follow_system():
			var os_dark: bool = _detect_os_dark_mode()
			var current_dark: bool = _get_saved_dark_mode()
			if os_dark != current_dark:
				_save_dark_mode(os_dark)
				apply_dark_mode(os_dark)
				dark_mode_changed.emit(os_dark)
```

#### 3.2.3 라이트/다크 테마 색상 요약

| 요소 | 라이트 테마 | 다크 테마 |
|------|-------------|-----------|
| 배경 | `#FFFFFF` | `#1A1A2E` |
| 전경 (본문) | `#212121` | `#E0E0E0` |
| 격자 셀 배경 | `#F2F2F2` | `#2D2D44` |
| 격자 셀 텍스트 | `#333333` | `#E8E8E8` |
| 하이라이트 | `#4A90D9` | `#5BA8F5` |
| 카드/패널 배경 | `#FAFAFA` | `#252540` |
| 테두리 | `#E0E0E0` | `#3D3D5C` |

> **참고**: 라이트/다크 테마 모두 WCAG AA 기준 대비율 4.5:1 이상을 보장해야 한다.

### 3.3 색맹 모드

색상만으로 정보를 전달하지 않도록 설계한다. 찾은 단어의 하이라이트 색상 팔레트를 색맹 유형별로 3세트 추가 제공한다.

#### 3.3.1 색맹 유형별 대응 전략

| 색맹 유형 | 영향받는 색상 | 대체 전략 |
|-----------|-------------|-----------|
| **Protanopia** (적색맹) | 적색-녹색 구분 어려움 | 적색 → 청색으로 대체, 녹색 → 노란색으로 유지 |
| **Deuteranopia** (녹색맹) | 녹색-적색 구분 어려움 | 녹색 → 청색/보라로 대체, 적색 → 주황으로 유지 |
| **Tritanopia** (청색맹) | 청색-노란색 구분 어려움 | 청색 → 적색으로 대체, 노란색 → 녹색으로 유지 |

#### 3.3.2 ColorblindPalette Resource

찾은 단어의 하이라이트 색상 팔레트를 색맹 모드별로 정의한다.

```gdscript
## colorblind_palette.gd
## 색맹 모드별 찾은 단어 하이라이트 색상 팔레트.
## Resource로 정의하여 Inspector에서 편집 가능하다.
class_name ColorblindPalette
extends Resource

## 팔레트 이름 (식별용)
@export var palette_name: String = ""

## 색맹 모드 타입

@export var found_colors: Array[Color] = []

@export var hover_color: Color = Color(0.85, 0.90, 1.0, 1.0)

@export var hint_color: Color = Color(1.0, 0.95, 0.8, 1.0)
```

#### 3.3.4 Protanopia (적색맹) 팔레트

적색-녹색 구분이 어려운 사용자를 위해, 적색 계열을 청색/보라 계열로 대체한다.

```gdscript
## Protanopia 팔레트
static func _create_protanopia_palette() -> ColorblindPalette:
	var p := ColorblindPalette.new()
	p.palette_name = "Protanopia"
```

### 3.4 고대비 모드

시력이 약한 사용자를 위해 전경/배경 대비를 강화한다.

#### 3.4.1 고대비 기준

- **WCAG AA 기준**: 일반 텍스트 대비 4.5:1 이상 (기본 모드에서도 보장)
- **고대비 모드**: 대비 7:1 이상으로 강화 (WCAG AAA 수준)

#### 3.4.2 고대비 시각 효과 변경

| 요소 | 기본 모드 | 고대비 모드 |
|------|-----------|-------------|
| 격자 셀 테두리 | 1px, `#E0E0E0` | 2px, `#000000` (라이트) / `#FFFFFF` (다크) |
| 선택 하이라이트 | 배경색 변경만 | 배경색 + 3px 굵은 테두리 + 내부 그림자 |
| 포커스 링 (TV) | 2px, 파란색 | 4px, 고대비 노란색 `#FFD700` |
| 버튼 테두리 | 없음 또는 1px | 2px 진한 테두리 |
| 텍스트 그림자 | 없음 | 1px 반대색 그림자 (가독성 향상) |

#### 3.4.3 고대비 적용 구현

## 4. 운동 접근성

### 4.1 탭 선택 모드 (드래그 대안)

손가락/손 움직임이 불편한 사용자를 위해 드래그 대신 탭으로 단어를 선택할 수 있는 대안 입력 방식을 제공한다.

#### 4.1.1 두 가지 입력 모드 비교

| 항목 | 드래그 모드 (기본) | 탭 선택 모드 (대안) |
|------|-------------------|-------------------|
| 조작 방식 | 첫 셀 누르고 → 드래그 → 마지막 셀에서 손가락 뗌 | 첫 셀 탭 → (선택적: 중간 셀 탭) → 마지막 셀 탭 → 자동 확정 |
| 필요 동작 | 연속 드래그 (손가락 계속 화면에 접촉) | 개별 탭 (손가락 올렸다 내렸다 가능) |
| 정밀도 요구 | 높음 (드래그 경로 유지) | 낮음 (셀 탭만 정확하면 됨) |
| 설정 위치 | Settings > 접근성 > 탭 선택 모드 | 동일 |

#### 4.1.2 탭 선택 모드 상세 흐름

```
[탭 선택 모드 활성화 상태]

1. 플레이어가 첫 번째 셀을 탭한다
   → 해당 셀이 "선택 시작" 상태로 하이라이트
   → 상태: IDLE → TAP_FIRST

2. 플레이어가 마지막 셀을 탭한다
   → 첫 셀 ~ 마지막 셀 사이의 8방향 직선 경로를 자동 계산
   → 유효한 8방향이 아니면 무시 (시각적 피드백)
   → 상태: TAP_FIRST → IDLE (확정)

3. 중간 셀 탭 (선택 사항)
   → 첫 셀과 중간 셀 사이의 방향을 미리 보여주는 가이드 표시
   → 방향이 결정되면 해당 방향으로 경로 프리뷰 표시

4. 취소
   → 첫 번째 셀을 다시 탭하면 선택 취소
   → 격자 외부 탭으로도 선택 취소
```

#### 4.2.1 최소 크기 규정

| 요소 | 최소 크기 | 기준 |
|------|-----------|------|
| 격자 셀 | 44 x 44dp | Apple HIG / WCAG 2.5.5 |
| 버튼 | 44 x 44dp | Apple HIG |
| 아이콘 버튼 | 48 x 48dp | Material Design |
| 버튼 간 간격 | 8dp 이상 | Material Design |
| 체크박스/토글 | 44 x 44dp (터치 영역) | Apple HIG |

#### 4.2.2 격자 셀 최소 크기 보장

```gdscript
## grid_layout.gd

## 격자 셀 최소 크기 (dp 단위)
## 화면 DPI에 맞게 px로 변환하여 적용한다.
const MIN_CELL_SIZE_DP: float = 44.0

## dp를 px로 변환한다.
## 기준 DPI: 160 (Android mdpi)
static func dp_to_px(dp: float) -> float:
	var screen_dpi: int = DisplayServer.screen_get_dpi()
	if screen_dpi <= 0:
		screen_dpi = 160  # fallback
	return dp * (float(screen_dpi) / 160.0)


## 격자 크기 계산 시 최소 셀 크기를 보장한다.
## 화면 크기와 격자 차원에 맞춰 셀 크기를 계산하되,
## MIN_CELL_SIZE_DP 이하로 내려가지 않도록 한다.
static func calculate_cell_size(
	available_width: float,
	available_height: float,
	grid_cols: int,
	grid_rows: int,
	cell_gap: float
) -> float:
	var min_px: float = dp_to_px(MIN_CELL_SIZE_DP)

	var max_cell_w: float = (available_width - cell_gap * (grid_cols - 1)) / grid_cols
	var max_cell_h: float = (available_height - cell_gap * (grid_rows - 1)) / grid_rows
	var cell_size: float = minf(max_cell_w, max_cell_h)

	# 최소 크기 보장
	cell_size = maxf(cell_size, min_px)

	return cell_size
```

버튼의 시각적 크기가 44dp보다 작을 경우, 투명한 터치 영역을 확장하여 최소 크기를 보장한다.

|------|:-------------------:|:------------:|
| Classic | 해당 없음 (시간 제한 없음) | - |
| Time Attack | O (x1.5) | 별도 카테고리 |
| Daily Challenge | O (x1.5) | "여유 모드" 표시 |
| Marathon | O (x1.5) | 별도 카테고리 |

## 5. 인지 접근성

### 5.1 모션 감소

화면 전환 애니메이션, 셀 효과, 배경 애니메이션 등 움직임 요소를 최소화하여 전정기관 과민증(vestibular disorder)이나 주의력 산만 문제가 있는 사용자를 지원한다.

#### 5.1.1 모션 감소 시 변경 사항

| 요소 | 기본 동작 | 모션 감소 시 |
|------|-----------|-------------|
| 화면 전환 | 슬라이드/페이드 애니메이션 (0.3초) | 즉시 전환 (0초) |
| 셀 선택 하이라이트 | 펄스 애니메이션 (반복 확대/축소) | 색상 변경만 (애니메이션 없음) |
| 정답 발견 효과 | 팝업 + 바운스 + 파티클 | 색상 변경만 (즉시) |
| 배경 애니메이션 | 느린 그래디언트 이동 | 정적 배경 (정지) |
| 코인 획득 연출 | 코인 아이콘 날아가기 + 숫자 카운트업 | 즉시 반영 (숫자만 갱신) |
| 단어 목록 취소선 | 슬라이드 인 취소선 애니메이션 | 즉시 취소선 적용 |
| 격자 등장 | 셀이 하나씩 나타나는 캐스케이드 | 모든 셀 동시 표시 |
| 팝업 등장 | 스케일 업 + 페이드 인 | 즉시 표시 |

### 5.2 명확한 피드백

모든 게임 이벤트에 대해 명확하고 일관된 피드백을 제공한다.

#### 5.2.1 사운드-시각 대응 테이블

모든 사운드 효과에 시각적 대응이 있어 소리 없이도 게임 상태를 인지할 수 있다.

| 게임 이벤트 | 사운드 효과 | 시각적 피드백 |
|------------|-----------|-------------|
| 셀 선택 시작 | 짧은 틱 소리 | 셀 배경색 변경 + 테두리 강조 |
| 드래그 중 셀 추가 | 톤 상승음 | 새 셀 하이라이트, 경로 라인 표시 |
| 정답 단어 발견 | 성공 차임 | 셀 색상 고정 + 팝업 효과 + 단어 목록 취소선 |
| 오답 시도 | 오류 버저 | 셀 흔들림 + 빨간 깜빡임 → 원래 상태 복귀 |
| 힌트 사용 | 스파클 사운드 | 힌트 셀 노란색 + 별 아이콘 |
| 스테이지 클리어 | 축하 팡파레 | 클리어 팝업 + 별 표시 + 점수 |
| 타이머 경고 (10초) | 경고 비프음 | 화면 테두리 빨간색 + 타이머 깜빡임 |
| 시간 초과 | 실패 사운드 | 실패 팝업 + 격자 흐릿해짐 |
| 코인 획득 | 코인 소리 | 코인 아이콘 + 숫자 증가 |

## 6. 청각 접근성

### 6.1 설계 원칙

본 게임에는 대화(dialogue)나 내레이션(narration)이 없으므로, 자막(caption) 시스템은 필요하지 않다. 대신 모든 게임 이벤트가 **시각적 피드백을 동반**하도록 설계한다.

### 6.2 청각-시각 대응 보장

| 원칙 | 적용 |
|------|------|
| 소리 없이도 게임 가능 | 모든 사운드 이벤트에 시각적 대응 (5.2.1 테이블 참조) |
| 타이머 경고 다중 채널 | 사운드 + 화면 테두리 색 변화 + 진동 (모바일) |
| BGM 없이도 분위기 전달 | UI 색상/레이아웃으로 긴장감 표현 |
| 설정 독립적 | 사운드 OFF 시에도 모든 정보 인지 가능 |

### 6.3 타이머 경고 다중 피드백

Time Attack 모드에서 시간이 부족할 때 3가지 채널로 경고한다.

```gdscript
## timer_warning.gd

## 타이머 경고를 표시한다.
## 소리, 시각, 진동 세 가지 채널로 동시에 경고한다.
func show_timer_warning(remaining_seconds: float) -> void:
	# 1. 사운드 경고 (설정에서 활성화된 경우)
	if SoundManager.is_sfx_enabled():
		SoundManager.play_sfx("timer_warning")

	# 2. 시각 경고: 화면 테두리 색 변화
	_flash_screen_border(Color(1.0, 0.3, 0.3, 0.6))

	# 3. 진동 경고 (모바일 + 설정에서 활성화된 경우)
	if _is_vibration_enabled():
		Input.vibrate_handheld(200)  # 200ms 진동


## 화면 테두리를 일시적으로 색상 변경하여 경고한다.
func _flash_screen_border(color: Color) -> void:
	var border_rect: ColorRect = _screen_border_overlay
	border_rect.color = color
	border_rect.visible = true

	if AccessibilityManager.should_animate():
		var tween := create_tween()
		tween.tween_property(border_rect, "color:a", 0.0, 0.5)
		tween.tween_callback(func(): border_rect.visible = false)
	else:
		# 모션 감소: 1초간 표시 후 숨김
		await get_tree().create_timer(1.0).timeout
		border_rect.visible = false


## 진동 피드백 가능 여부를 확인한다.
func _is_vibration_enabled() -> bool:
	# 모바일 플랫폼인지 확인
	if not (OS.has_feature("mobile") or OS.has_feature("android") or OS.has_feature("ios")):
		return false
	# 접근성 설정에서 진동이 활성화되어 있는지 확인
	return SaveManager.load_value("accessibility/vibration_feedback", true)
```

## 7. TV 접근성 특수 고려

Android TV, Fire TV 등 리모컨/게임패드 환경에서의 접근성을 보장한다. P01_03(입력 시스템) 및 CC_02(TV 모드)와 연동한다.

### 7.1 D-pad 전체 UI 접근

리모컨의 방향키(D-pad)만으로 모든 UI 요소에 접근할 수 있어야 한다.

| 요구사항 | 구현 방식 |
|---------|----------|
| 모든 인터랙티브 요소에 포커스 가능 | `focus_mode = Control.FOCUS_ALL` 설정 |
| 포커스 순서 직관적 | 시각적 레이아웃 순서와 일치하도록 `focus_neighbor_*` 설정 |
| 포커스 트랩 없음 | 모든 화면에서 Back 버튼으로 이전 화면 복귀 가능 |
| 초기 포커스 자동 설정 | 각 화면 진입 시 가장 중요한 요소에 `grab_focus()` 호출 |

### 7.2 포커스 시각 효과

포커스된 요소가 명확히 보여야 한다. 색상 + 크기 변화 + 애니메이션 세 가지를 복합적으로 사용한다.

```gdscript
## tv_focus_manager.gd
## TV 모드 포커스 시각 효과 관리.
## 포커스된 Control에 강조 효과를 적용한다.

## 포커스 효과 적용
func apply_focus_effect(control: Control) -> void:
	# 1. 테두리 색상 변경
	var focus_style := StyleBoxFlat.new()
	focus_style.border_color = Color(0.35, 0.65, 1.0, 1.0)  # 밝은 파란색
	focus_style.border_width_left = 3
	focus_style.border_width_right = 3
	focus_style.border_width_top = 3
	focus_style.border_width_bottom = 3
	focus_style.corner_radius_top_left = 6
	focus_style.corner_radius_top_right = 6
	focus_style.corner_radius_bottom_left = 6
	focus_style.corner_radius_bottom_right = 6
	control.add_theme_stylebox_override("focus", focus_style)

	# 2. 크기 변화 (1.05x 확대)
	if AccessibilityManager.should_animate():
		control.pivot_offset = control.size / 2.0
		var tween := control.create_tween()
		tween.tween_property(control, "scale", Vector2(1.05, 1.05), 0.1) \
			.set_ease(Tween.EASE_OUT).set_trans(Tween.TRANS_QUAD)
	else:
		control.scale = Vector2(1.05, 1.05)

	# 3. 고대비 모드에서 추가 강조
	if AccessibilityManager.is_high_contrast():
		focus_style.border_width_left = 5
		focus_style.border_width_right = 5
		focus_style.border_width_top = 5
		focus_style.border_width_bottom = 5
		focus_style.border_color = Color(1.0, 0.84, 0.0, 1.0)  # 골드


## 포커스 효과 해제
func remove_focus_effect(control: Control) -> void:
	control.remove_theme_stylebox_override("focus")

	if AccessibilityManager.should_animate():
		var tween := control.create_tween()
		tween.tween_property(control, "scale", Vector2(1.0, 1.0), 0.1) \
			.set_ease(Tween.EASE_OUT).set_trans(Tween.TRANS_QUAD)
	else:
		control.scale = Vector2.ONE
```

### 7.3 Back 버튼 네비게이션

리모컨의 Back 버튼은 항상 이전 화면으로 돌아갈 수 있어야 한다.

```gdscript
## tv_back_handler.gd
## TV 모드 Back 버튼 처리.
## 각 화면에서 ui_cancel 액션을 처리하여 이전 화면으로 복귀한다.

## 화면 스택 (뒤로 가기 경로)
var _screen_stack: Array[String] = []


func _unhandled_input(event: InputEvent) -> void:
	if event.is_action_pressed("ui_cancel"):
		_go_back()
		get_viewport().set_input_as_handled()


func _go_back() -> void:
	if _screen_stack.is_empty():
		# 스택이 비어있으면 홈 화면으로 이동
		# (홈 화면에서 Back은 앱 종료 확인 팝업)
		_show_exit_confirmation()
		return

	var previous_screen: String = _screen_stack.pop_back()
	get_tree().change_scene_to_file(previous_screen)
```

## 9. accessibility_manager.gd 구현

### 9.1 설계 결정

`AccessibilityManager`는 **Autoload 싱글턴이 아니다**. SaveManager 내 접근성 설정을 참조하며, 전역 접근성 상태를 관리하고, 설정 변경 시 Signal을 발행한다. 다른 매니저들이 Signal을 수신하여 반영한다.

> **Autoload가 아닌 이유**: 접근성 상태는 SaveManager가 이미 저장/로드를 담당하므로, 별도 싱글턴을 만들지 않는다. 대신 GameManager(Autoload) 내에 인스턴스로 포함하거나, static 함수로 접근한다.

### 9.2 클래스 전체 구조

```gdscript

## 전역 접근성 상태 관리 및 설정 변경 Signal 발행.
```

## 10. 첫 실행 시 OS 설정 자동 감지

앱 최초 실행 시 OS의 접근성 관련 설정을 감지하여 초기값을 결정한다.

### 10.1 감지 항목 및 API

| 감지 항목 | Godot 4.6 API | 적용 설정 |
|-----------|-------------|-----------|
| OS 다크 모드 | `DisplayServer.is_dark_mode()` | `dark_mode` |
| 모션 감소 | `DisplayServer.is_reduce_motion_enabled()` (가용 시) | `reduce_motion` |
| 화면 DPI | `DisplayServer.screen_get_dpi()` | 셀 크기 계산에 활용 |

### 10.2 첫 실행 감지 구현

```gdscript
## accessibility_manager.gd

## 첫 실행 시 OS 접근성 설정을 감지하여 기본값을 설정한다.
## SaveData에 "accessibility_initialized" 키가 없으면 첫 실행으로 판단한다.
func check_first_run_accessibility() -> void:
	var initialized: bool = SaveManager.load_value(
		"accessibility_initialized", false
	)
	if initialized:
		return

	print("[AccessibilityManager] 첫 실행: OS 접근성 설정 감지 중...")

	# 1. OS 다크 모드 감지
	var os_dark: bool = false
	if DisplayServer.has_feature(DisplayServer.FEATURE_ACCESSIBILITY):
		os_dark = DisplayServer.is_dark_mode()
	SaveManager.save_value("accessibility/dark_mode", os_dark)
	SaveManager.save_value("accessibility/dark_mode_follow_system", true)

	# 2. 모션 감소 감지
	# Godot 4.6에서 DisplayServer.is_reduce_motion_enabled()가
	# 지원되지 않을 수 있으므로 안전하게 처리한다.
	var os_reduce_motion: bool = false
	if DisplayServer.has_method("is_reduce_motion_enabled"):
		os_reduce_motion = DisplayServer.call("is_reduce_motion_enabled")
	SaveManager.save_value("accessibility/reduce_motion", os_reduce_motion)

	# 3. TV 플랫폼 감지 시 기본 설정 조정
	if LayoutManager.is_tv:
		# TV에서는 탭 선택 모드가 기본 (D-pad 조작이므로 무관하지만 일관성)
		# 글꼴 크기를 '크게'로 기본 설정 (TV는 시청 거리가 멀기 때문)
		SaveManager.save_value("accessibility/font_scale", 2)
		# 진동 비활성화 (TV 리모컨은 진동 미지원)
		SaveManager.save_value("accessibility/vibration_feedback", false)

	# 4. 초기화 완료 표시
	SaveManager.save_value("accessibility_initialized", true)
	SaveManager.flush()

	print("[AccessibilityManager] 첫 실행 설정 완료: dark=%s, reduce_motion=%s" % [
		os_dark, os_reduce_motion
	])
```
