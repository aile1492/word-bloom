## 1. 게임 용어

### 1.1 핵심 구조

**Grid (격자)**: 단어 찾기 퍼즐의 게임판을 구성하는 2차원 배열. 행(Row)과 열(Column)로 이루어지며, 각 칸에 하나의 글자가 배치된다.

**Cell (셀)**: Grid를 구성하는 개별 칸 하나. 각 Cell은 좌표(row, col), 표시 문자, 상태(기본/선택 중/발견 완료) 정보를 갖는다.

**Word Pack (단어 팩)**: 특정 테마에 속하는 단어들의 묶음. 하나의 Word Pack은 여러 스테이지를 포함하며, 플레이어가 선택하여 진행할 수 있는 콘텐츠 단위이다.

**Theme (테마)**: Word Pack을 분류하는 상위 카테고리. 예를 들어 "동물", "음식", "여행" 등이 있으며, 시각적 스타일(배경, 색상 팔레트)도 테마에 따라 달라진다.

**Stage (스테이지)**: 플레이어가 한 번에 풀어야 하는 하나의 퍼즐 단위. 하나의 Grid와 찾아야 할 단어 목록으로 구성된다.

### 1.2 난이도 시스템

**DDA (Dynamic Difficulty Adjustment, 동적 난이도 조절)**: 플레이어의 실시간 퍼포먼스(정답률, 소요 시간, 힌트 사용 빈도 등)를 분석하여 다음 스테이지의 난이도를 자동으로 조절하는 시스템.

**False Lead (거짓 단서/가짜 단서)**: Grid 내에서 정답 단어처럼 보이지만 실제로는 정답이 아닌 문자 배열. 난이도를 높이기 위해 의도적으로 배치되며, DDA에 의해 그 밀도가 조절된다.

**Sawtooth Pattern (톱니 패턴)**: 난이도가 점진적으로 상승한 뒤 일시적으로 하락하는 것을 반복하는 난이도 곡선 형태. 플레이어에게 성취감과 휴식을 번갈아 제공하여 이탈을 방지한다.

**Rest Stage (휴식 스테이지)**: Sawtooth Pattern에서 난이도가 의도적으로 낮아지는 구간에 해당하는 스테이지. 플레이어가 쉽게 클리어하며 자신감을 회복할 수 있도록 설계된다.

### 1.3 인게임 메커니즘

**Streak (연속 기록)**: 플레이어가 힌트 없이 연속으로 스테이지를 클리어한 횟수. 연속 기록이 높을수록 추가 보상이 주어진다.

**Combo (연속 발견)**: 짧은 시간 간격 내에 단어를 연속으로 찾았을 때 발동하는 보너스 시스템. 콤보 수가 높아질수록 획득 코인이 증가한다.

**Hint (힌트)**: 플레이어가 막혔을 때 사용할 수 있는 도움 기능. 특정 단어의 첫 글자를 강조하거나, 해당 단어가 위치한 방향을 알려주는 등의 방식으로 작동한다.

**Booster (부스터)**: 게임 플레이를 유리하게 만드는 일회성 아이템. 힌트와 달리 Grid 전체에 영향을 주거나, 시간을 연장하는 등 보다 강력한 효과를 제공한다.

**Bonus Word (보너스 워드)**: 스테이지의 필수 정답 목록에는 포함되지 않지만, Grid 안에 숨겨져 있어 찾으면 추가 보상을 받는 단어. 탐색의 재미를 높인다.

### 1.4 게임 모드

**Classic Mode (클래식 모드)**: 시간 제한 없이 자유롭게 단어를 찾는 기본 게임 모드. Word Pack을 순서대로 진행하며, 모든 단어를 찾으면 스테이지가 클리어된다.

**Time Attack Mode (타임어택 모드)**: 제한 시간 내에 최대한 많은 단어를 찾아야 하는 모드. 단어를 찾을 때마다 시간이 소량 추가되어 긴장감 있는 플레이를 제공한다.

**Marathon Mode (마라톤 모드)**: 스테이지를 연속으로 풀어 나가며 최대한 오래 진행하는 모드. 스테이지가 진행될수록 난이도가 점차 상승한다.

**Daily Challenge (데일리 챌린지)**: 매일 새롭게 제공되는 특별 퍼즐. 모든 플레이어에게 동일한 Grid가 주어지며, 완료 시 한정 보상을 획득할 수 있다.

### 1.5 수익화 및 보상

**IAP (In-App Purchase, 인앱 결제)**: 앱 내에서 실제 화폐를 사용하여 게임 아이템이나 콘텐츠를 구매하는 시스템. 코인 패키지, 광고 제거, 프리미엄 Word Pack 등이 포함된다.

**Rewarded Ad (보상형 광고)**: 플레이어가 자발적으로 광고를 시청하고 그 대가로 게임 내 보상(코인, 힌트 등)을 받는 광고 형식. 사용자 경험을 크게 해치지 않으면서 수익을 창출한다.

**Coin (코인)**: 게임 내 기본 재화. 스테이지 클리어, 보상형 광고 시청, Bonus Word 발견 등으로 획득하며, 힌트와 부스터 구매에 사용된다.

**Gem/Jewel (젬)**: 게임 내 프리미엄 재화. 주로 IAP를 통해 획득하며, 코인보다 희소성이 높고 프리미엄 아이템이나 특별 콘텐츠 해금에 사용된다.

### 1.6 메타 시스템

**Achievement (업적)**: 특정 조건을 달성했을 때 부여되는 보상 및 기록. 예를 들어 "100개 단어 발견", "10연속 Streak" 등이 있으며, 플랫폼 업적 시스템과 연동된다.

**Collection (컬렉션)**: 게임 진행 중 수집할 수 있는 요소들의 모음. 테마별 일러스트, 뱃지, 배경 등을 모으는 시스템으로, 장기 플레이 동기를 부여한다.

**Leaderboard (리더보드)**: 플레이어 간 점수나 기록을 비교할 수 있는 순위표. 글로벌, 친구, 주간/월간 등으로 세분화될 수 있으며, 플랫폼 리더보드 서비스와 연동된다.

### 1.7 기술적 게임 용어

**SeededRandom (시드 랜덤)**: 동일한 시드(seed) 값을 입력하면 항상 같은 난수 시퀀스를 생성하는 의사 난수 생성기. Daily Challenge에서 모든 플레이어에게 동일한 퍼즐을 제공하는 데 사용된다.

**Word Bank (단어 목록)**: 게임에서 사용할 수 있는 전체 단어 데이터베이스. 테마별, 난이도별, 언어별로 분류되어 관리되며, Grid 생성 시 이 목록에서 단어를 선택한다.

**Found Color (발견 색상)**: 플레이어가 단어를 성공적으로 찾았을 때 해당 셀에 적용되는 시각적 강조 색상. 단어마다 다른 색상이 순차적으로 지정되어 구분을 돕는다.

**Hint Color (힌트 색상)**: 힌트 기능 활성화 시 해당 글자나 셀에 적용되는 색상. Found Color와 시각적으로 구별되어야 하며, 일반적으로 점멸 애니메이션이 동반된다.

**Direction Snap (방향 스냅)**: 플레이어의 드래그 입력을 가장 가까운 유효 방향(가로, 세로, 대각선)으로 자동 보정하는 기능. 터치 입력의 부정확함을 보완하여 조작 편의성을 높인다.

## 2. Godot 엔진 용어

### 2.1 핵심 개념

**Node (노드)**: Godot 엔진의 가장 기본적인 구성 단위. 모든 게임 오브젝트는 Node이며, 트리 구조로 부모-자식 관계를 형성한다. Node2D, Control, AudioStreamPlayer 등 다양한 유형이 있다.

**Scene (.tscn 씬)**: 하나 이상의 Node로 구성된 재사용 가능한 단위. `.tscn` 파일로 저장되며, 다른 Scene에 인스턴스로 삽입할 수 있다. 본 프로젝트에서 Grid, Cell, UI 패널 등이 각각 Scene으로 구현된다.

**Signal (시그널)**: Node 간 통신을 위한 Godot의 이벤트 시스템. Observer 패턴을 기반으로 하며, 특정 이벤트 발생 시 연결된 콜백 함수를 호출한다. `emit_signal()` 또는 `signal_name.emit()`으로 발신한다.

**Autoload (오토로드/싱글톤)**: 게임 시작 시 자동으로 로드되어 전역에서 접근 가능한 Node. 씬 전환 시에도 유지되므로, GameManager, AudioManager 등 전역 관리 객체에 사용된다.

**PackedScene**: Scene 파일을 메모리에 로드한 리소스 객체. `instantiate()` 메서드로 실제 Node 인스턴스를 생성할 수 있다. Cell이나 UI 요소를 동적으로 생성할 때 사용된다.

**SceneTree**: 현재 실행 중인 모든 Node의 계층 구조를 관리하는 객체. `get_tree()`로 접근하며, 씬 전환(`change_scene_to_packed()`), 그룹 관리, 일시 정지 등의 기능을 제공한다.

**GDScript**: Godot 엔진 전용 스크립팅 언어. Python과 유사한 문법을 가지며, 엔진과의 긴밀한 통합으로 높은 생산성을 제공한다. 본 프로젝트의 주 개발 언어이다.

### 2.2 GDScript 키워드 및 콜백

**@export**: GDScript에서 변수를 Inspector 패널에 노출시키는 어노테이션. 에디터에서 직접 값을 수정할 수 있어, Grid 크기나 애니메이션 속도 등의 파라미터 조정에 활용된다.

**@onready**: Node가 Scene Tree에 진입한 직후(`_ready()` 호출 시점) 변수를 초기화하는 어노테이션. `$NodePath` 참조를 안전하게 캐싱할 때 사용된다.

**_ready()**: Node가 Scene Tree에 추가되고 모든 자식 Node가 준비된 후 한 번 호출되는 콜백 함수. 초기화 로직을 배치하는 데 사용된다.

**_process(delta)**: 매 프레임마다 호출되는 콜백 함수. `delta`는 이전 프레임으로부터의 경과 시간(초)이며, 타이머 표시 업데이트나 애니메이션 보간 등에 활용된다.

**_input(event)**: 사용자 입력 이벤트가 발생할 때마다 호출되는 콜백 함수. 터치, 마우스, 키보드 입력 등을 처리하며, Direction Snap 로직이 이 콜백에서 구현된다.

### 2.3 UI 시스템

**Control (UI 노드)**: 2D 사용자 인터페이스를 구성하는 기본 Node 타입. Button, Label, TextureRect 등 모든 UI 요소의 부모 클래스이며, 앵커와 마진 시스템으로 레이아웃을 제어한다.

**Container (컨테이너)**: 자식 Control 노드를 자동으로 정렬하는 레이아웃 노드. HBoxContainer(가로 배치), VBoxContainer(세로 배치), GridContainer(격자 배치) 등이 있으며, 반응형 UI 구성에 필수적이다.

**Theme Resource (.tres)**: UI 요소의 시각적 스타일(폰트, 색상, 아이콘, StyleBox 등)을 정의하는 리소스 파일. 테마 전환 기능 구현 시 Theme Resource를 교체하는 방식으로 처리한다.

**Viewport**: 렌더링 결과가 표시되는 영역. 기본 Viewport 외에 SubViewport를 사용하여 별도의 렌더링 영역을 만들 수 있다. 해상도 독립적 UI 구현에 활용된다.

**CanvasLayer**: 별도의 렌더링 레이어를 생성하는 Node. UI 요소를 게임 월드와 분리된 레이어에 배치하여, 카메라 이동이나 줌에 영향받지 않는 HUD를 구현할 때 사용된다.

### 2.4 애니메이션 및 오디오

**Tween (트윈)**: 코드에서 동적으로 생성하는 보간 애니메이션 시스템. 속성 값을 시작값에서 목표값으로 부드럽게 변환하며, 셀 선택 효과, UI 등장/퇴장 애니메이션 등에 사용된다.

**AnimationPlayer**: 키프레임 기반 애니메이션을 재생하는 Node. 여러 속성의 변화를 타임라인으로 관리할 수 있어, 복잡한 연출이나 셀 발견 이펙트 등에 활용된다.

**AudioStreamPlayer**: 오디오 파일을 재생하는 Node. 2D/3D 공간 음향이 필요 없는 BGM, UI 효과음 재생에 사용되며, 볼륨과 피치 조절이 가능하다.

### 2.5 데이터 및 입력

**FileAccess**: 파일 시스템에 접근하여 파일을 읽고 쓰는 클래스. 세이브 데이터 저장, Word Bank JSON 파일 로드 등에 사용된다. `user://` 경로를 통해 플랫폼별 저장 위치를 추상화한다.

**ConfigFile**: INI 형식의 설정 파일을 읽고 쓰는 유틸리티 클래스. 게임 설정(볼륨, 언어, 접근성 옵션 등)을 저장하고 로드하는 데 적합하다.

**InputEvent**: 사용자 입력(터치, 마우스, 키보드, 게임패드 등)을 나타내는 이벤트 객체. `_input()` 콜백의 매개변수로 전달되며, 입력 유형에 따라 다양한 하위 클래스가 있다.

**InputMap**: 논리적 액션 이름과 물리적 입력 키를 매핑하는 시스템. "select", "hint_use" 등의 액션을 정의하고, 키보드/게임패드/터치 입력을 하나의 액션으로 통합 관리한다.

**Resource**: Godot에서 데이터를 저장하고 공유하는 기본 클래스. `.tres` 또는 `.res` 파일로 직렬화되며, 커스텀 Resource를 정의하여 Word Pack 데이터, DDA 파라미터 등을 구조화할 수 있다.

**Timer**: 지정된 시간이 경과하면 `timeout` 시그널을 발생시키는 Node. Time Attack 모드의 카운트다운, 콤보 유지 시간 측정 등에 사용된다.

## 3. 기술 약어

### 3.1 아키텍처 및 알고리즘

**FSM (Finite State Machine, 유한 상태 기계)**: 시스템이 유한한 수의 상태 중 하나에만 존재하며, 이벤트에 따라 상태가 전이되는 설계 패턴. 게임 흐름(메뉴 → 플레이 → 결과)이나 셀 상태 관리에 활용된다.

**DDA (Dynamic Difficulty Adjustment, 동적 난이도 조절)**: 플레이어의 행동 데이터를 기반으로 게임 난이도를 실시간 조정하는 알고리즘. 게임 용어 섹션의 DDA 항목 참조.

**LCG (Linear Congruential Generator, 선형 합동 생성기)**: `X_{n+1} = (a * X_n + c) mod m` 형태의 간단한 의사 난수 생성 알고리즘. SeededRandom 구현에 사용되며, 동일 시드로 동일한 난수 시퀀스를 재현할 수 있다.

### 3.2 데이터 형식 및 개발

**JSON (JavaScript Object Notation)**: 경량 데이터 교환 형식. Word Bank, 스테이지 정의, 세이브 데이터 등을 저장하는 데 사용된다. Godot의 `JSON.parse_string()`으로 파싱한다.

**CSV (Comma-Separated Values)**: 쉼표로 구분된 값의 텍스트 파일 형식. 대량의 단어 데이터를 스프레드시트로 관리하고 게임에 임포트할 때 사용된다.

**SDK (Software Development Kit, 소프트웨어 개발 키트)**: 특정 플랫폼이나 서비스와 연동하기 위한 개발 도구 모음. AdMob SDK, Firebase SDK 등이 프로젝트에서 사용된다.

**API (Application Programming Interface, 응용 프로그래밍 인터페이스)**: 소프트웨어 간 상호작용을 위한 인터페이스 정의. 플랫폼 서비스(리더보드, 업적, 결제)와의 연동에 사용된다.

### 3.3 빌드 및 배포

**APK (Android Package Kit)**: Android 앱의 설치 패키지 파일 형식. 개발 및 테스트 단계에서 직접 설치하여 확인할 때 사용된다.

**AAB (Android App Bundle)**: Google Play Store에 업로드하는 Android 앱 배포 형식. APK보다 최적화된 앱 전달이 가능하며, Google Play Console에서 요구하는 표준 형식이다.

**SKU (Stock Keeping Unit, 재고 관리 단위)**: IAP 상품을 고유하게 식별하는 코드. 각 코인 패키지, Word Pack, 구독 상품마다 고유한 SKU가 부여된다.

### 3.4 비즈니스 지표

**ARPDAU (Average Revenue Per Daily Active User)**: 일일 활성 사용자 1인당 평균 수익. 게임의 수익화 효율을 측정하는 핵심 지표이다.

**DAU (Daily Active User, 일일 활성 사용자)**: 하루 동안 게임에 접속한 고유 사용자 수. 게임의 일일 활성도를 나타내는 기본 지표이다.

**MAU (Monthly Active User, 월간 활성 사용자)**: 한 달 동안 게임에 접속한 고유 사용자 수. DAU/MAU 비율은 사용자의 재방문 빈도(Stickiness)를 나타낸다.

**ARPPU (Average Revenue Per Paying User, 결제 사용자당 평균 수익)**: 실제 결제를 수행한 사용자 1인당 평균 수익. IAP 가격 정책 수립 시 참고하는 핵심 지표이다.

### 3.5 UX 및 성능

**UX/UI (User Experience / User Interface)**: UX는 사용자가 제품을 이용하면서 느끼는 총체적 경험이며, UI는 사용자와 시스템 간 상호작용이 이루어지는 시각적 인터페이스이다.

**FPS (Frames Per Second, 초당 프레임 수)**: 1초에 렌더링되는 화면 프레임 수. 본 프로젝트에서는 60 FPS를 기본 목표로 하며, 저사양 기기에서도 30 FPS 이상을 유지해야 한다.

### 3.6 라이선스

**OFL (SIL Open Font License)**: SIL International에서 제정한 오픈소스 폰트 라이선스. 상업적 사용이 가능하며, 본 프로젝트에서 사용하는 폰트의 라이선스 유형이다.

**CC0 (Creative Commons Zero)**: 저작권을 포기하고 퍼블릭 도메인으로 공개하는 라이선스. 아무런 제한 없이 자유롭게 사용할 수 있는 에셋에 적용된다.

### 3.7 푸시 및 광고

**FCM (Firebase Cloud Messaging)**: Google이 제공하는 크로스 플랫폼 푸시 알림 서비스. Android 기기에 Daily Challenge 알림, 이벤트 공지 등을 전송하는 데 사용된다.

**APNs (Apple Push Notification service)**: Apple이 제공하는 iOS/macOS용 푸시 알림 서비스. iOS 사용자에게 푸시 알림을 전달하는 데 사용되며, FCM과 연동하여 통합 관리할 수 있다.

**ATT (App Tracking Transparency)**: Apple이 iOS 14.5부터 도입한 사용자 추적 동의 프레임워크. 앱이 사용자의 활동을 추적하기 전에 명시적 동의를 받아야 하며, 광고 식별자(IDFA) 접근에 필수적이다.

**AdMob**: Google이 제공하는 모바일 앱 광고 플랫폼. 배너 광고, 전면 광고, 보상형 광고 등 다양한 광고 형식을 지원하며, 본 프로젝트의 주요 광고 수익원이다.

**IMA SDK (Interactive Media Ads SDK)**: Google이 제공하는 광고 통합 SDK. 동영상 광고 및 인터랙티브 광고를 앱에 삽입하는 데 사용되며, AdMob과 함께 광고 시스템을 구성한다.

## 4. 플랫폼 용어

### 4.1 TV 플랫폼

**Android TV**: Google의 텔레비전용 Android 운영체제 플랫폼. 리모컨 및 게임패드 기반 조작을 지원하며, D-pad 네비게이션에 최적화된 UI가 필요하다.

**Fire TV**: Amazon의 스트리밍 미디어 및 게임 플랫폼. Android TV와 유사하지만 Amazon Appstore를 통해 앱을 배포하며, 별도의 빌드 및 인증 과정이 필요하다.

**Leanback (린백)**: Android TV 앱 개발을 위한 UI 프레임워크 및 디자인 철학. 사용자가 소파에 편하게 기대앉아(lean back) 리모컨으로 조작하는 사용 환경을 전제로 설계된다.

### 4.2 TV 조작

**D-pad (방향 패드)**: 리모컨이나 게임패드의 상하좌우 방향 버튼. TV 환경에서 Grid 탐색 및 메뉴 네비게이션의 주요 입력 방식이며, 대각선 입력을 지원하지 않는 경우가 많다.

**Focus Navigation (포커스 네비게이션)**: TV 및 데스크톱 환경에서 D-pad 또는 키보드 방향키로 UI 요소 간 포커스를 이동하는 시스템. Godot의 `focus_neighbor` 속성으로 이동 순서를 지정한다.

**Safe Area (안전 영역)**: TV 화면에서 콘텐츠가 잘리지 않고 확실하게 표시되는 보장 영역. 일반적으로 화면 가장자리에서 5~10% 안쪽 영역이며, 중요한 UI 요소는 반드시 이 영역 안에 배치해야 한다.

### 4.3 빌드 및 배포

**Export Preset (내보내기 프리셋)**: Godot에서 특정 플랫폼(Android, iOS, Windows 등)용 빌드를 생성하기 위한 설정 묶음. 플랫폼별 아이콘, 권한, 서명 정보 등을 포함한다.

**Keystore (키스토어)**: Android 앱 서명에 사용되는 디지털 인증서 저장소 파일. APK/AAB 빌드 시 앱의 신원을 보증하며, 분실 시 앱 업데이트가 불가능하므로 안전하게 보관해야 한다.

### 4.4 플랫폼 서비스

**Google Play Games Services**: Google이 제공하는 게임 플랫폼 서비스. 업적, 리더보드, 클라우드 세이브, 멀티플레이어 등의 기능을 제공하며, Android 빌드에서 연동한다.

**Game Center**: Apple이 제공하는 iOS/macOS용 게임 플랫폼 서비스. 업적, 리더보드, 친구 시스템 등을 제공하며, iOS 빌드에서 연동한다.

### 4.5 스토어 및 콘솔

**Google Play Console**: Android 앱을 Google Play Store에 게시하고 관리하는 개발자 플랫폼. 앱 등록, AAB 업로드, IAP 상품 관리, 사용자 통계 확인 등의 기능을 제공한다.

**App Store Connect**: iOS 앱을 Apple App Store에 게시하고 관리하는 개발자 플랫폼. 앱 심사 제출, IAP 상품 관리, TestFlight 베타 테스트, 매출 분석 등의 기능을 제공한다.

**Amazon Appstore**: Amazon이 운영하는 앱 배포 플랫폼. Fire TV 및 Fire 태블릿 사용자를 대상으로 앱을 배포하며, Amazon IAP API를 통한 별도의 결제 시스템을 사용한다.
