## 1. 스테이지 연속 진행 규칙

스테이지 클리어 또는 실패 후 ResultScreen에서 플레이어가 선택할 수 있는 3가지 동작을 정의한다. 각 동작은 `GameController`의 상태 전이와 연동된다.

### 1.1 동작 정의

| 동작 | 버튼 텍스트 | 설명 | 상태 전이 |
|------|-----------|------|-----------|
| **Next** | "다음 스테이지" | 스테이지 +1, 새 랜덤 테마, 새 퍼즐 | COMPLETED → PLAYING |
| **Retry** | "다시 하기" | 동일 스테이지, 새 퍼즐 생성 (동일 테마 보장 안함) | COMPLETED/FAILED → PLAYING |
| **Home** | "홈으로" | 타이틀 화면으로 복귀 | COMPLETED/FAILED → IDLE |

### 1.2 버튼 표시 조건

| 결과 상태 | Next 버튼 | Retry 버튼 | Home 버튼 |
|-----------|:---------:|:----------:|:---------:|
| **COMPLETED** (클리어) | 표시 | 표시 | 표시 |
| **FAILED** (실패) | **숨김** | 표시 | 표시 |

> **설계 의도:** 실패한 스테이지를 건너뛰는 것을 방지하기 위해 FAILED 상태에서는 Next 버튼을 제공하지 않는다. 플레이어는 반드시 Retry로 해당 스테이지를 클리어하거나 Home으로 돌아가야 한다.

### 1.3 Next 동작 상세

```gdscript
## result_screen.gd 내부
## "다음 스테이지" 버튼 클릭 시 호출
func _on_next_pressed() -> void:
	var next_stage: int = _result.stage + 1

	# 1. 테마 랜덤 선택 (이전 테마와 다르게)
	var new_theme: String = ThemeRandomizer.pick_theme(_result.theme)

	# 2. 스테이지 번호 저장
	SaveManager.set_current_stage(next_stage)
	SaveManager.save()

	# 3. 게임 시작
	GameController.start_stage(next_stage, _result.mode, new_theme)

	# 4. 화면 전환: ResultScreen pop → 새 GameScreen push
	ScreenManager.pop_screen()
	var config := GridCalculator.get_stage_config(next_stage)
	ScreenManager.push_screen("game", {
		"stage": next_stage,
		"config": config,
		"theme": new_theme,
		"mode": _result.mode,
	})
```

### 1.4 Retry 동작 상세

```gdscript
## "다시 하기" 버튼 클릭 시 호출
func _on_retry_pressed() -> void:
	var same_stage: int = _result.stage

	# 동일 스테이지, 새 퍼즐 (테마는 ThemeRandomizer가 새로 선택)
	var new_theme: String = ThemeRandomizer.pick_theme("")

	GameController.retry_stage()

	ScreenManager.pop_screen()
	var config := GridCalculator.get_stage_config(same_stage)
	ScreenManager.push_screen("game", {
		"stage": same_stage,
		"config": config,
		"theme": new_theme,
		"mode": _result.mode,
	})
```

### 1.5 Home 동작 상세

```gdscript
## "홈으로" 버튼 클릭 시 호출
func _on_home_pressed() -> void:
	# 진행 상태 저장
	SaveManager.save()

	# 상태 초기화

	# Push 스택 모두 제거 → Tab 화면 복원
	ScreenManager.pop_screen()  # ResultScreen 제거
	ScreenManager.pop_screen()  # GameScreen 제거
	# Tab 화면(TitleScreen)이 자동으로 표시됨
```

## 2. 테마 랜덤 선택 알고리즘

### 2.1 ThemeRandomizer 설계

테마 선택 시 직전에 플레이한 테마와의 중복을 방지하여 신선한 경험을 제공한다. 해금된 테마가 2개 이상일 때만 중복 방지가 적용된다.

```gdscript
## theme_randomizer.gd
## 해금된 테마 중 하나를 랜덤으로 선택한다.
## 직전 테마와의 중복을 방지한다.
## TSD 3.2.18 ThemeRandomizer 기반.
class_name ThemeRandomizer
extends RefCounted


## 해금된 테마 목록에서 랜덤으로 하나를 선택하여 반환한다.
## previous_theme과 다른 테마를 우선 선택한다.
##
## [매개변수]
##   previous_theme: 직전에 사용한 테마 ID (빈 문자열이면 제한 없음)
##
## [반환값]
##   선택된 테마 ID 문자열
static func pick_theme(previous_theme: String = "") -> String:
	var unlocked: Array[String] = SaveManager.get_unlocked_themes()

	# 해금된 테마가 없으면 기본 테마 반환
	if unlocked.is_empty():
		return "animals"

	# 해금된 테마가 1개면 그대로 반환 (중복 방지 불가)
	if unlocked.size() == 1:
		return unlocked[0]

	# 2개 이상: 직전 테마를 후보에서 제외
	var candidates: Array[String] = []
	for theme in unlocked:
		if theme != previous_theme:
			candidates.append(theme)

	# 후보가 비어있으면 (이론적으로 발생하지 않지만 안전 처리)
	if candidates.is_empty():
		candidates = unlocked

	# 랜덤 선택
	return candidates[randi() % candidates.size()]


## 가중치 기반 테마 선택 (확장형).
## 최근 사용 빈도가 낮은 테마에 높은 가중치를 부여한다.
## 향후 통계 기반 최적화에 사용할 수 있다.
static func pick_theme_weighted(
	previous_theme: String,
	recent_themes: Array[String]
) -> String:
	var unlocked: Array[String] = SaveManager.get_unlocked_themes()

	if unlocked.size() <= 1:
		return unlocked[0] if not unlocked.is_empty() else "animals"

	# 최근 사용 빈도 계산 (최근 10회)
	var recent_count: Dictionary = {}
	for theme in unlocked:
		recent_count[theme] = 0
	for theme in recent_themes.slice(-10):
		if theme in recent_count:
			recent_count[theme] += 1

	# 가중치 계산: 빈도가 낮을수록 높은 가중치
	var weighted_pool: Array[String] = []
	for theme in unlocked:
		if theme == previous_theme:
			continue
		var weight: int = maxi(1, 10 - recent_count[theme] * 2)
		for i in range(weight):
			weighted_pool.append(theme)

	if weighted_pool.is_empty():
		return unlocked[randi() % unlocked.size()]

	return weighted_pool[randi() % weighted_pool.size()]
```

### 2.2 테마 선택 흐름도

```
pick_theme(previous_theme)
     │
     ├── unlocked_themes 조회 (SaveManager)
     │
     ├── 1개 이하? ─── Yes ──→ 기본 테마 반환 ("animals")
     │       │
     │      No
     │       │
     │       ▼
     ├── candidates = unlocked - previous_theme
     │
     ├── candidates 비어있음? ─── Yes ──→ unlocked 전체에서 랜덤
     │       │
     │      No
     │       │
     │       ▼
     └── candidates에서 랜덤 선택 → 반환
```

## 3. 테마 해금 조건 표

### 3.1 전체 테마 목록 및 해금 조건

총 8개의 테마를 제공하며, 플레이 진행에 따라 순차적으로 해금된다.

| # | 테마 ID | 테마 이름 (EN) | 테마 이름 (KO) | 해금 조건 | 기본 해금 |
|---|---------|---------------|---------------|-----------|:---------:|
| 1 | `animals` | Animals | 동물 | 기본 제공 | O |
| 2 | `food` | Food & Drink | 음식 | 기본 제공 | O |
| 3 | `space` | Space & Universe | 우주 | Classic 모드 **5 스테이지** 클리어 | X |
| 4 | `sports` | Sports | 스포츠 | Classic 모드 **10 스테이지** 클리어 | X |
| 5 | `science` | Science | 과학 | Classic 모드 **20 스테이지** 클리어 | X |
| 6 | `music` | Music | 음악 | Time Attack 모드 **5회** 클리어 | X |
| 7 | `ocean` | Ocean & Sea | 바다 | Daily Challenge **7일 연속** 스트릭 달성 | X |
| 8 | `mythology` | Mythology | 신화 | **모든 테마 각 1회 이상** 사용하여 클리어 | X |

### 3.2 해금 확인 로직

```gdscript
## theme_unlock_checker.gd
## 테마 해금 조건을 확인하고 새로 해금된 테마를 반환한다.
class_name ThemeUnlockChecker
extends RefCounted

## 모든 테마의 해금 조건을 정의한다.
const UNLOCK_CONDITIONS: Dictionary = {
	"animals": {"type": "default"},
	"food": {"type": "default"},
	"space": {"type": "classic_clears", "required": 5},
	"sports": {"type": "classic_clears", "required": 10},
	"science": {"type": "classic_clears", "required": 20},
	"music": {"type": "time_attack_clears", "required": 5},
	"ocean": {"type": "daily_streak", "required": 7},
	"mythology": {"type": "all_themes_used"},
}

## 현재 저장 데이터를 기반으로 새로 해금 가능한 테마를 확인한다.
## 새로 해금된 테마의 ID 배열을 반환한다.
static func check_new_unlocks() -> Array[String]:
	var newly_unlocked: Array[String] = []
	var current_unlocked: Array[String] = SaveManager.get_unlocked_themes()
	var stats: Dictionary = SaveManager.get_stats()

	for theme_id in UNLOCK_CONDITIONS:
		# 이미 해금된 테마는 건너뛰기
		if theme_id in current_unlocked:
			continue

		var condition: Dictionary = UNLOCK_CONDITIONS[theme_id]
		var is_met := false

		match condition.type:
			"default":
				is_met = true

			"classic_clears":
				var classic_clears: int = stats.get("classic_clears", 0)
				is_met = classic_clears >= condition.required

			"time_attack_clears":
				var ta_clears: int = stats.get("time_attack_clears", 0)
				is_met = ta_clears >= condition.required

			"daily_streak":
				var streak: int = SaveManager.get_daily_streak()
				is_met = streak >= condition.required

			"all_themes_used":
				# animals, food 제외한 6개 테마 + 기본 2개 = 7개 이상 해금 및 사용
				var used_themes: Array = stats.get("used_themes", [])
				var non_myth_themes := UNLOCK_CONDITIONS.keys().filter(
					func(t: String) -> bool: return t != "mythology"
				)
				var all_used := true
				for t in non_myth_themes:
					if t not in used_themes:
						all_used = false
						break
				is_met = all_used

		if is_met:
			newly_unlocked.append(theme_id)

	return newly_unlocked


## 스테이지 클리어 후 해금 확인 및 적용을 수행한다.
## 새로 해금된 테마가 있으면 SaveManager에 저장하고 알림을 발행한다.
static func process_after_clear(result: GameResult) -> Array[String]:
	# 사용된 테마 기록 업데이트
	SaveManager.add_used_theme(result.theme)

	# 해금 확인
	var new_themes: Array[String] = check_new_unlocks()

	# 새로 해금된 테마 저장
	for theme_id in new_themes:
		SaveManager.unlock_theme(theme_id)

	if not new_themes.is_empty():
		SaveManager.save()

	return new_themes
```

### 3.3 해금 알림 플로우

```
[스테이지 클리어]
     │
     ▼
ThemeUnlockChecker.process_after_clear(result)
     │
     ├── 사용 테마 기록 업데이트
     ├── 해금 조건 확인
     │
     ├── 새 해금 없음 → 종료
     │
     └── 새 해금 있음
           │
           ├── SaveManager에 해금 테마 추가
           ├── SaveManager.save()
           │
           └── ResultScreen에서 해금 알림 표시
                 │
                 ▼
           "새 테마 해금! 🌊 바다"
           (Toast 또는 팝업)
```

## 4. 스테이지 번호 저장 및 복원

### 4.1 저장 데이터 구조

스테이지 번호는 `SaveData`의 `current_stage` 필드에 저장한다.

```json
{
  "current_stage": 15,
  "stats": {
    "classic_clears": 14,
    "time_attack_clears": 3
  }
}
```

### 4.2 저장 시점

| 이벤트 | 저장 항목 | 설명 |
|--------|----------|------|
| 스테이지 클리어 (Next) | `current_stage += 1` | 다음 스테이지 번호로 업데이트 |
| 스테이지 클리어 (Home) | `current_stage` 유지 | 현재 클리어한 스테이지의 다음 번호를 저장 |
| 스테이지 실패 | 변경 없음 | 실패 시 스테이지 번호 변경 없음 |
| 앱 종료 | 현재 `current_stage` | 자동 저장 |

### 4.3 복원 로직

```gdscript
## title_screen.gd 내부
## 앱 시작 시 또는 홈 화면 진입 시 현재 스테이지를 표시한다.
func _update_stage_display() -> void:
	var stage: int = SaveManager.get_current_stage()
	stage_label.text = "Stage %d" % stage

	# 격자 크기 미리보기
	var config := GridCalculator.get_stage_config(stage)
	grid_size_label.text = "%dx%d" % [config.width, config.height]

	# 단어 수 표시
	word_count_label.text = "%d words" % config.word_count
```

### 4.4 스테이지 번호 무한 진행

스테이지 번호에는 상한이 없다. 격자 크기는 스테이지 41 이후 **10x10** 최대 크기로 고정되며, 단어 수는 **12개** 최대로 고정된다. 난이도는 DDA 시스템(P03_01)에 의해 단어 선택 전략으로 조절된다.

## 5. 최고 기록 시스템

### 5.1 기록 항목

모드별로 최고 기록(최단 클리어 시간)을 저장한다.

| 모드 | 기록 키 | 초기값 | 설명 |
|------|---------|--------|------|
| Classic | `best_times.classic` | `999.0` | Classic 모드 최단 클리어 시간 (초) |
| Time Attack | `best_times.time_attack` | `999.0` | Time Attack 모드 최단 클리어 시간 (초) |

### 5.2 저장 구조

```json
{
  "best_times": {
    "classic": 45.2,
    "time_attack": 28.7
  }
}
```

### 5.3 신기록 판정 로직

```gdscript
## best_time_tracker.gd
## 모드별 최고 기록을 추적하고 신기록 여부를 판정한다.
class_name BestTimeTracker
extends RefCounted

## 기본 최고 기록 (미달성 상태)
const DEFAULT_BEST_TIME: float = 999.0


## 스테이지 클리어 시 신기록 여부를 확인하고 갱신한다.
## 신기록이면 true를 반환한다.
static func check_and_update(result: GameResult) -> bool:
	if not result.is_cleared:
		return false

	var mode_key: String = _get_mode_key(result.mode)
	if mode_key.is_empty():
		return false

	var current_best: float = SaveManager.get_best_time(mode_key)
	var is_new_record: bool = result.clear_time < current_best

	if is_new_record:
		SaveManager.set_best_time(mode_key, result.clear_time)
		SaveManager.save()

		if OS.is_debug_build():
			print("BestTimeTracker: 신기록! %s: %.1fs (이전: %.1fs)" % [
				mode_key, result.clear_time, current_best
			])

	return is_new_record


## GameMode enum을 저장 키 문자열로 변환한다.
static func _get_mode_key(mode: int) -> String:
	match mode:
		GameControllerClass.GameMode.CLASSIC:
			return "classic"
		GameControllerClass.GameMode.TIME_ATTACK:
			return "time_attack"
		_:
			return ""


## 특정 모드의 최고 기록을 포맷팅된 문자열로 반환한다.
## 기록이 없으면 "--:--"를 반환한다.
static func get_formatted_best_time(mode_key: String) -> String:
	var best_time: float = SaveManager.get_best_time(mode_key)

	if best_time >= DEFAULT_BEST_TIME:
		return "--:--"

	var minutes: int = int(best_time) / 60
	var seconds: int = int(best_time) % 60
	return "%02d:%02d" % [minutes, seconds]
```

### 5.4 결과 화면에서의 신기록 표시

```gdscript
## result_screen.gd 내부
## 결과 데이터를 받아 UI를 업데이트한다.
func _display_result(result: GameResult) -> void:
	# 등급 계산
	result.calculate_grade()

	# 코인 계산
	result.calculate_coins()

	# 신기록 확인
	result.is_new_record = BestTimeTracker.check_and_update(result)

	# UI 업데이트
	grade_label.text = result.grade
	time_label.text = _format_time(result.clear_time)
	coins_label.text = "+%d" % result.coins_earned

	# 신기록 표시
	if result.is_new_record:
		new_record_banner.visible = true
		_play_new_record_animation()
	else:
		new_record_banner.visible = false

	# 버튼 표시 (실패 시 Next 숨김)
	next_button.visible = result.is_cleared
	retry_button.visible = true
	home_button.visible = true


## 시간을 "MM:SS.s" 형식으로 포맷한다.
func _format_time(seconds: float) -> String:
	var minutes: int = int(seconds) / 60
	var secs: int = int(seconds) % 60
	var tenths: int = int(seconds * 10) % 10
	return "%02d:%02d.%d" % [minutes, secs, tenths]


## 신기록 달성 애니메이션
func _play_new_record_animation() -> void:
	new_record_banner.scale = Vector2.ZERO
	new_record_banner.pivot_offset = new_record_banner.size / 2.0
	var tw := create_tween()
	tw.tween_property(new_record_banner, "scale", Vector2.ONE, 0.5)\
		.set_ease(Tween.EASE_OUT).set_trans(Tween.TRANS_ELASTIC)
```
